In message <200109242325.KAA006.02@tatsu.flybynight.com.au>, "Simon Coulter" writes:
>Q/  Does Xitami provide full support for HTTP/1.1?  Specifically HTTP 
>header directives relating to compression. This is part of the Content 
>Encoding and supports must_server and must_client (something like that) tag 
>directives.

As far as I know Xitami doesn't support compression at all at present.
As you noted the Xitami 2.5 beta (currently 2.5b5) does support many of
the core HTTP/1.1 features, such as virtual hosting, and keep alive
handling, but not all of them are completely debugged.

>If so, is it likely that they will be supported?  If so, any idea on 
>time frame?

They'll be supported when someone writes the code to support them :-)

Xitami maintenance is largely on a "as time is available" basis at
iMatix, as evidenced by the relatively unchanged nature of Xitami over
the past year or two.  Contributions of source code to support these
features is always welcomed :-)
