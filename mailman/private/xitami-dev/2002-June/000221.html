Hi all,

I have a client app written to POST compressed and encrypted data to a LRWP 
process. The problem is that the LRWP process only receives a few bytes 
instead of the whole data stream. Basically, it appears that the WSX module 
xilrwp.c punts on the first NUL it encounters in the stream. POST'ing text 
data works like a charm though...

Looking at the source code, it would appear that this issue has been 
resolved already (as indicated by the comments at the top of xilrwp.c). But 
the problem still exists as of the latest 2.5 codebase...Or am I missing 
something?

Has anyone written a fix for this issue (I'd like to know before I do it).

Regards,
Fabien.
