it's been my experience that the real port xitami uses for http is whatever 
you set it to, +80.  if you check your xitami config you'll likely see that 
your http server is set to port 0, and actually works on port 80.  it's very 
strange.  all that I know for sure, is that one day, my xitami stopped 
working, i tried everything, changed this setting to 80 (which previously had 
yielded only working http on port 160) and everything worked again
