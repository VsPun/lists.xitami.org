>I have a problem with receiving ASCII files with a xitami ftp server.  It
>is version 2.4d9 and running on NT 4 SP6a.
>The problem is with the carriage return line feed CRLF.  Xitami replaces it
>with line feed LF.
>I did found that question on this mailing list but never the answer.
>I guess there is a way to change that behavior or get xitami to put a CRLF

I don't quite understand your question. Xitami is a server, so you (or 
somebody else) put files on it so the whole world (or a subset of it) can 
get them. Do you mean that the files you download from your own server have 
lost their CRLF, replaced by a LF?

I don't think Xitami changes anything in the files.
It doesn't even know what a CRLF in a file is, it just see files as a chunk 
of binary data.
Probably the files were created using Unix?
