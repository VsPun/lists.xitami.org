I'm not sure I understand you correctly. Do you wish to redirect a
certain document
to a certain other server/document and again other documents to other
server/document123
or are all the requests regardless of requested document to be
redirected to the same url?

What does the query string parameter "invasion" represent? does the cgi
executable
have to use this parameter to put together the new url it will redirect
the user to?

If latter, you would need to remove the .exe extension from your
executable just as if it
were a directory and pick up the "invasion" from the environment extra
path info.
So you would have to alter the way your executable processes the
parameter. 
Alternatively you could copy your executable as many times as there are 
?invasion -type parameters and rename all these identical executables to
these
specific parameters without the .exe extension - so that they would be
individual 
documents.

It all depends on how you are to use the parameter "invasion".

I hope the above helps.
