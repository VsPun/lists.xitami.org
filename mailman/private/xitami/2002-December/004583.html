> do what you are looking for.  And 3rd, the "log file format string" might
> allow you to put a DNS/Hostname variable that writes the DNS name to the log

Finally, one should think VERY carefully about recording domain names
INSTEAD of IP addresses in a log file, because at least an IP address is
traceable, whereas a rogue client might also have faked reverse DNS set
up to camouflage the source and send you off on a wild goose chase.

In short, ALWAYS record the IP address, and also, if you wish, the domain
name into which rDNS translates this.  (However, of course, any decent
log analysis program would be able to perform that rDNS operation at any
later time: after all, this is not rocket science, and I recall doing
this on FTP servers that I was administering back in the 1980s, long
before TimB-L had invented the web.)
