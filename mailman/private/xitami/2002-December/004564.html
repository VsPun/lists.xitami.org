> If it is Windows based, try to get the latest PHP from:
> 
> http://www.php.net/downloads.php
> 
> When you install it, it should be able to find out how
> to install/write the
> correct settings on Xitami.
> 
> At last xitamis conf should look like this:
> 
> in the "filters" page --
> 
> file extension----------filter comand or script
> .php------------------c:\php\php.exe (the path to your
> php.exe)
> 
> Then you can add more extensions as .php3, phtml, etc.
> 
> You just tell xitami to "pass" this extensions to
> php.exe for processing.
> 
> One more thing:  many times you want an specific script
> that you found on
> the internet to work. This script is not working and you
> think php support
> is not working.
> But the most of cases is that the script is messing with
> the paths,
> specially when we talk Windows, (Unix Linux people are
> well aware of such
> things), or require mysql support for example.
> 
> The way to be sure that your php support is wrong
> configured is to run the
> test it comes with.
> 
> Then start reading this if  is not working yet
> 
> http://cvs.php.net/co.php/php4/win32/install.txt
> 
> Carlos
