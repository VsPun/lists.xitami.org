I can fully understand if there's a need to change IRC server for this 
project. I'm an IRC beginner myself, so I have no special reason for 
using one or the other. I will gladly join another server, if I'm told 
how and where. :o)

Let's have the info Russel, so we can get this project started for real.

Also I've read a lot of posts on this community subject. It seems as if 
there's as many opinions as there are a members on the list! On the 
whole I do think we can agree on one thing: An active and living 
community would be good for Xitami, and if it's good for Xitami, it's 
good for the people who use it.

So what to do? Should we just stay with the current solution, or should 
we try to create some sort of "hub" for Xitami info/discussion?

I would personally love to see a combined solution. Something along the 
lines of the mailing-list, a web forum and an IRC channel.

All this could be centered around a website where we tried to keep up 
with tips and tricks, problem solutions, case stories and general 
debate. I will gladly help out with hosting such a site.

Let's get this debate on the road. Xitami is IMHO a great program - it 
deserves a great platform for information/support/whatnot..
