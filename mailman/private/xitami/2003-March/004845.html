As for "virtual domain" vs. "subdomain" -- example, I have one static IP
address that I host 10 domain names on, like www.abc.com, www.xyz.com etc.
Are you referring to "sub domain" as 123.abc.com, 456.abc.com, 789.abc.com
etc. with the abc.com being constant and the 123, 456, 789 being the sub
domain?
So hence I am referring to a virtual domain being a top level domain names
(such as abc.com, def.com etc.) That is my case, several different TLDs.
I want to be able to set up PHP in a virtual domain that is stated in the
defaults.cfg file as such --

    www.abc.com=abc.cfg
