Anyone familiar with the Content Management System called Tiki? I am
attempting to run Tiki under the following conditions:

Xitami (latest version, win binary) on XP
mySQL (latest version, win binary)
PHP 4.2.3 (last version that had PEAR as part of the distro which is
required for Tiki)
Tiki 1.5

I have done the usual:

Loaded Xitami (no problems as usual :) )
Loaded mySQL (no problems as usual :) ) Added a user with full GRANT
privileges for Tiki use.
Loaded PHP4, no major hiccups, had to add a directory for SESSION_DATA
workspace as it was not added on installation.
Loaded Tiki.

After a little path editing everything seems to work well. A phpinfo() pops
back all the right stats.

The Tiki seems to load. However there is this annoying litle error reported
back by PHP --

         Notice: Undefined index QUERY_STRING in
......\Webpages\Tiki\tiki-setup.php in ln530

I went into Xitami to see if the QUERY_STRING in the CGI section was set. It
was not. flipped the bit but still get the error above. 

Xitami is working just fine. But I suspect that there is some other
configuration option I have overlooked!? If there are any Tiki developers
that know the solution, pass it on. 

TIA

John McGinnis
