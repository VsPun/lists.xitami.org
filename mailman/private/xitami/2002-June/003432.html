Well, FWIW, I've seen it too for some PHP-based mail programs, NOCC specifically. Basically, on my pitiful little server, NOCC needs one second per message in the inbox. I increased the PHP.INI timeout value from 30 to like 240, and that helps solve the timeout issue, but not the lack of speed issue.

Sometimes I will get the exact error message you are reporting immediately, even if there aren't that many messages in an inbox. I've just attributed it general slowness with PHP/CGI for Xitami for windows for some PHP functions.

In contrast, another PHP mailers, UbiMeao (am I spelling that right?) will present the inbox within about 5 seconds, regardless of how many messages there are. Save server and Xitami version.
