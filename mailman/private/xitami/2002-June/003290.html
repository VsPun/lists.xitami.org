In message <20020508213710.55508.qmail@web10105.mail.yahoo.com>, Ken Wiebe writes:
>seem to work best is if, rather than accessing the web
>interface via http://www.myserver.com/admin, it could
>be set to a port separate from the web server.  

Xitami doesn't support listening on multiple ports at once
unfortunately.  The closest that I can think of for what you want to do
is to use a web proxy in reverse-proxy mode in front of Xitami to filter
out the URLs you don't want to allow in; or just to rely on Xitami's
address checking feature in the .aut files, and ensure your firewall
allows that source address only to come from the internal machines.
