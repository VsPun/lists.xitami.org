Yeah, I get that:

Virtual        Real
/foo/            C:\blahdir\
/foo/            C:\xitami\blahdir\
/foo/            D:\something\blahdir

But what I am looking for is a way to put photos in a directory that are
unaccessable from a direct URL path.  I am trying to prevent users who
are not logged in from getting to photos they don't have access to see.
If I move them to an aliased directory, then users can simply type in
the alias path and still get to them.  If I move them non-aliased path
or a non webroot path, then the only way I know to serve the photos is
to have Perl serve up raw image data.  But that can take up alot of
server CPU.  If the files are in a non-aliases or non webroot path, then
a simple <img src=/photos/foo.jpg> won't work, the /photos/foo.jpg must
be in a valid HTTP path for the image tag to work.
