> > PASSWORD, and File Data.

There is absolutely no possibility of encrypting the username/password,
used for authentication by FTP.  This is because the protocol was
designed (over twenty years ago) to expect this information to be
transmitted in the clear (just as the same happens with the telnet
protocol for remote login to other machines).

So for his query:

> >    1. Is there a method to secure the actual data being transmitted
> > using the Xitami Server?

the answer might be yes, with third-party encryption of the data file
contents, but absolutely no way for the authentication phase.  The only
way that might be achieved would be through an ssh session to other
software on the server, and for the latter to permit FTP "login" only
from localhost.

FTP has been around a long time: I've never heard of anyone wanting
encryption of the authentication phase.  I suppose it might be
attainable, through one-time passwords: this would require changes to the
FTP server process.

> > provide this
> >       protection. I am just curious if this can be done using Xitami.
