> No, you don't have to edit each file.
> The http server adds these headers automatically, using a table located  
> in your configuration file (xitami.cfg)
>
> Example of your xitami.cfg file:
> [Mime]
>      default=*/*                         #   Default MIME type
>
>      #  Text MIME types
>      etx=text/x-setext
>      htm=text/html
>      htp=text/html
>      html=text/html
>      rtx=text/richtext
>      tsv=text/tab-separated-values
>      txt=text/plain
>      .=text/plain                        #   Files with no extension
>
>      #  Image MIME types
>      gif=image/gif
> (and so on)
>
>
> Of course, you could add individual headers to each file (that can be  
> done for cgi applications, so you can send the file in "chunked" mode,  
> or gzip the file or whatever that request non-standard headers) but it's  
> easier with one single configuration file.
> Don't modify the xitami.cfg file, but create (or modify) default.cfg
