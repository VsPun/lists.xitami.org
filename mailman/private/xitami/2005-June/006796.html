> To second what Gene said, NTFS can handle massive numbers of files per
> directory.  In fact, I'm not sure that there is a limit to the number of
> files per folder.  Technically, the maximum number of files on a single
> NTFS volume is over 16 billion billion, although WindowsXP imposes a limit
> of just over 4 billion.
>
> I think your issue is going to be performance of NTFS rather than quantity
> of files.  Microsoft recommends disabling short-file name generation if
> you're going to have more than 300,000 files per folder on an NTFS
> volume.  They provide some information and instruction for this at
>
http://www.microsoft.com/resources/documentation/Windows/XP/all/reskit/en-us/prkc_fil_tdrn.asp
>
> I would venture to say that your Xitami performance is only going to be
> limited by the performance of NTFS.

> > believe) so it's pretty good on bulky directories.  You can increase
> > performance two ways through windows.  Use a directory tree index scheme
> > to limit your folder/files per directory or (and I think the bigest
> > performance gain for you) would be windows disk Cache.  Windows disk
