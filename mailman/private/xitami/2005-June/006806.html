Indeed; however, the original exhortation to RTFM can still apply.
Sadly, hardly anyone these days bothers to read more than a smidgeon of
any documentation before launching into activity (perhaps the correct
verb is "floundering") on the grounds that /everything/ in computing is
meant to be intuitive.  It ain't.

When I first start using a new product, I read the /entirety/ of the
manual before doing anything with it (and, yes, when I got a VAX back in
1982, I /did/ read all 22 shelf-feet of manuals).  By reading a manual
(or collection thereof), even if one does not understand all the concepts
at first, many bits of jargon are introduced, and better enable one to
understand the rest.  [Also, I'm blessed with an eidetic memory, so will
know after one reading that "the bit about virtual hosts" starts half-way
down the right-hand page about one-third of the way through the book.]

Reading manuals on the screen don't really help one to understand
(although fine for "command reference" type reminders once one is
familiar with all the principles), so print out all 400+ pages or
whatever and settle down for an evening's reading.

Of course, in this day and age we're lucky if there IS a manual with any
software product.
