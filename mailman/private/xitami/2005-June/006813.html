Sorry about the confusion,


Your settings:
(Defaults.cfg)
[Alias]
phpmyadmin=3D"C:\phpmyadmin"

Your previous reference (where c:\pippo was the alias):
   ...when i navigate ""http://localhost/pippo/index.php"
=20
Your statement, in your web page, should have been simply =
"pippo/index.php", a simple file reference that allows the original =
alias to substitute for "pippo".

     As an example: <a href=3D"pippo/index.php"> refers to a local file =
(in your system) without having to resolve an HTTP reference to =
wherever.  I'm not certain whether an http reference will even use the =
alias substituition so you'll be looking for index.php in the pippo =
folder of your default directory.

SO, your current alias example would be simply referenced as =
"phpmyadmin/index.php", without no HTTP.  This, I believe, is the common =
shorthand reference for a local file.  The long way would be something =
like file://phpmyadmin/index.php. =20

BTW,
> In my php.ini I configured so:
>cgi.force_redirect =3D 0
>cgi.fix_pathinfo =3D 1
>register_globals =3D On

These settings only deal with php and have nothing to do with the alias. =
 It could interfere with the viewing of the index.php page, if the =
brower sees it.  The cgi.force_redirect=3D0 setting was the only =
required I had to make for my php application work.

Hope that was a little clearer,
Charles
