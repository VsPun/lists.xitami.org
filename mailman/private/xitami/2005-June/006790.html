In message <42A5F7E9.3020608@bleisteiner.net> Martin Bleisteiner writes:

[Quoting myself, but failing to provide an attribution]

> >What a strange browser your visitors must be using; an FTP server has no
> >capability of sending a Content-Type header, so it's purely a binary (or
> >text) stream to the client end.
> >A browser has absolutely no right to invent its own concept of a
> >Content-Type, *especially* when acting as an FTP client.
> >
> Every browser gets to know the file name from the server, including the 
> file ending. The browser may decide what to do with the downloaded file, 
> dependent on the user settings. Maybe the settings are to open the file 
> immediately using the default application according to the file ending. 
> Opera and MSIE do so.

It surprises me not one iota that MSIE is "helpful" enough to mislead its
users through imagining that the last few characters of a file name might
have some particular significance (after all, many viruses and worms rely
upon such broken behaviour).  However, I am a little surprised that Opera
should have joined the great unwashed.

Perhaps the OP should tell his/her visitors to cut'n'paste the relevant
parts of an ftp:-schema URI into a proper FTP client, and avoid browsers
altogether :-(

> --------------040404030403020608080404
> Content-Type: text/html; charset=ISO-8859-15
> Content-Transfer-Encoding: 7bit
> 
> <!DOCTYPE html PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN">

Please do not waste everyon'e time and storage by retransmitting
identical information in a more prolix format: HTML belongs ONLY on web
servers, and not in messages sent to mailing lists or newsgroups.
