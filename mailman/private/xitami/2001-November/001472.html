For those interested in helping out here's what I'd like to get from you:

1) A list of specific functions and/or features that the system must support
2) Any graphical or user interface ideas
3) Language considerations (PHP, ASP, whatever)
4) Database support if needed and what database

5) Ideas on how to build a system to prevent multiple users from overwriting
each others data since there is only 1 defaults.cfg file.
I'm currently thinking of storing all data in a database and when changes are
made to commit all changes to the database and then import the defaults.cfg file
and re-write the entire file from the database. However, this would mean that
you could not use the Xitami admin html screen to modify the settings.
