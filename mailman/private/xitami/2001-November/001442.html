I was just... messing around with my scripts and found one thing about
$PHP_SELF. Indeed, I encountered the problem long ago and just
invented a fix in my header file and even forgot about it :))

here's the PHP-side fix:

// win32 localhost workaround  -- berkus 2001.08.02 01:26 YEKST
if( strstr($SCRIPT_NAME, "\\\\") )
{
   $PHP_SELF = substr($SERVER_URL,0,-1).str_replace("\\\\", "/", $SCRIPT_NAME);
}
