This is how I got Perl running in Xitami
it allows me to use,

<!--#exec cgi="/cgi-bin/test.pl" -->

This SSI command
which is a potential security risk for your system
is only accepted if the environment variable
SSI_INSECURE is set to 'yes'
Under Windows NT
in the System Control Panel
under 'Environment
-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-
Start->Settings->Control Panel
 
Double click on System
 
>From the pop-up box select the Environment tab
 
 In the Variable entry field type SSI_INSECURE
 In the Value entry field type YES
 (Make sure it goes where you want it,
  System Variables probably.)
 
Click OK
 
Reboot?  (I don't know if this is required.)
-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-

under Windows 95/98 by adding
SET SSI_INSECURE=yes to the autoexec.bat. 
to run SSI.
If Perl is installed correctly?
On a windows system,
You then should be able to call
Perl from anywhere in your system.
This allows the shebang to be #! perl -w
and does not need to be like this
#! c:\perl\bin\perl.exe
this also allows the use of
#! /usr/bin/perl
for the shebang. so as long
as we don't use win32 specific modules
or system calls we can keep our programs
cross platform Compatible.

Running Windows98
Xitami v2.4d6

in your
autoexec.bat

SET PATH=C:\PERL\BIN;%PATH%
SET SSI_INSECURE=YES

defaults.cfg
[Filter]
    .shtm=perlssi
    .shtml=perlssi 
    .ssi=perlssi   
[Ssi]
    Enabled=1
    Exec=1
