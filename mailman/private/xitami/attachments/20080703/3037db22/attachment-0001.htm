<DIV><FONT face=Arial size=2>I believe Xitami Pro only allows you to have SSL 
running on the primary/real domain bound to the primary IP of the server and it 
won't work on virtual 'named' </FONT><FONT face=Arial size=2>domains.&nbsp; How 
exactly are you binding Xitami's multiple IPs to specific domains?&nbsp; Are you 
depending solely on DNS - it won't work using DNS based hosting?</FONT></DIV>
<DIV><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV><FONT face=Arial size=2>Read this info on Virtual Hosts: <A 
href="http://legacy.imatix.com/html/xitami/index8.htm#vhosts">http://legacy.imatix.com/html/xitami/index8.htm#vhosts</A></FONT></DIV>
<DIV><FONT face=Arial size=2></FONT><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV><FONT face=Arial size=2>1.&nbsp; Physical server must have multiple 
IPs</FONT></DIV>
<DIV><FONT face=Arial size=2>2.&nbsp; Xitami must be specifically bound to the 
desired IPs</FONT></DIV>
<DIV><FONT face=Arial size=2>3.&nbsp; The virtual hosts section in the .cfg file 
must have IP to vhost associations</FONT></DIV>
<DIV><FONT face=Arial size=2>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ie: 
x.x.x.255 = d_domain.cfg</FONT></DIV>
<DIV><FONT face=Arial size=2>4.&nbsp; You cannot bind named domains to SSL 
vhosts</FONT></DIV>
<DIV><FONT face=Arial size=2>&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp; ie: <A 
href="http://www.***d.com">www.***d.com</A> = d_domain.cfg</FONT></DIV>
<DIV><FONT face=Arial size=2>5. Make certain your [Server] global configurations 
are not conflicting your vhost configurations</FONT></DIV>
<DIV><FONT face=Arial size=2>6. Use DNS to make certain <A 
href="http://www.d_domian.com">www.d_domian.com</A> points to x.x.x.225 IP 
address and that Reverse DNS is properly working, SSL uses Reverse DNS to verify 
authenticity.</FONT></DIV>
<DIV><FONT face=Arial size=2></FONT>&nbsp;</DIV>
<DIV><FONT face=Arial size=2>Make certain the above is all in order and let us 
know if works out or not.</FONT></DIV>
<DIV><FONT face=Arial size=2>-Russel</FONT></DIV>
