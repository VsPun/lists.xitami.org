When you do the installation of Xitami it automatically creates a folder
in the WEBFOLDER directory called PRIVATE<br>
This is password protected.<br><br>
I have note made use of it myself as I use different techniques that are
more or less hidden from Xitami even, but study the user manual for
password and security settings for this folder and you will find out how
to do it.<br><br>
<br>
You should be able to define a sub folder for this folder for each
individual user you wish to grant specific access, and simply specify in
the defaults.cfg or xitami.cfg and related aut files how users usernames
and passwords are related to certain sub directories.<br><br>
I hope this helps you a bit on the way.<br><br>

Some attempts to block such downloads can only be done in HTML code with
the pragma meta tag for NOCHACHE however it doesn't work really well
because of how different browsers read the code.<br><br>
The best protection from download, is to not put the documents in the
public directory at all.<br>
If you need them online for your internal use, try keeping a copy only in
a password protected folder and not in the public path.<br><br>
