Just wanted to add my 2 cents about the Abyss web server.<br>
<br>
I only use it locally to run scripts and made the switch so I could
easily view page containing SSI calls, due to having problems with
Xitami not able to do this.<br>
<br>
<pre wrap="">Marc Doigny said: "Doesn't perform cgi-bin as well as Xitami".

Since I use Abyss to run scripts locally I have to say I've never had any problems at all with running scripts from the cgi-bin and found no difference between the 2 servers.

I don't use it to run any external sites so that may make a difference but I don't see how it would :)   Just wanted to mention this in case others want to try Abyss and may be turned off by the negative comments.

I'm also using Abyss, but i'm not impressed:
- Sometimes forget the correct headers (content-type....)
  In case i use content-encoding: gzip it means the user gets junk
displayed (had to switch off compression!)
- Doesn't perform cgi-bin as well as Xitami
- No hierarchical configuration (need to configure each host separately
instead of using globals)
- No easy plain ini files
- Doesn't handle multiple connections as well as Xitami
  (can't run an exe while objects on a page are being transfered)

It's not a router thing (tried 5 different brands), it's not a OS thing
(win XP SP2), Xitami was running fine on this server before.
Can't switch back since i put a lot of time rewriting all executables to
Abyss format (but i would love if i had plenty of time!)
