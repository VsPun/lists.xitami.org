At 07:25 AM 10/3/2002 -0700, john.uhlig@coconutcomm.com writeth:
>Good morning all...
>
>Is there a way to combine monthly access logs so that my log analyzer can
>process more than one month at a time?
>I have Xitami set up to cycle the logs monthly and I keep them, and I would
>like to get the trend over several months in one log analysis report.
>
>Thanks!

I would probably figure out how to put the logs into a database and then
extract whatever I need from there into a temporary file, analyze it, and
delete it.  This can allow the data to be located elsewhere until needed
locally.  By doing it this way, it also allows multiple servers to use the
same storage location for easy log monitoring access.

Database logs are by far more versatile to their file-based counterparts.
Databases will also guarantee data integrity, whereas most file systems can
get corrupted (I've seen Xitami mess up the log files before due to a
server crash), good DBMS' rarely face corruption issues.

Hope this helps!
