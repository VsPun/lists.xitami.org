>  trying to get the oracle web forms server to run under xitami...

Looking at that mess that Oracle wants you to set, I think you would
be best-off running Apache. (Or smacking some common sense into
Oracle, but that won't happen.)

Yes, the aliases probably map to Xitami aliases, but they are prone
to not work right when things get complicated.

SetEnv you could do just by starting Xitami from a BAT file that
sets environment variables, or with NT's global or user
environments.

SetEnvIf.... I wonder if you take out the if and set that/those
variables globally, will it work? Or did Oracle really get so messy
that it needs someone else (the webserver) to tell it where it is,
differently for different requests?

It is perfectly possible to run Xitami on port 80 and Apache on
another port on the same machine. I have two machines here working
that way. IMHO a better plan would be to run them on separate
machines.
