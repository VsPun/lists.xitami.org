> I didn't do anything different to use FP to build sites hosted..
> using xitami.

Sure. FP is capable of emitting 99%-good simple HTML.

BUT.... FP is -also- capable of sticking in Forms, Administration,
Hitcounters, Multi-user Authoring, Chat, and who know what else
features that will not work unless Front Page Extensions are loaded
on the server. Hey, they have to do something to buck Apache!

See http://office.microsoft.com/assistance/9798/hostingfaq.aspx

This is, as RO guessed, a set of CGI-like programs. (However MS
hates to do CGI, preferring their own proprietary code tricks. They
use "the Internet Server Application Programming Interface (ISAPI),
the near-universal Web server extension mechanism." I would not call
it "near-universal" at all... does anybody but MS and their
supporters use ISAPI?)

I may have been a bit hasty. There are FP Extensions for unix
servers. Specific servers on specific unixes. These do use the CGI
interface. It might be possible to hack one of those into working
with Xitami.

And isn't there a product to support ISAPI on Xitami? Seems to me
I've seen that. And ignored it, because ISAPI looks like a bad idea
to me, and bug-bait too.

As a "quick answer", I still think "No" is correct for FP Extensions
on Xitami.
