> The .cgi program is in the Main HTML Directory's cgi-bin directory
> which is directly under the Main HTML Directory (A.K.A. root
> directory)

Normally you do NOT want to do that: put the cgi programs inside the
public HTML directory system.

The usual setup is:

Web pages: c:/xitami/webpages
CGI files: c:/xitami/cgi-bin

Try putting your PL files there, and verify that Xitami's config
knows that this folder is CGI, and that any URL with "cgi-bin" in it
should point there.

It is possible to serve CGI from anywhere. But it opens extra
security issues, and is usually not wise.

As for CGI inside SSL --- I never got that to work. Probably I am
stupid. Possibly the rules are different than when calling CGI
directly.
