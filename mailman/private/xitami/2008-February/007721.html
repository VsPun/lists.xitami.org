>  Comments on the directory/file structures and what kind of DB you would

I'd suggest as a start that we port into a simple structured XML
format that can be reused by different people afterwards.  There
should be a tool that does this for pipermail archives... if it does
not exist, it would be a nice thing to make.

Then, secondly, there are different ways to use this.  Some people
have suggested a forum/email combination.  That would be lovely, it's
something I'm pushing for in Wikidot.  If we ever get it, we could
import the email data into that... but it's a future prospect.

So in the short term, importing the email data into a searchable
database seems doable, and definitely useful.
