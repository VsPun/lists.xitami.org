>>

>>them with another cfg file (and this is why you should never change the
>>Xitami.cfg and Xitami.aut files).

>>>>Thirdly,
>>>>
>>>>the configuration files in Xitami are read in cascading order.
>>>>At the very core is the Xitami.cfg - which you should never ever
>>>
>>>change.
>>>
>>>
>>>
>>>>After that there is the Defaults.cfg - used to customize the server.
>>>>In the Defaults.cfg are your virtual host configuration entries..
>>>>These entries specify individual .cfg files for all your virtual
>>>
>>>hosts.
>>>
>>>
>>>
>>>>Then these specified .cfg file are read by Xitami for further
>>>>customization per virtual host.
>>>>
>>>>So the priority Xitami follows right now for all server rules is
>>>>
>>>>1) solarcharts.cfg
>>>>2) defaults.cfg
>>>>3) xitami.cfg
>>>>
>>>>Any rule missing in the solarcharts.cfg is read from defaults.cfg and
>>>
>>>if
>>>
>>>
>>>
>>>>not there, it's read ultimately fron Xitami.cfg . 
>>>>
>>>>The same applies to the .aut files because they were specified.
>>>>Because in your defaults.cfg you are stating everything is
>>>
>>>solarcharts.
>>>
>>>
>>>
>>>>Then Xitami goes to solarcharts.cfg and sees that solarcharts has an
>>>>.aut
>>>>file of its own, the solarcharts.aut. It goes to solarcharts.aut and
>>>>sees
>>>>
>>>>[/Admin]
>>>>  admin=-
>>>>
>>>>AND THIS IS THE ADMIN for Xitami. 
>>>>The user name is "admin" and the password is "~" .
>>>>No matter what your defaults.aut says Xitami will read the
>>>>solarcharts.aut 
>>>>because according to your defaults.cfg right now, 
>>>>"*" everything = solarcharts.cfg .
