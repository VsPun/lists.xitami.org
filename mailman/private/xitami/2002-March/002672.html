In message <003501c1c4a1$0b0e7480$037ba8c0@pooh>, "Stan Schretter" writes:
>I am using an  LRWP connection to Xitami from a Visual Basic program. All
>seems to work OK most of the time, but occasionally it seems that the Xitami
>server locks up and doesn't send anything over the inerface. Needs to be
>manually reset. Possibly the TCP connection is hung.

Chances are really strong that your LRWP program isn't "playing by the
protocol" in some respect, and what you're seeing is a deadlock rather
than a hung TCP connection.  You might want to double check your code is
sending the right amounts for each thing, and at least as
importantly that it's being flushed out to the network not buffered
somewhere.

Network sniffing/snooping can be very valuable in debugging this type of
issue.
