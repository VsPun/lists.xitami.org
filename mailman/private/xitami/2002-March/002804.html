In message <1371986706282.20020321152058@rci.rutgers.edu>, Paul Reid writes:
>> simulate a slow internet connection
>
>Put the files on a floppy disk, and point your browser at it.
>
>That's not an exact measurement. But if it sucks from floppy, it
>will suck on modem.

This isn't a bad "first cut" metric.  Floppy drives tend to get
30-50KB/second, which is roughly equivilent to a slow ADSL or cable modem 
link (say, one up to 512Kbps).  

You need to make sure the floppy isn't cached though; so write them to
floppy on one machine, and then test it on another one, to be sure.

As someone else mentioned there is some traffic shaping code in the
Xitami 2.5 beta that you could try.  Some operating systems, such as
FreeBSD (and IIRC recent Linux kernels) also have traffic shaping code.
A friend of mine used the FreeBSD traffic shaping to test one large
website for "modem speed" to good effect.

Don't forget the KISS solution -- grab a PC with a modem, dial out to
the Internet, and then try it :-)  (This is also useful for checking
that you didn't accidentally include any links that only work
internally.)
