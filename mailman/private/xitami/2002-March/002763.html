In message <web-980150@mi-ke.org>, "mike" writes:
>I want two web server running on one computer. The first 
>one should be xitami on port 80 listening to all IP 
>addreses. The second one is a zope server running on port 
>8080 only listening to 127.0.0.1 (localhost). If there is 
>a request for the zope server (maybe 
>http:/my.xitami.server/my-zope-server from the internet 
>the xitami server should take this request, should ask the 
>zope server for content and should give the content back 
>to the internet user.

This isn't "rewriting" so much as "proxying".  Not that it matters too
much in this instance -- Xitami doesn't support doing either at present.
You could probably write a LRWP program to do something like this if you
wanted.  (Webservers like Apache support both URL rewriting and proxying
like you mention.)
