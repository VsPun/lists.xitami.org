Nothing in there "should" delete files. However, this code may work
quite oddly under Windows:


sub createDirectories {
    mkdir ("users", 0777) unless (-d "users");
    chdir("users");
    for ($i = 0; $i < 128; $i++) {
                mkdir($i, 0777) unless (-d $i);
    }

If I read it right, it makes folders with names based on IP numbers,
and chmod style permissions. This would work on unix. It "should"
degrade gracefully on NT and Win9x, but you are in an area that I
think WinPerl isn't 1,000% tested.

I'm also curious how it knows where to find its files.

You haven't told us if you run Windows, or if it has been configured
correctly, or if there is a support website that we might study.
