>Ewen McNeill wrote:
>> In message <018c01c16715$49549a60$0a00a8c0@enigmacomputers.com>, "Soulkoff" writes:
>> >Is there any way to get /admin to give a 404 or some such, as if it doesn't
>> >exist?  Disabling the WBA doesn't seem to do this.
>> 
>> You need to disable the WSX entry as well.  Unfortunately (due to a bug)
>> I think the only way you can do this is to edit xitami.cfg, and comment
>> out the line:
>> 
>>     admin=xiadmin
>
>[Wsx]
>  admin=""
>
>works for me in defaults.cfg. 

What this says is 'there is a /admin, and it is handled by the "" module', 
which is why you get the 501.  To completely hide the URL, so there is 
no /admin, you need to comment out the line in xitami.cfg instead, and
then you should get the 404.

If you can live with the 501, then your entry in defaults.cfg is tidier,
and less likely to break next time you upgrade xitami.
