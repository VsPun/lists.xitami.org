In message <EIEFKMGPDJCOPNFNDKIACEGODCAA.phpshop@phpshop.org>, "Pablo" writes:
>I have a site that has a directory called "Admin" just off the root
>directory.  I try to access it and I get the xitami admin login prompt!  How
>do I rename the admin area path to something else?  

You _should_ be able to rename this by editing xitami.cfg, in the [WSX]
section, and changing the "admin=xiadmin" entry to something else, eg
"xitamiadmin=xiadmin".  I'm not certain that everything there is set up
so that it can be renamed, but if you remove that entry then it should
be completely gone.

Note: you'll have to edit xitami.cfg (which we don't usually recommend),
rather than trying to override it in defaults.cfg; there is a bug
reported for this (ie, that it should be possible to disabled WSXes in
defaults.cfg)
