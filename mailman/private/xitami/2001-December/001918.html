General comment on ftp through cable modems. 
 
My ISP was blocking port 80 but not 21, so I had to use a different www
port (I ended up using 443, ssl port, which got around problems with
some firewalls [a friend's at work] which was blocking unusual ports to
unknown ip addresses)
 
The problem was that Xitami would accept the FTP connection and login on
port "which ever" (other than 21), but when my ftp client asked for a
"list" there was either no response from Xitami, or for some reason my
Linksys cable router, or the cable company was blocking the return list.
I got around it by disabling Xitami's FTP and used another FTP program.
 
Frank Harrell
http://CowboyFrank.net  http://asgra.org

Default will be port 21 and your ISP (must allow this on port 21.)
The default port address of 21 can be changed using the portbase=
in the [Server] section of defaults.cfg if needed. 

In the defaults.cfg file:   (this is the minimum requirement)
[FTP]
enabled=1                        #enable ftp
user-file=ftpdirs.aut        #point to password/authorization file

In the ftpdirs.aut file:
[MyFtp]                                    #folder name
user1=password1              #user1 password is password1
users2=password2            #user2 password is password2
