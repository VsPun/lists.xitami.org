Ewen, I think Bruce was asking "How do I use HTTP for FTP-type
stuff?" rather than "How is HTTP better?"

It isn't quite clear what Bruce is doing. But I turn to FTP for
many small chores. My text-editor has a "Save As... FTP" that
opens an FTP server as if it were a local disk (almost), but no
HTTP protocol there. Sucking a LOT of files from one machine to
another (as when moving a site from one host to another) is
natural in a smart FTP client (WS_FTP understands "move directory
structure" as a sequence of basic FTP commands). 

While I'm dimly aware that HTTP can support all this, it isn't so
often done. And I thought Xitami did not natively support HTTP
PUT?
