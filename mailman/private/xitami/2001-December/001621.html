In message <002d01c17d4d$a9d66b40$0ddcf4d0@dds.davidge.com>, "Erik A." writes:
>My TCP/IP is installed on the machine, but i keep getting that message that
>says:
>Could not open HTTP port 80-Port is already used by another server
>(WSAEADDRINUSE).

In that case there is probably another HTTP server running -- perhaps
IIS if you are using Windows.  (All sorts of things install bits of IIS
because they think you need it :-( )

>From a shell prompt do:

netstat -na

and look for a "...:80...... LISTEN" line; if you see one (before
starting Xitami) then you've definitely got another webserver running.
In which case look at the task list for likely suspects.

If you don't want to stop the other server, you can run Xitmai on 
another port with:

xitami -b BASE

where base is the number to _offset_ all the Xitami ports by, eg:

xitami -b 5000

makes the HTTP port 5080, so you need to use URLs like:

http://localhost:5080/....
