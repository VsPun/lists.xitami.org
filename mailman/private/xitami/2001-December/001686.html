In message <3C0A8E0A.C220BC82@russianteam.org>, Sergey writes:
>Is there any way to limit number of simultaneous request per IP, or may
>be you know another way to disable webaccelerators?

There's not, currently, a way to limit the number of requests by IP
address.

One thing you might want to consider (in the 2.5 beta) is using the
bandwidth shaping feature, to limit the amount of traffic that is sent
out to particular locations -- so that they don't get any benefit from
making more than one request in parallel.  See pipedef.xml, and the
related documentation.
