> If it's a "cgi" then it should be "/cgi-bin/CGINAME"; ie, it should be a
> relative URL not a full URL.  This is particularly necessary with Xitami
> 2.4 and the PerlSSI, which takes a fairly naive approach to resolving
> the path (stick the CGI directory name in front of CGINAME and hope for
> the best).
>
> If you're having trouble make sure you've got execution of CGIs turned
> on, and check cgierr.log for any errors running the CGI.
