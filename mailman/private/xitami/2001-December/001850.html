In message <5C6A0688-FB65-11D5-8692-0003933D8C18@mac.com>, Bruce Axtens writes:
>If I ftp from my iBook to my desktop, via ethernet crossover cable and 
>static non-routable class C addresses, no problem.
>But if I ftp from my iBook across the internet I can log on and chdir, 
>but I every time I try to NLST, it times out. I've tried 3 different ftp 
>clients to the same (non) effect.

Do you have a firewall in between the Internet and your Xitami server?  
It looks a whole lot like the data channel connection is being blocked
by your firewall.  You might want to try active mode FTP (there's a
slightly better chance of it working in the case of the FTP server being
behind a firewall); most programs default to passive mode these days
(because it's got a better chance of working when the _client_ is behind
a firewall).

(And yes, if both the client and server are behind different firewalls
it is quite possible for FTP to be completely broken, so that neither
passive or active FTP will work.  Don't use FTP.  Use HTTP.  It's much
cleaner.)
