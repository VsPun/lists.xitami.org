In message <20011129084009.5C39.SCOTT@therubberroom.org>, Scott Tyson writes:
>"Cannot spawn child process".  So my questions are:
>1. Can Xitami support PHP using aliases.

Not really, no.  In theory it should work, but in practice, it doesn't
at present.

>2. Can Xitami support PHP using DNS based vitual hosting (I need to host 3 doma
>ins on one IP address)

Yes, virtual hosts are fine with PHP -- in fact this is the recommended
work around for problems with PHP and aliases.

>3. Recomendations on how to run Xitami so it is not running as the system 
>account.  Just create a basic user like IIS does and give that user access 
>to only my WWW directories?

Yup, that's the easiest solution.  You can tell the Xitami service which
user to run as, and if you set up a uesr _just_ for Xitami (like IIS
does for itself) then you can restrict what it can do quite easily.
