Hi!
Surely it's a question of the used version.
As far as I know the SSI-functionallity is included since version 2.5.
I use version 2.5b4 and the option is available exactly as Marcio =
describes it.
The best solution would be  to update your Xitami.
Otherwise you have to add to the filter-section of your defaults.cfg =
file:

   [Filter]
       shtml=3Dperlssi               #   Parse files with .shtml =
extension and pass it to perlssi which is a "FakeSSI"

and further the doku says:

#   The exec cmd=3D tag is only recognised if you set the environment =
variable
#   "SSI_INSECURE=3Dyes", in your autoexec.bat or system environment =
(for NT).



Best regards

Torsten
