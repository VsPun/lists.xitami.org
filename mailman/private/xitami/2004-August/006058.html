The version I am running is 2.5c2 - downloaded a week or so ago. All the HTTP 
directories are in WEBPAGES in the master XITAMI directory and work just 
fine. I don't really care where I put the files for FTP downloading - I have tried 
an FTPROOT directory in the XITAMI folder and also the same directory in the 
WEBPAGES folder.

In any case, I made the defaults.cfg copy and despite tinkering with the ftp 
directory in the file, I typically get some version of a "file not found" 
notice when I try anonymous FTP access. Occasionally I will get a directory 
listing where the text files suggest that I have ended up in the PULBLIC directory, 
but that is rare. 

By the way, both HTTP and FTP on the same bunch of files has been working for 
seven years on an old machine running Microsoft PWS so I know the basic 
internal lnks are sound. The HTTP side of the site is now much superior to the old 
one now that I am using Xitami, so it is just a bit frustrating trying to get 
the FTP elements working. A sample can be found on:

http://taggart.glg.msu.edu/gyro/gbee.htm

If you scrioll down the page, you will find a link to the GYROBEE LOGO - the 
link points to:

ftp://taggart.glg.msu.edu/gblogo.gif

and would, with the old system, result in the retrieval of the GIF file. All 
the FTP links are intended to be that simple - just a way to retrieve 
miscellaneous files that are stored on the system and intended for use by visitors to 
the site.

Thanks for your reply.....Ralph
