>I'm running Xitami with 3 virtual web sites, two of which are mapped to the
>default config on Windows NT 4.0.  [....]
>Periodically, all web accesses return HTTP error 403 and I can't get access
>to any pages unless I reboot the server.  Stopping and starting the web 
>service doesn't help.

403 is "Forbidden", which generally means that there is some access rule
which is prohibiting access at all (ie, not even a password would be
enough).

I'm puzzled as to what Xitami could be doing such that this error would
suddenly occur, and then not go away when Xitami was restarted, but go
away when the server was restarted.  If Xitami was causing it due to
some strange bug I'd expect that it'd either go away when the web
service was restarted, or it'd last after the server was rebooted.

Does this apply to all attempts to access the pages (including from the
machine running Xitami, the local LAN, etc)?  What is Xitami logging for
those requests (in access.log, error.log)?
