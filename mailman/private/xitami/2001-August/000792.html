>It "should" work to specify "localhost" as the "remote" server, and TCP
>as the protocol if that is needed.
>
>Then when PHP opens a connection to the "remote server" named
>"localhost", the TCP stack re-directs it to the same machine. Change
>"localhost" to "server2.your_isp.com" for production. 
>
>In some cases, "localhost" isn't correctly specified. "127.0.0.1" should
>also work to mean "this same machine". 
>
>In some cases, for no good reason, Windows will insist on connecting to
>an outside DNS server even for hosts it has in its own "hosts" file.
>Microsoft just doesn't understand DNS or TCP. It may work if you unplug
>your modem phone line and wait patiently for it to give up trying to
>reach an external DNS server. You can check that by seeing if opening
>"localhost" in your browser works without Windows trying to reach out
>for DNS service.
