In message <01C10E1F.0F9877F0.boyan@sanbolic.com>, Boyan Brezinsky writes:
>In the non-working case the generated output is (from *.flt):
>X-Powered-By: PHP/4.0.4
>Location: main.php3?menuAction=boMyDCL.show
>Set-Cookie: DCLINFO=1/0/dfe2c07de313f0d7ca27146e068767f1/en/default
>Set-Cookie: DCLTYPEMENU=0
>Content-type: text/html
>[....]
>As you can see, the difference is in the "Location:" header. The obvious 
>reason is that Xitami can't find the file specified in the Location header 
>and returns 404.

Ah.  Xitami basically expects that there will be an absolute reference
in the Location: line, and assumes that anything without a leading "/"
is in the root directory.  Hence the behaviour your saw.  

Other webservers are more forgiving (or more accurate, but AFAICT this
is an extension) and try the referenced document in the same directory
as the one that generated the reference.

IIRC we've already got a bug against this issue, and when someone gets
the time we'll try to make the Location: handling a little smarter.

The Meta refresh works in this instance because it relies on the web
browser to do the reloading, and the web browser has its own ideas about
defaults.
