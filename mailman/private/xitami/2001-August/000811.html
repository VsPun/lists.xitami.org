In message <BHEDLLPJFABBFBAIOHGOMEFOCJAA.LeeStewart@bigfoot.com>, "Lee Stewart" writes:
>I have a CGI program that needs to query another program that's running on
>the server.  I'm trying to get the title of the app using FindWindow() and
>GetWindowText().  My program works fine, but fails when running as a CGI
>program ("The specified module could not be found").
>
>Is there something weird about the way that Xitami calls a CGI program?

Possibly.  Xitami calls the program as, IIRC, a console program.  If
Windows is anything like OS/2 in terms of calling API, this will mean
that it can't call "windows" like functions -- such as
FindWindow()/GetWindowText().  

One thing you might want to try is having a small batch file that Xitami
can run as the CGI which then runs your program.  That way
command.com/cmd.exe will be involved, which does some hunting around to
figure out what type of program to run, and starting it the right way.
