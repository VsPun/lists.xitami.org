In message <3B7D7C2E.1C73002D@rci.rutgers.edu> Paul Reid writes:

> And what is all this "\\"? Use "/". I know DOS says "\", and c uses "\"
> as a metacharacter so you get "\\", but Xitami will translate "/" to the
> necessary path separator. And BTW, recent Windows accept "/" internally. 

It's not just "recent Windows"; internally (e.g. in functions to
open/close/delete files within a programming language, such as C, or a
scripting language, such as Rexx) one can use the forward slash as a path
separator.  I have done this in programs which have run under OS/2,
Windows 3.1, DOS (right back to v3.0), etc.

It is only if one must generate a command-line and pass it to a CLI, that
one requires backslashes.  If I find myself needing that in a program, I
have to remember to pass the '/' through a translate routine to '\'.
