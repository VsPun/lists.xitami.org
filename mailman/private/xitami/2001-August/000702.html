In message <3B78D8AE.5A458481@sovik.no>, Andreas Zieritz writes:
>My specific problem is that, after compiling the program and copying the
>configuration file to the /cgi-bin directory I am constantly getting a
>'I could not open the configuration file' error. The upload.cfg has
>664-rights in the /cgi-bin directory.

Your CGI script and Xitami differ as to their idea of what the current
directory should be when the cgi script is run, and your CGI script is
using a relative path.  

By default Xitami sets the current directory to the CGI directory when
running CGI scripts; but it seems that your CGI script expects something
else.

You can either change your CGI script to have a full path, or change its
assumption about what the current directory will be, or change what
Xitami sets the current directory to when running CGI scripts ([CGI]
workdir).
