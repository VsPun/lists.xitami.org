> I've configured Xitami to use port 5080 , although it is displaying as port 5160 in the admin console, 

Xitami assumes port 80; the number you put in the config is ADDED to 80
to get the actual port. So "0" = Port 80; "5080" means Port 5160
(5080+80). 

This is noted in a comment if you edit the file by hand: 

    portbase=0                          #   e.g. 5000 for port 5080

In the Admin screens, HELP tells you: 

portbase - Defines the IP port base for the server. Add 80 to this to
calculate the actual port used for the HTTP service. For example, to run
the server on port 6080, define portbase as 6000. We use a portbase in
place of an absolute address so that multiple protocols (e.g. the FTP
server embedded into Xitami) can be managed as one unit. 
Default: 0 (zero)
