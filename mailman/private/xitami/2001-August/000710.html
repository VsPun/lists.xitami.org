> <form action="../myscript.py" method=get>

Unless I'm mistaken, the GET method appends the form data to the URL:

http://mysite.com/myscript.py?var1=3&var2=foobar

The & symbol is used to seperate variable name/value pairs, so could it be
that when the form input is parsed, the ampersand is (incorrectly, in this
case) used as a delimiter?  I'm pretty sure this would cause the
problem you're experiencing.  Would using POST instead of GET as the form
method make a difference?  Hope this points you in the right direction.
