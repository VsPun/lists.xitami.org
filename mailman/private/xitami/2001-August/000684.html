In message <20010812075143.1555.cpmta@c000.lhr.cp.net>, Raffaele Corbisiero writes:
>I am trying to use servlets with Xitami. After downloading and successfully 
>installing Xitami and the servletrunner everything starts up fine 
>including the servletrunner. It finds my servlet.
>However, when a request for a servlet is made 
>the servletrunner crashes with; 
>  
>C:\Xitami\sr-beta3>java com.imatix.ServletRunner.ServletRunner 
>java.lang.AbstractMethodError 

Either (a) you're using the wrong version of the JSDK (servlet API) when
you compiled your servlet, or (b) you're using the wrong version of the
JSDK when you're running your servlet.  Hence the methods
provided/required don't match, hence the error you get.

The iMatix Servlet Runner uses a fairly old JSDK (1.0 or 1.1, I forget)
and won't work with the newer ones without changes because, as usual
with Java programs, the interface keeps changing all the time.
