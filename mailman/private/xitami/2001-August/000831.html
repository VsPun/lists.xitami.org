You can compile php as a cgi. If you want a compiled in server extension (WSX)
you would have to write the interface yourself, take a look at the various sub
directories of the php 'sapi' directories to see how various server api's are
implimented - unfortuately Xitami isn't one of them.

Quite frankly I wouldn't take this route as LRWP is a fairly simple protocal &
is likely to cause you far less grief than WSX. You could use the fastcgi
interface in php's sapi/fastcgi directory as a starting point & just replace the
necessary bits with calls to the Xitami lrwp functions that are supplied with
the Xitami source. This is likely to give you far fewer headaches & if you write
a simple process manager to manager / kill/ re-start the lrwp php process you
will have a very robust server & php is such a joy to program in, you get the
best of both worlds.

Regards, Paul
