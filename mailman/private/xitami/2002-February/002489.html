> catch all requests from a given adress (http://site_x.com) and
> direct it to a specific folder on the web server.

_FROM_ a specific address, or TO ?

By default, a web server will serve any request that comes to it. If
you have many DNS names all pointed to the same physical
machine/server (same IP or a milti-IP machine), and leave Vhost
alone, all requests to any name of the machine will get the
specified folder.

If you want requests to certain names to get different folders, then
you use Vhosts.

If you want a request FROM "joe.q.public.com to see
"/wwwfiles/public", but a request from "adrian.insider.com" to see
"/wwwfiles/insiders"....  Xitami won't do that, make decisions based
on the viewer's hostname. You could do something with PHP, or a
fancy firewall.
