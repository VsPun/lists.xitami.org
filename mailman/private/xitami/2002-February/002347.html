cgi-bin is a acronym for "Common GateWay Binary" directory. You can place any
executable in the directory and run it as long as the executable takes it's
input from stdin and environment variables and sends it's output to stdout. That
is you can write your own cgi exe's as long as you adhere to the published CGI
protocol.

Deplhi and C++Builder are good examples of development environments that are
delivered with the necessary components to create CGI binaries, many people use
third party libraries to write vb programs for cgi.

Perl and php are most commonly used because they transparently handle the CGI
protocol for you. However, if perl or php are used then the perl binary (it's
large) or php.exe have to be launched in order to interpret / compile the
script. Sometimes a small stand alone exe can perform far better under load.
It's a matter of trading convenience / performance.

For REALLY GOOD performance an LRWP is the answer with Xitami.
