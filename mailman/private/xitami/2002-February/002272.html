Thanks for everyone's support.  Many good thinks came out of this =
thread, and I must give credit to Jeff Hamplin for the final winning =
moves. He said to resolve the situation where an ISP is blocking port 80 =
(and they are at some RR locations, not all) the thing that's needed is =
to first set the base port for Xitami's HTTP to 9000. This will in =
effect make the port for HTTP be 9080 as Xitami adds 80 to your entry in =
that field. Notice it is by default a 0. Then add a forward to the =
router for 9080. Lastly, FTP always worked right from the start as RR =
was not blocking that. However when you tell Xitami to use 9000, you are =
also without knowing it telling Xitami the FTP port is 9021. You can =
enter an FTP port of -8979 to put your FTP port back to 21 and keep web =
at 9080. This is done by manually editing the Xitami default.cfg file =
and adding an entry:

[FTP]
   Port=3D-8979

That should be -8979  -- a negative number. 9000-8979=3D21


Thus users will still be able to access FTP the usual way, however for =
HTTP there is the requirement now to add the port to the URL

http://mikemcdaniel.dynu.com:9080

ftp://mikemcdaniel.dynu.com

Again, thanks for everyone's help.=20
