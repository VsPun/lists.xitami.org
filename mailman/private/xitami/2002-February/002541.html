In message <NGBBLLHPELKGGCABCJJNIEDHCAAA.j.scullion@ayrcoll.ac.uk>, "Jim Scullion" writes:
>I'm running Xitami under Windows 98, and I'd like to be able to change the
>permissions for cgi scripts etc.  I know that the Unix Chmod command does
>this.  How can it be done in a Windows environment?

It can't.  Windows 98 doesn't have any notion of "executable" in the
same way that unix systems do.  Instead anything that is *.exe, *.com,
*.bat, etc is considered executable.  Xitami also considers any script
that starts with a "#! interpreter" type line in a cgi-bin directory to
be executable.
