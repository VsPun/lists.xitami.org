> Now by default all hits on www.mydomain.com, mydomain.com and

Because the name's resolves to the same IPaddress of the server.
 
> I also run several VHosted domains with different domain names.

Different names at the same address get put in the same mail box.
 
> I specify a cfg and aut file for a VHost path to a different directory now
> say we call the directory D:\OTHER\
> 
> I want to set other.mydomain.com to go in this directory only and not to the
> basehost but whatever I try keeps being bumped into the basehost root path.

mydomain.com/other will send to other folder.
 
> I have also specified it under the Virtual Hosts section in the default.cfg
> to no avail.
> And I also pointed to the correct path where I keep the cfg and aut files
> for this cname header.
> 
> Does anyone have an idea/solution to this?
> 
>    Thanks in advance

You would need different IP addresses for each of the 'other' vhost folders to
be considered different places. 

Xitami is small enough you could run a different site each with it's own VHost
name on different ports. 
IE: 
Register ipaddress:80 for main, ipadress:8080 for vhost1 & using different ports
for each vhost site.

Or just do like the big guys (& the rest of us) do & put each site at
maindomain/~vhost for each different site. Which is a lot easier.
