Windows does start swapping after following its rules about when to use
swap space:

1)  When the Available memory goes to nil, start using system cache.
2)  When the system cache goes to nil, swap out memory of existing
processes to the hard drive to recover the system cache.
3)  Repeat step #2 until allowed to resume with #1.

Obviously, Windows isn't being terribly smart when to use swap.

Your case, however, seems really odd.  Explorer.exe loads in a ton of junk
at boot, when you first right-click, etc.  Try running Process Explorer to
see what DLLs are loaded in, and, IIRC, it also gives you the amount of
memory each one is using.  If it does, it will be a rough estimate, but
good enough to find the trouble DLL.  Explorer.exe doesn't do much - what
it loads, however, could do a lot - that's why a lot of trojans/etc. love
explorer.exe - they can hide inside the EXE and not be seen from Task Manager.
