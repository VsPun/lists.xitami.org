>Is anyone using Blat in a Win32 environment?


I used Blat some time ago. Very fine program, but after all not well suited 
for a web application. It could not handle all the borderline conditions 
that could happen.
I ended using a mail server (Mercury from www.pamail.com). It's free, and 
acts like a real mail server, taking care of all error conditions. Of 
course there is some configuration to do, but once it's running, it's worth 
the time used for the configuration. You can the use it is a local mail 
server and you can also access it from the internet.
Mercury is very modular, so you need only to install the modules you need:
- core module
- smtp client (to send out mail)
- pop server (to be able to read the failed mails).

Mercury has a mail drop facility (like most mail servers). Just drop a 
correctly configured message in a special folder and Mercury will pick it 
up and mail it automatically.

Advantages:
- no need to start Blat for each message to send
- asynchronous: the web application can continue it's job, all the mail 
processing is done by Mercury
- no need to take care of transient error conditions (receiver mail server 
offline, ...)

Disadvantages:
- does not handle encoding of attachments: you must provide the final message
