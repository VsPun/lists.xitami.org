On 29 Aug 2002 at 15:43, Marcio Marchini wrote:

> | unix machines traditionally have a full mail server. Windows
> | machines traditionally do not. If they do, it is used so differently
> | than unix mail that it isn't going to work the way unix-oriented
> | scripts and script handlers expect.
> 
> 
> 	It *is* possible to send email from PHP under Windows. All you need to
> configure is the SMTP server in the PHP.ini file which sits in the Windows
> folder.
> 
> notepad c:\winnt\php.ini
> Edit->Find , SMTP
> 
> 
> [mail function]
> ; For Win32 only.
> SMTP = smtp.server.com
> 

I actualy do have that line installed.  SMTP = my.mailserveraddress.net

But there is another line for unix and was wondering if the OS/2 PHP required that 
instead ss the "mail" still command fails.

> 
> 
> 	You should also configure sendmail_from. It is the From field when PHP sends
> email.
> 
> 
> ; For Win32 only.
> sendmail_from = my@email.address.com
