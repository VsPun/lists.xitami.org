Recently I downloaded & installed Xitami Webserver on my Linux =
2.4.19/KDE 3.0 (reiserFS) machine.
I am trying out CGI scripts but always get :

Error 500 : internal error=20
can not create CGI process - program not found.

My setup is as follows:
HTML dir : /mnt/hdc1/webpages
Cgi-bin dir : /mnt/hdc1/cgi-bin
(Both directories on WIN95 (vfat) partitions)

My web pages are getting loaded perfectly. But CGI scripts gives error =
500.
Xitami documentation/FAQ says that error 500 is mostly because of wrong =
header or syntax error.

Following is the script "server_info.cgi" in cgi-bin directory, I am =
trying to execute:
#----------------------------------------------
#!/usr/bin/perl

use strict;
use CGI;

print <<END_OF_HTML;
Content-type: text/html

<HTML>
<HEAD>
    <TITLE>About This Server</TITLE>
</HEAD>
<BODY>
    <H1>Server Information</H1>
    <HR>
    <PRE>
    Server Name       : $ENV{SERVER_NAME}
    Listening on port : $ENV{SERVER_PORT}
    Server Software   : $ENV{SERVER_SOFTWARE}
    Server Protocol   : $ENV{SERVER_PROTOCOL}
    CGI Version       : $ENV{GATEWAY_INTERFACE}
    </PRE>
    <HR>
</BODY>
</HTML>
END_OF_HTML
#-----------------------------------------------

I have perl 5.7.0 (perl+base+devel+.....many more libs) installed on my =
system and /usr/bin is where perl can be found.
I have enabled CGI and SSI on my server and made my scripts executable =
using chmod.

I could compiled my script using
#perl -wcT server_info.cgi
which gave "syntax OK" message.

The so called magic first line "#!perl" also does not work. It simply =
lists the contents of the file.
I tried to rename the script with .pl extension to overcome the problem =
but with no result.
Also tried to set filters for .cgi and .pl extensions with perl but that =
too did not work.

Xitami's "testcgi?" programs work OK, but they all seem to be C =
programs, not perl scripts.

Do I have to install any other perl libs specific to CGI on my system or =
I have missed out any settings?
I would appreciate If anybody can help me out in getting started with =
CGI.

Thanks & Regards,
Rajesh Deshpande.
