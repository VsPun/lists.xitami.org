>If your TCP/IP stack crashes, then your whole system will die (BSOD).
>TCP/IP is generally implemented within a Virtual Device Driver (VxD) or
>System Device Driver (SYS) and if it crashes, the whole thing goes with it
>because drivers run in Ring 0 along with the OS.  Applications run at Ring
>3.  If Microsoft bothered to do everything correctly, drivers would run at
>most Ring 1.
>
>If your stack is messed up, then it might be time to re-install Windows or,
>this is a long-shot, get a different network card with better drivers (I
>recommend a good PCI 3Com card).  Good drivers makes for a more stable OS.
>A generic network card comes with, probably, sloppy/poorly written drivers
>and will tend to not work well.
>
