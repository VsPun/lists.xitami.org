> Installed Xitami in c:\archivos de programa\xitami

Arg. Do yourself a favor and put Xitami in C:\Xitami.

"Program Files", even in Spanish, has spaces in the file
spacification. While Xitami can work through a file system with
spaces in file specifications, there are a lot of quirks. The
easiest answer is to avoid spaces in folder or file names, at least
at first, so you see it all work.

> this works whatever shebang I use, even if I purposely write a
> totally incorrect one.

With Xitami-Win, I usually use #! fee/fie/foo/perl. Really.

Windows isn't unix. Normally the shebang would be ignored. This
would break all unix-inspired Perl scripts. Xitami's clever
workaround is: if there seems to be a goo-looking shebang, but the
specific folder isn't found, then it trawls PATH trying to find the
program. This works on Windows, with the usual Perl setup, without
forcing you to edit every script or maintain an odd unix-y directory
on your windows machine.

> Main HTML directory = c:/website
> Main CGI directory = c:/website/cgi-bin

Usually a BAD idea to have the CGI folder below the public HTML
folder.

> The Admin script tells me that "The value for the PATH_TRANSLATED
> environment variable [c:/website] is incorrect.

Yes, that sounds familiar. Try forcing a trailing slash (I think
Xitami omits it, some popular server adds it, and some scripts
assume something that isn't clearly specified).
