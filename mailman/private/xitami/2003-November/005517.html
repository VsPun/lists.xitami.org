Hello Xitami users,

 For  some  reason  Xitami  crashed on my server the other day...still
 can't work out what caused it. It happens very occassionaly after the
 web  site  has been accessed. The software I use for monitoring ports
 sends  out  a  notification  email  when it polls a port and finds it
 unavailable,  so I got an email 10 minutes later. However, there's no
 feature  within  this  software  to perform any other action, such as
 restart Xitami.

 Does  anyone  know  of  a  utility/daemon  that could poll Xitami and
 restart  it  should  it fail to find an instance running? It would be
 extremely useful to have such a 'backup' in place for Xitami.

Ah, there is a mightily simple way of doing this that someone on this list
pointed me to.
Create a batch file which reads:

:start
xigui32.exe                     (for 2.4d10)
goto start

Make sure you have "recover from crashed silently" checked in Admin panel.
After most crashed Xitami will restart automatically.
