Hi,

When trying to access a PHP-file via an alias, xitami seems to ignore
configured aliases:
"usbhtdocs" is configured as CGI-Alias in defaults.cfg.=20
When I try to access e.g. http://localhost/usbhtdocs/info.php, the debug
log reads this:


2004/03/24 12:40:23: Resolving URL request: /usbhtdocs/info.php
2004/03/24 12:40:23:  - URL handled by filter 'C:/Server/PHP/php.exe'
2004/03/24 12:40:23: - requested file
'C:/Server/htdocs/usbhtdocs/info.php' not found

This is what my defaults.cfg says:

[CGI-Alias]
	/usbhtdocs =3D D:\htdocs

Xitami should be looking for the file in D:\htdocs\info.php

The alias btw works fine when reading a non-php file.

Thanks for your hints!
