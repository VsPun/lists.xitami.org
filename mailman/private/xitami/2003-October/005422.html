At 09:21 AM 10/22/2003 -0400, Bowden, Zeb writeth:
>Yeah I've seen the OpenSSL options and the one I want to implement is
>NO_SSLv2.
>But with Xitami Pro (on Windows anyway) has OpenSSL compiled into the
>executable so how would I tell it to use the NO_SSLv2 option?
>
>It looks like the appropriate place would be the sslhttp.cfs file, but
>that doesn't seem to work ... Is there another way? Or any way that I
>can do it in that file? Or maybe I'm just out of luck?

You are out of luck since iMatix has chosen to not dynamically link OpenSSL
and OpenSSL options have to be specified at compile time.  If iMatix had
dynamically linked XiPro, they could have used the Win32 OpenSSL
Installation Project and avoided this issue altogether.

You can always try asking for a special build that uses either dynamic
linking or no SSL v2 support.

Hope this helps!
