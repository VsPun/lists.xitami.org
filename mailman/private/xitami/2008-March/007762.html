Jesus Criminy - this is entertaining.  Had no idea my questions were going to spur such passionate responses.  

But, in a nutshell - Ports are IP based.  One range of ports per IP address.  So if you have 3 IP addresses on one NIC, then theoretically you have 3X ports available.  And I say 'theoretically' because it depends on how the application is using the IP addresses.  I thought I covered all this in my first response.

However, for those of you who know that TCP Port 80 is per IP address, then please help me understand what I might be doing wrong or where I can look at that is causing a conflict.  In the past, I am pretty confident I had two or three instances of xitami running on Port 80, each on its own IP address (all on the SAME NIC) but can't recall what, if anything, special I did to get it working.  Its been a while.

I have come up with a solution that fixed my original issue of needing a second production server to begin with, however I would still like to move forward with a second server as a test server running on Port 80.  Port 80 is important, and not using some other port, because then my new code can be tested identically to how it would run before making it the production code.  As well, alternate ports other than port 80 can cause some outbound firewall issues that I would rather not deal with.

Config one:
/xitami/defaults.cfg
     * IP : 192.168.20.20
     * Service-Name: Xitami Production Server
     * port:0

/xitamitest/defaults.cfg

     * IP : 192.168.20.30

     * Service-Name: Xitami Test Server

     * port:0

On the test server I even went so far as to copy the xiwinnt.exe file to xiwintest.exe thinking that might help prevent any "service" based conflicts (within Windows 2003 Server).  This did nothing, and the test server would not initialize until I changed it to Port:5080 (or anything other than port 80).  In addition, if I comment out (#Port:0) in the test server, the server service will actually start but nothing will respond on that IP port 80.  

Not certain whats going on but as basic as these configs should be, I believe I may have discovered some kind of compatibility bug between Xitami v2.4d9 and Windows 2003 Server.  I am going to test running multiple copies on Windows XP under the same conditions to test if its a 2003 Server issue.

Comments?
