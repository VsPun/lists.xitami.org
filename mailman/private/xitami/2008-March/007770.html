General wrote, On 2008-03-26 01:11:
> So what you are saying if I do a port scan on my server with 28 IP 
> addresses bound to my one NIC card I should see 29 port 80’s is that 
> correct?

Yes, if you scan an IP /range/. You don't scan an adapter; in fact, it 
doesn't matter (for TCP/IP, in this context) if you have 28 NICs with 
one IP each or one NIC with 28 IPs. It's the same thing. You try to 
connect to a TCP socket (which is an IP/port combination, which an 
application is listening to) and you look for a response. So:

192.168.0.1:80
192.168.0.2:80
192.168.0.3:80
192.168.0.4:80
...etc...
192.168.0.28:80

...and there you have your 28 port 80s.
