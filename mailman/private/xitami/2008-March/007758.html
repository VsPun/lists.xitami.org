Oh boy, where to begin. Firstly, if you have a router, you must forward 
a port in said router to your computer, and /forward it to an IP 
address/. I'm sure you have done this. This should serve as an 
indication that ports are specified /per IP interface/; else, you could 
as well forward it to the adapter's MAC address.

>  You have one port 80, so any traffic is routed to that
> port.

No, you "have" as many ports as you have IP adresses, including the 
several million addresses in the loopback range (127.x.x.x). In fact, 
you better learn that you don't actually "have" ports; don't think of it 
as a "resource", rather, a way to address a specific software that has 
registered to be the recipient of traffic tagged that way.

>  Even if you set up many IP Addresses bound to that one NIC card you
> still only have one Port

No, you "have" one port per IP address, including the several million IP 
addresses in the loopback range (which you don't have to specify).

That is, of course, because your firewall works that way, and has

> The only

No, you could configure your web server to listen to a specific IP 
address. I don't know how to do that in Xitami (it's probably possible 
but I can't be bothered to look it up right now), but here's how to do 
it in Apache (including the comment):

**********begin quote**********
# Listen: Allows you to bind Apache to specific IP addresses and/or
# ports, instead of the default. See also the <VirtualHost>
# directive.
#
# Change this to Listen on specific IP addresses as shown below to
# prevent Apache from glomming onto all bound IP addresses (0.0.0.0)
#
#Listen 12.34.56.78:80
Listen 80
**********begin end**********

Now, the commented line "Listen 12.34.56.78:80" will bind the Apache 
process to a specific address. If you don't bind it to an address, 
however, and use the "Listen 80" line, then it will indeed listen to 
port 80 on all IP addresses bound to the computer, including the 
loopback range. But you have the option to run 28 separate processes 
listening to port 80 on your 28 IP addresses.

Nearly all server level software has the option to bind to a specific IP 
address.

> Now am I missing something?

Actually, without being too blunt, you should really read up on TCP/IP 
on the most basic level before posting more in this thread.
