Well it is doable.  On my OS/2 boxes that run Web/2 it filters HTTP_REFERER and 
blocks users that come from it's ban list.  I run a art site so it's a valuable feature 
blocking troll forums and porn sites that chose to hot link.  A few users don't give a 
HTTP_REFERER.  But most do.  So it works.  My Windows machines are running 
Xitami and I was wondering/hopping if there was a way to block on 
HTTP_REFERER and USER_AGENT to help curb some abuse.  Spammers are 
also sucking up more and more bandwidth every month (like dummified e-mail 
harvesters).  Ways to block abuse are becoming more and more needed (IMO).
