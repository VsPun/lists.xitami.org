Ok, NOW I understand what you are referring to, the first email was a bit vague in the origins of the "thumbs.db" file.  So, the "thumbs.db" file are created locally on the users machine as hidden files and get uploaded during drag/drop FTP sessions and clutter up your server with useless, hidden files.

There are two ways to do this:
    1.  Filter and prevent "thumbs.db" from being uploaded, this most likely will require another FTP server.  Xitami's FTP server is a) not that reliable b) very outdated c) and doesn't provide you any effective filtering methods and d) is no longer supported.
    2.  Write an external script (perl/php/asp, etc) that combs your user and/or upload directories to find and delete the thumbs.db file.  Automate the runnning of the script by adding it to your "scheduled task" list.  Depending on the frequency of uploads, or the appearance of the thumbs.db file, I would run it at least once a day around 3am.

Other than that, I am not certain what to suggest.

  hmmm...
      perhaps i should explain "thumbs.db", and how (and where) it's originating.

  i'm not sure what version of windows first introduced the thumbnail view of images within win-explorer (as an alternative to standard file icons), but it's this exact feature which creates and makes use of the " thumbs.db" files i describe.

  for example, if you're viewing the contents of a folder in explorer, then reach up to the "view" menu and select "thumbnail"... the icons for all common graphics-types, will instantly become thumbnails-images.  some windows users simply keep their folder-view settings adjusted this way right across the board...  i believe, it may actually be the default folder-view with fresh windows installs.  (it's something that i always turn off immediately) 

  regardless...  the moment that a windows user switches to the thumbnail-view, "thumbs.db" (a hidden system file) gets created within that folder... and acts as a thumbnail cache of sorts, so windows doesn't have to keep re-building the thumbnails each time the folder is viewed. 

  (each and every folder will have it's own "thumbs.db" file, if/when applicable.)

  it means that someone could easily have one folder with five jpeg images in it... and only be able to see a total of 5 files... but in actual fact, there are 6 (if you include the hidden "thumbs.db").

  if that entire folder (or a select-all of it's contents) is uploaded to my (xitami-only) server via ftp, all six files will transfer during the upload session ("thumbs.db" included)... and not just the 5 which were intended by the user.  these " thumbs.db" files aren't even visible to the user until after they've been uploaded... which causes most of them a great deal of confusion, if and when they are found.

  if a folder is dragged and dropped into an ftp-session, it makes sense to me that all it's contents will be transferred with it, whether some of the file-attributes are set to "hidden" or not.

  this brings me back to my original question actually... because i feel if i had a way to instruct xitami-ftp to simply disallow the upload of any file by the name "thumbs.db", my problem would instantly be solved. 

  i clearly understand what these files are, and how they are being uploaded by users on my system unintentionally... i just want to firewall them, if that's at all possible.

  sorry if i've bored most of you with my lengthily description... but would really appreciate any advice that anyone here may be able to provide.
