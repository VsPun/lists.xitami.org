Heavy server usage testing via a tool I wrote primarily for testing the
speed of scripting languages under various conditions...instead it has
become a tool I use now for debugging this particular DoS/DDoS.  It really
doesn't matter if you don't know 50 people.  As long as a malicious person
is willing to use a distributed attack tool, all they have to do is find a
CGI on a Xitami server and within 2-3 seconds, the server will grind to a
halt and start returning zero bytes of data to most clients (it will appear
to them that the server is busy or down and then proceed to go elsewhere).
