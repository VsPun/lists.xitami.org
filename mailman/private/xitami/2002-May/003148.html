    Well I'm no expert but I like Xitami.  But I'll try and earn my keep here and answer a 
question.  :)  Be warned..  if you use Xitami FTP in high bandwidth you may get corrupted files.  
In they case a dedicated FTP program would be better.  Like for me so far the FTP seems to 
be working over my 768SDSL.  However if I ftp from one system to another over my local  
100Mbs switched network 98% of the time I get corrupted files.  I am still using the Xitami FTP 
because it is very convenient and easy to set up.

If you want to set yours up just add the needed info to your ftpusers.aut in your /xitami 
directory.  Now you don't even need to reload xitami when you update this file.  Xitami checks 
it after so many seconds (set in the xitami.cfg file).  Ok set up an account by adding these 
lines.

[chosen-name]       # <--This is the FTP login name you will use.
access=GPUDMR  # <--This is what permissions you want to give. (I have chosen all here)
root=e:\xitami\webpages\  # <--Set this to your folder you want to give access.
use-quotas=1        # <-- Do you want to limit space this account can use? 1=yes 0=no
soft-quota=50.0     # <-- If above=1 they get a warning when this is reached.
hard-quota=60.0    # <-- This account may not upload more than is in Mbytes.

Some of my personal experience with Xitami ftp.  When you set the "root=" directory.  Be very 
sure you set this directory to one that exists!  If misspelled or you get the address wrong your 
Xitami web server *WILL* crash every time the account is logged into. :)

Hope this helps some..  Gene Angel
