I would bet you have an invalid character in the directory name. 

Also note that "webserver" is not an 8.3 filename; the ones that work
for you are 8.3 style. Which command prompt are you using (Command or
CMS) and what editor are you using? 

Directory corruption isn't handled by DeFrag, except as a side effect.
Open the Properties of that drive, select Tools, click "Error Checking". 

Or (believe it or not) at the command line, run "CHKDSK D:". Just like
DOS. And like DOS, it won't fix unless you add " -F". One difference: on
NTFS, you can't do a ChkDsk with fix-up while the volume is mounted, but
it will offer to run it at the next boot.

Which offers another thought-- NT's filesystem is usually good but
sometimes loses track. Re-boot is rarely needed but often worth a try
when things seem wrong. NT does most sanity-checking at boot.
