In message <004101c1357d$f0799ad0$ea409fc1@ednic>, Joubin Far writes:
>I have installed the zip file bw3224d9 without any problem and i have 
>gotten the following error message while starting: Xitami was suspended
>Could not open HTTP port 80 - Port is already used by another 
>server(WSAEADDRINUSE).
>My OS W2K runs on 1200MHz AMD.  I had never this problem with Win95 and WinNT.

Did you install IIS when you installed W2K?  (Or did W2K decide to
install it for you along with something else?)

You can check if something is definitely using port-80 in a cmd window
with:

netstat -na

Then look through the output for a line mentioning ":80" or ".80" (at
the end of an address, and "listening".  If there is one there, then
you've definitely already got a webserver running.

Either you need to change that webserver to another port, or you need to
run Xitami on a different port.  All Xitami's ports can be shifted with
the portbase option: eg "xitami -b 5000" will move all the ports up by
5000, giving 5080 for HTTP, etc.
