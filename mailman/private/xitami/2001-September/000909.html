In message <sb9e8702.070@gwise.louisville.edu>, "Mark E. Crane" writes:
>I've just installed Xitami for windows 2k, and I've noticed that the latest
>version, unlike earlier versions, does not show directory contents by
default.

The [security] dirlist option can turn off the directory listings (but
AFAIK the shipped default is [security] dirlist=1, which means listings
will be generated).

If you are using 2.5b5 beware that there'll be no directory listing if
it can't find/run the supplied GSL scripts.

The server debugging may give you more information about what Xitami is
trying to do.
