On Sun, 18 May 2003 02:13:45 -0400, WhiteShepherd wrote:

>If anyone knows.  I was thinking of getting a second DSL line to 
>reverse proxy.  What I would like is for my Xitami server to serve 
>both IP's on a single adapter.  Can win2k and OS/2 without 
>problem bind more than one IP to one ethernet adapter?  If so 
>would Xitami web serve both of those IPs?

I know that OS/2 can do it, but only on the newer 32 bit TCP/IP stacks. 
It is undocumented, but you can use aliases to configure it. With:

ifconfig lan0 192.168.0.100 netmask 255.255.255.0 metric 1 mtu 1500
REM *** assign IP address to same NIC
ifconfig lan0 alias 192.168.0.200

It works. You can tell Xitami to use virtual hosts based on IP address, 
so you can have multiple domains running with it (that's how I've been 
using it) - all on OS/2 though.
