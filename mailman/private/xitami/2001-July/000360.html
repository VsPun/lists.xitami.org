In message <3.0.6.32.20010628112420.008be100@pop.surrey.ac.uk>, Andrew Tate writes:
>Sometimes ftp servers can 'restrict to home directory' and stop browsing
>outside of the home directory tree defined for the user.  Other times
>you may not see the other disks (as ftp has roots in unix) so is not used 
>to the disk drive letter idea but if you put 'cd d:/' it may take it and 
>act correctly.

Xitami doesn't handle drive letters like that (as you say, some Windows
based FTP servers do).  Instead Xitam insists that you define an alias
that points at the other drive, and that people use that alias instead.

As you've found there are some issues with the Xitami FTP support, and
it doesn't always seem to play nicely with others.  A number of these
issues are on our bug list, but meanwhile if you need some of the
features in other FTP servers you are probably better disabling the
internal FTP server in Xitami, and using another one alongside it.
