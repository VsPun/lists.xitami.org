> for some reason either xitami or win2k was giving  realland.exe the priority

Of course. Xitami looks for the exact name. If that fails, it tries the
name with EXE COM and BAT appended. If you ask for "foo", and there is
no "foo", it looks for "foo.exe", "foo.com", "foo.bat". 

So realland.exe will be found, and realland.pl won't be found unless you
give the full file name. 

> using no extension instead of .pl or .cgi forced windows to rely on the #! perl line in the script 

Actually, Windows does not know #! at all, Xitami is doing that.
