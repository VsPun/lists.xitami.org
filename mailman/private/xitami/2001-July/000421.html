>Debug.log does not have anything. When I load default.htm and click on the

>testssi.inc. When I view the page source in Explorer the SSI line has been

>I also notice that the GIF images in the test webpages don't show up. 

Are you _sure_ you're loading your pages through Xitami?  Both those
problems sound a whole lot like you're just getting IE to load the files
form disk -- in which case it's no surprise that Xitami isn't doing any
SSI translation: no one is asking Xitami to do so.

You should be using a URL like:

http://localhost/

when you are testing.  Not one file file:///.....

The file:///.... one would have problems with image urls like "/image/...";
but Xitami would definitely find them in the default configuration with
a url like that.  This would also explain your empty access.log, and
empty debug.log.
