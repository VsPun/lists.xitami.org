In message <001401c10bda$f34117b0$0100a8c0@leiter1>, "Brian Leiter" writes:
>Help me please... I am running Xitami 2.5b5 on Windows 2000.  When I 
>FIRST setup the virtual hosts everything was fine... Then Xitami started 
>crashing in the admin config section and now they don't work?  I even 
>uninstalled Xitami and reinstalled it with NO LUCK on getting the 
>virtual hosts to work again.

Did you start with a new set of configuration files?  If not it sounds
like it might be in your configuration file.  Xitami tries to recognise
the host name passed in and use that to figure out which directory to
use: for some reason it sounds like in your case the names aren't
matching.  You'd probably be best checking spelling, etc, carefully.  Or
turning on the server debugging ([Server] debug=1), and checking the
debug log carefully for what Xitami is doing internally.
