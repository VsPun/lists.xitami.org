In message <200107261643255.SM00258@default>, "Julian Rickards" writes:
>I have reviewed several SSI tutorials from the net but although the 
>set command is not described very often, from the few that I found 
>that described its use, it would appear that I am using it correctly.  I 
>am wondering if Xitami's implementation of SSI is either deficient or 
>has a bug in it.

Xitami's SSI is not Apache's extended SSI.  If you call that
"incomplete" then I guess it's "incomplete".  But it never claimed to be
an implementation of the Apache extended SSI.

>However, I want the QUERY_STRING text, in this case a filename, to 
>be the virtual filename as follows:
>
><!--#include virtual="QUERY_STRING" -->

That is becuase you ask for a file called "QUERY_STRING" to be included;
probably you don't have one.

>When my instructor for my webmaster course showed us this trick, 
>he used a slightly different procedure whereby he "set" the contents 
>of a variable to be the QUERY_STRING as follows:
>
><!--#set var="bodycontentfile" value="$QUERY_STRING" -->

As far as I know variable interpolation is also an extension to the
SSI processing, and as such it's not supported by Xitami.

As I said in another message, if you need extensions that are
implemented only in one webserver, then, well, you'll need to use that
webserver.

Offers to implement these extensions for Xitami are always welcome.
