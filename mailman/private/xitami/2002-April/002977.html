I misunderstood because I've been generating pages of text from

I wonder if any server that would allow you to *include* very large 
(mb) text files would also a security risk when they slow or crash 
doing memory manipulation.  Does IIS allow larger includes?

My guess is that Xitami also prefers outside scripts and software to 
take the memory and security risks so that the server doesn't crash 
~ like the scripts many people write.
