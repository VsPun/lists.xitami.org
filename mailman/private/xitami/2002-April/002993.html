In message <01c001c1e650$95c173e0$0200a8c0@mine2>, "Jan de Lint" writes:
>Would I have more luck (stability) with 2.5b5 which I abandoned because XML
>is/was not handled naturally?

.xml files in the default Xitami 2.5b5 configuration are handled by GSL
scripts.  If you don't want this, you can edit the xitami.cfg file, and
remove the [Wsx] .xml mapping.  (Unfortunately you can't override it in
the defaults.cfg file.)

As for stability issues, 2.4d9 is more likely to be stable than the
2.5b5 beta.  However both do have issues with "dying" from time to time,
unfortunately.

If anyone finds a repeatable pattern to the dying (eg, do this, and then
it dies, every time I do that) then please let us know.  Unfortunately
"it just died" isn't very useful for trying to figure out what is causing
the problem.
