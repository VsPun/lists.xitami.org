On Thu, 18 Apr 2002 09:54:41 +1200, Ewen McNeill wrote:

>>In lieu of an explanation, if someone can guide me to a way I can create 
>>a simple service listener, I could deal with option.  In other words, a little 
>>app that listens to Xitami, checking to see if it is still running, and, 
>>if not, restarts it.  
>
>You don't want to "listen" to Xitami; you want to "check up" on Xitami.
>What I do with some servers I monitor is have a special test URL (eg, a
>HTML page with some recognisable text on it), and then have a small perl
>program (eg, using LWP) check that URL from time to time.  If it doesn't
>get the "right" response, then it restarts Xitami (or whatever is being
>monitored).

Or you start it from a simple script that runs 'forever'. On OS/2 it looks like: 

[runxitami.cmd]

/* */
Do Forever
'Xitami.exe'
end
exit

Couldn't be much simpler. And reactivates instantly ;-)
