In message <5.1.0.14.0.20020329141428.03c06a30@canexus.com>, Matt Hunter writes:
>I don't know how many of you are familiar with mod_backhand for the apache 
>server, but it allows you to have a cluster of servers with one master 
>server that handles all incoming connections.  Based on how busy each 
>machine is in the cluster, backhand decides if the master machine should 
>handle the request, or if it should redirect the request to another machine...

Sounds very clever.  (I've never used it, but I help look after some
sites that are getting to a point where something like that would be
useful.)

>Is there anything like this available for use with Xitami on a windows 
>platform??

Xitami doesn't have any equivilent of Apache's mod_proxy code (to talk
to a backend server, and then pass the results back) yet.  So no,
there's nothing in there to do load balancing like that.

You could possibly run Apache at the front and Xitami at the back
though.

FWIW, I've been thinking for a while that Xitami would be quite a good
choice for some proxy/load balancing code like this.  So in the unlikely
event that I get some free time (or the slightly more likely event that
there's actually a paying client that wants it), it might get written.
