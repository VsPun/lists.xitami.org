incidents in the past month or so since this issue arose.  The most recent

under heavy loads.  There was an earlier one with Win98 crashing on a low

Remember that filters are CGIs in "disguise."  They have somewhat different
properties than CGIs, but are basically the same.

Also, unless you have 50-60 simultaneous users on the system using CGIs,
this problem will go unnoticed.  However, a malicious attacker can easily
cause a DoS/DDoS with a simple tool that just requests CGI-oriented scripts
on the remote server to be executed - the server is quickly brought to its
knees and other users are unable to get webpages (usually the browser
appears to hang when the server replies with zero bytes of data - a flaw in
one of the Windows API calls that the browser has little control over).

Hope this helps!
