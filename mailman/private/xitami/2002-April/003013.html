In message <NGBBJAFHALILDOAFEDNAEEECCAAA.egbert@familie-egberts.tk>, "E. Egberts" writes:
>I have setup a small Perl script containing some simple print statements for
>testing purposes. When I execute the script, the browser prompts to download
>a file. This file contains the expected output but I want it to be printed
>in the browser!. Which Xitami configuration parameter (if any) controls this
>output behavior?

Is the script in your CGI-BIN directory?  Does it start with #! /usr/bin/perl
or similar?

Only "executable looking" scripts in your cgi-bin directory will be
executed; the rest will be offered to the browser "as is".
