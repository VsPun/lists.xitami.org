In message <007701c1eb04$d6418820$a419bb92@ewu.edu>, Steve Shervais writes:
>Recently, I tried setting up virtual hosts (using the wizard, mostly) for
>different teams, and trying to give each team their own logs, so they can
>see what their errors look like.  Xitami created the logs for me
>(teamXX.log, teamXXerr.log), and in the correct directory.  However, none of
>the logs has any entries [....]
>
>In debug I see:
>2002/04/23 13:12:06: Looking for VH using Host value: eaglebay.ewu.edu
>2002/04/23 13:12:06:  - looking for VH using IP address: 146.187.25.27
>2002/04/23 13:12:06: - using default virtual host values

For some reason it is not matching your virtual hosts -- and hence is
returning the "default" webpages, and hence logging in the default logs.

You'll need to make sure that the defaults.cfg file points at your
virtual hosts with entries like:

[virtual-host]
  domainname=config

and also that the domain name is being passed in with a HTTP/1.1 Host:
header (which it appears to be from the snippet that you list).
