At 02:23 PM 4/8/2003 +0200, Mirek Scholze jr. writeth:
>Hi all,
>
>I'm running Xitami 2.5b5 on Win2k Pro, Duron 600MHz, 256+128MB RAM, 100Mbit 
>Ethernet card.
>
>Sitting on another machine (Win XP Pro, Athlon 1.2G, 256MB RAM, 100Mbit 
>Ethernet card) at our building 100Mbit intranet and accessing my Xitami ftp 
>server I get these results:
>
>download 200MB file from server:
>* speed about 250-300 kBytes per second,
>* server CPU usage below 10%, by Xitami aprox. 2-4%
>
>upload the same file to server:
>* speed about 7.5-8 MBytes per second,
>* server CPU usage about 99%, by Xitami aprox. 95%
>
>Could someone explain or give an idea to me, why there is so big diference 
>between download and upload speed?

Xitami shouldn't be consuming that much CPU time.  Apparently Xitami is
lacking some sort of throttle mechanism for uploads (seems to be doing fine
on downloads).  The CPU usage for any webserver should be 10% at most to be
able to handle periodic spikes up to 50%.  Anything over 50% is asking for
trouble on a high-speed connection like yours.  If you don't want to mess
with modifying code, you could throttle the network card back to 10Mb
instead of the 100Mb you are currently running at (which should effectively
slow things down).

Hope this helps!
