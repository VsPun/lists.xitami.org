In message <3BCB4C94.22CC001@win.co.nz>, daniel carter writes:
>Has anyone written one of these for xitami? I'd like to have xitami
>installed as a service on my linux box.

It's pretty straight forward to do -- copy an existing one, and then
(a) change to the base directory of Xitami's config files
(b) start xitami as a server "xitami -d"

To kill it, look for the xitami process (there's only one) in the
process table and kill it off.  (Xitami doesn't (yet) write a PID file
unfortunately.)

>Also, is there a way to have xitami running as a non-root user like
>other servers do?  I'd like to have it running as an unprivildged user
>as a security precaution, but i'm running into the port 80 problem.

You can get Xitami to change user after it has started (avoiding the
port 80 problem), by using the [server] setuid functionality.  Certain
things won't work when that's active (eg, restarting the server) because
it no longer has the privileges required to do things like open port 80.
