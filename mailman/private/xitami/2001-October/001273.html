In defaults.aut, the word before the = is your username, and the word after 
the = is your password. For example, this will let you log in to 
http://127.0.0.1/admin as a user "eviloverlord" with password "topsecret":

[/Admin]
    eviloverlord=topsecret

If you have admin=xxx and SecureName=xxx lines, you can log in as either of 
the users "admin" or "SecureName", with password "xxx" in either case. I 
assume you don't want this.

The one exception to this is "webmask=something", which sets which 
computers can access the admin screen (webmask=local will restrict it to 
the computer the server is actually running on).

If you're using Netscape 6 or Mozilla, you may need to edit defaults.cfg 
and add this:

[Console]
    Javascript = "0"

(if the [Console] section is already there, either add the Javascript key 
or edit it to switch it to 0). I couldn't use the Javascript admin 
interface with Mozilla 0.94, but it works now (in Mozilla 0.95), so your 
mileage may vary. I actually considered sending iMatix a patch to use 
different Javascript function names to avoid this problem, but the names 
are hard-coded into their form generating tool :-(
I might still try adapting it so the Javascript isn't critical: if I do, 
I'll definitely let iMatix know so they can merge it into some future 
version.

If you have a really screwed config, adding this to defaults.cfg may help:

[Wsx]
    admin=xiadmin

although this shouldn't be necessary.
