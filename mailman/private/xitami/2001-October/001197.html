In message <9177F5D4FF30D511BDAC00A0C9B856BF049412@762NATO>, Raymond Judge writes:
>I am forced to use port 8080 to run my Xitami web server due to firewall
>restrictions from my cable provider.  I would still like to run my FTP
>service on port 21, yet when I run Xitami enabling the FTP service it
>unavailable on port 21.  Any suggestions?

The only way to run HTTP on port 8080 in Xitami at present, is to set
the "port base" to 8000; which shifts _everything_ up by 8000.  So FTP
becomes 8021.  As a hack you can move FTP back down by giving it a
negative value (== 21-8000 = 7979).

Adding a better way of just moving the HTTP service is on our todo list.
