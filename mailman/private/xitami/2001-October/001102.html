In message <3BA9B55F.7000108@freesbee.fr>, Francis Turner writes:
>I just noticed that the SSI include tag (e.g. the one on the testssi 
>page <!--#include file="testssi.inc"-->) cuases the rest of the 
>including document (e.g. testssi.ssi) after the tag not to get delivered 
>if the included document itself uses SSI as the example include does.

The problem is actually with any recursive use of SSI -- there's an
"off-by-one" issue which means that Xitami was storing an
end-of-document marker before the next bit of included SSI, and hence
running into problems.

This is fixed in our version in CVS
(see http://cvs.imatix.com/cgi-bin/viewcvs/)
but not in any of the binaries that have been built.

>Also in the process of debugging and playing I noticed that Xitami has a 
>limit on files with SSI includes of about 35,000+ bytes (one file 
>displayed upto the 35922nd byte, another upto the 35553rd) and any data 
>thereafter is abruptly truncated. 

I think this has also been reported, and I think it may have been fixed
in the version in CVS.

Thanks for the reports.
