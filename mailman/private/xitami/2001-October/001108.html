In message <F166jvM9KfYDovvQ0Pu00004180@hotmail.com>, "Mike Scholman" writes:
>I have set up a Xitami Server on an NT Server with NTFS. I am able to
>run simple PERL and PHP scripts, but if they try to write to disk
>(ie update files, etc...) they fail. 

In which case whichever user it is they are running as does not have
write permissions for the relevant directories.  The relevant user
depends on which version of Xitami you are using (NT service, windows
gui, etc); if you are using the windows GUI version it is the user that
started Xitami running.  If you're using the NT service it's specified
in the service panel (IIRC), and is often the SYSTEM user.

You can assign permissions in NT Server through the GUI as well as with
a command line utility I believe.
