Reading the Xitami-manual about language support for browser-requested
html-files I don't quite understand how this should work.

If Xitami is asked to request for the file 'file.htm' it.

1.) first looks if the file 'file.html' is available.

2.) If not, it looks after (take french in this example)...

    file.htm.fr
    file.htm_fr
    ..
    ..

In my opinion 1.) and 2.) should change position. You should have a
default file, but that should only be taken if none of the other is
available.
