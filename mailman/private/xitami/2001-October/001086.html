Hello.
One possible solution of user rights at NT is running Xitami as a service
logged as defined account.
You can set the account under which will be Xitami running
in Settings/Services/Startup... - Log on as ...
Use existing account or create new one especially for Xitami and
set the permisions on files as you like for that account.
Used account have to be allowed to "Log on as a servrice"
in Administrative tools/User manager for domains/Policies/User rights/
then check the box "Advanced user rights", find "Log on as service"
and add an "user".
This works very nice for me as I need to have cgi scripts running under
"normal" user account to be able to access database server.
