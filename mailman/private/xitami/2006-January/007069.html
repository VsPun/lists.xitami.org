On 1/4/06, Francis Turner <francis.turner at gmail.com> wrote:

> If you look at the /admin page there seems to be a fairly simple way
> to cause the server to reset (or halt).

The problem here is that if the server has died, that's not going to work.

What we generally do - on Linux, but the same works on Win32 - is to
monitor the process continuously from a script that runs on the same
box as the server, using wget, and if the server fails to respond
within a second, kill it and then restart it.

Under win32, you can stop/restart the server by using the server
manager (net stop / net start),  but since the net stop command needs
the server to respond, I'd suggest using kill.exe.

You can do this in Perl but it's really not worth it, it's just a few
lines of batch commands.
