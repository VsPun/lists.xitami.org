> I have a whole ordering system running as executable (more than 100
> executables), never had this sort of problem...

Maybe you would be interested in the WTP (web transaction protocol) 
that is included in Xitami 2.5?

WTP is a simple but very efficient transaction processing monitor 
that lets you write large database applications to run under Xitami.  
More sophisticated than LRWP, much more efficient than CGI, only for  
C applications at present.

http://www.sms-at.com/wtp/smsat is an example application (you will 
need a Belgian GSM number to register and use it, unfortunately).

You will find WTP, related tools (such as fdlgen/fmlgen) and sample 
applications in the Studio directory of the Xitami source packages.
