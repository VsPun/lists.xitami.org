  I am able to do sub domains like www.mydomain.com/sub or =
sub.mydomain.com. Now of course the first one is just a directory and =
doesn't need it's own IP address. However with the second one I have to =
set up an IP address for it. Is there a way to set up sub.mydomain.com =
without it's own IP.=20

  Ron
You need two things to make this work.

You need a CNAME record in your DNS server pointing to mydomain.com
Example:
A-record : mydomain.com =3D 192.168.0.1
CNAME-record : www =3D mydomain.com   (This would get resolved as =
www.mydomain.com =3D mydomain.com =3D 192.168.0.1. NOTE: some DNS =
services requite CNAME www.mydomain.com =3D mydomain.com. Check their =
instructions first.)
CNAME-record : sub =3D mydomain.com  (This would get resolved as =
sub.mydomain.com =3D mydomain.com =3D 192.168.0.1)

Now, you need to set up a virtual host in Xitami, where sub.mydomain.com =
=3D config.file

When a request for sub.mydomain.com comes in, it will get passed to the =
virtual host you setup.


Tom
