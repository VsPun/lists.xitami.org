The mail server working fine, but you are not authorised.
Mail servers are configured so that they only relay mail from authenticated
users.
There are different ways of authenticate:

pop before smtp is the most widely use: pop your mailbox and the mail server
will allow that IP to relay during a limited time frame. It doesn't work if
you have different servers for outgoing and incoming mail.

the smtp protocol allows identification, but the mail client that want to
send mail must connect directly to the server, not via the ISP relay (the
isp relay will not pass the authentification).

This is just a basic review, since this is not a mail server mailing list.
