I think we are a little off track here. The problem is that
webcam.ackerman-family.com points to port 80 via the DNS. The A record for
ackerman-family.com points to internal IP 192.168.1.3, the CNAME point
www.ackerman-famly.com to ackerman-family.com. So any subdomain that is not
defined in the DNS will go to 192.168.1.3 port 80 (80 is opened on the
router and is what Xitami is set for). So port 8080 will not work. I have
even tried opening port 8080 on the router but you can't specify a port in
dns records (that I know of). I think I can have the DNS server point
webcam.ackerman-family.com to a different IP with an A record to that
effect.

No need to learn PHP (although PHP *is* cool) as HTML already can do this
for
you.

A standard HTML anchor will do the trick from a webpage:

 Click <a href="http://webcam.domain.com:8080/">here</a> for the webcam.

If you are wanting to redirect when someone hits a page:

 <META HTTP-EQUIV=Refresh CONTENT="10; URL=http://webcam.domain.com:8080/">

Put the above in the <head></head> section and after 10 seconds most modern
browsers will redirect users automatically.
