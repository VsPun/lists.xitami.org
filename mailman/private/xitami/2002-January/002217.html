In message <3C582EB7.C7593916@rci.rutgers.edu>, Paul Reid writes:
>Alternatively: hack the open-source code for Xitami, find the special-case
>where Xitami/Win looks for "#!....perl" as the first line of a file and
>searches PATH for a copy of perl.exe, 

What special case would this be?

The code in sflproc*.* looks for a #! line that is something like:

#! /path/path/executablename flags

and if it can't find the literal /path/path/executablename, it then
takes just the executablename and tries to find that on the path.

It should work fine with Python.

>and add similar lines to look for "#!...python" and search PATH for 
>python.exe. But recompiling Xitami on Windows seems to want an MS C 
>compiler, and IIRC the GUI version needs a commercial package so isn't 
>totally "free". 

You can rebuild the GUI version from source with MSVC.  I guess that
counts as needing a "commercial package", but it's a reasonably widely
available one.  (The source to the GUI was merged into the open source
version with 2.4b9/2.5b5.)

In theory it should be possible to compile it with other compilers, but
particularly the GUI version would take a bit of work to get the
resources compiled properly and linked in.
