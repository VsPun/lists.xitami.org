Brinkle, Jason wrote:

>How do I get rid of those percent signs in the url?
>Example:  http://10.191.32.111/@site/irps/omniweb/005501/planmenu%2Epdf
><http://10.191.32.111/@site/irps/omniweb/005501/planmenu%2Epdf> 
>
If you are refering to links created by Xitami when you view directories 
on the server then the answer is to edit the file template/httpdir.gsl

replace the line
<A HREF="$(urlbase:)$(nameurl:)"><IMG SRC="$(icon)" 
BORDER=0>$(name:)</A></TD>
with the line
<A HREF="$(urlbase:)$(name:)"><IMG SRC="$(icon)" BORDER=0>$(name:)</A></TD>

You may also want to add the following lines near the start (before the 
lines that say
.-  We want the first character of the sort order
.directory.order = substr (order,0,,1) )
This fix allows your users to reverse sort the directory correctly

.- Bug fix - remove leading '=' from directory.order if present
.-  The attribute directory.order has a leading '=' if it is not the 
default
.-  order. For example:
.-  If the order in Xitami's defaults.cfg is 'nx' then when you execute the
.-  gsl for 'http://localhost/directory/' then directory.order is 'nx'
.-  but if you then click on the URL for Size and thus execute gsl for
.-  'http://localhost/directory/?order=snx' then directory.order is '=snx'
.if substr (order,0,,1) = "="
.  order = substr (order,1,,)
.endif
.- End bug fix - remove leading '=' from directory.order if present
