In message <3C4F836B.A5A74F90@videotron.ca>, Greg Stienstra writes:
>Compiling sflbits...
>this is c.conf seting up parameters for the imatix c procedure
>ar: creating archive libsfl.a
>ar: sflbits.o;: No such file or directory
>ar: ranlib: No such file or directory

For some reason it appears that the file name is being seen as "sflbits.o;"
which isn't right, and that "ranlib" is being seen as an argument to the
ar command, which also isn't right (ranlib is a command in its own
right).

The command line ought to look something like:

ar r libsfl.a sflbits.o; ranlib libsfl.a

but from the errors that you show it seems like it looks something like
this:

ar r libsfl.a "sflbits.o;" ranlib libsfl.a

which won't work.

I'd suggest finding the line in the script calling "ar", and adding some
echos before it to display what it is running; a common thing that I do
is copy the line and add echo " before it, and " after it. (Although in
this instance that may not show exactly what we want, but it should
point you in the right direction.)

>Could it be that I need to regenerate the 'build' scripts? How?

The error that you show is clearly within the "c" script (which handles
adding the right suffixes to things as it goes), so you don't need to
rebuild the other build scripts.  (Unlike, say, automake/autoconf, the
Makefiles that are generated by otto are pretty generic, with all the
customisation handled in the "c" compile script.)
