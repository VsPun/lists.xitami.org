For escape characters:

"\n" is not recognized by any browser because it is perl-syntax, not html.
The same applies for other escapes.

If you use

print "I want this\nand another line";

you will get one line in the browser but two lines in the source code (as
seen from the browser).

In order to have two lines, you have to use

print "I want this<br>and another line";

which will be one line in source code but two lines in the browser.

Best regards,

Hugo

By the way, after much pain and suffering, I discovered from this Q&A list
that I had to include the ".pl=3Dapplication/x-http-perl" statement in the
Mime section and "pl=3DC:\perlpath\perl.exe" in the Filter section of the
Defaults.cfg file in order to get my NT 4.0 to execute the Perl5 script.
