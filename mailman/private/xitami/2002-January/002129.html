In message <3C4CDF8E.F5ADE35A@videotron.ca>, Greg Stienstra writes:
>I am unable to get xibuild to work under OpenBSD, it appears OpenBSD makes use
>of an "ar" (object librarian) program of a different lineage than do linux, and
>freebsd, and refuses to do its job if the 's' switch is used.

IIRC "ar s" is a SysVism that the GNU tools also have.  OpenBSD is a BSD
true'n'true, and they tend to use ranlib instead.

The "c" compile script will support this, and it is controlled by the
"RANLIB" variable in the script.  The existing script has an entry for
FreeBSD which does basically this; you probably want to make that one
match OpenBSD too (and presumably NetBSD).
