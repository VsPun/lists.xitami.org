CGI written in VB (visual basic i suppose) don't need a VB kernel, since 
the code is compiled (resulting in an exe file)
When a normal script (like a dos batch file) executes the command processor 
has to be loaded first, the source compiled line per line (every time your 
script is called) and the resulting commands executed.

btw vb is not that efficient when it comes to handling cgi: vb is written 
with forms and user interaction and other visual stuff in mind. When vb 
loads, a lot of unnessary ballast is loaded altogether. cgi handling is 
actually just plain text processing, no visual effects (that part is done 
by the client browser). Should try Power Basic Console Compiler.
