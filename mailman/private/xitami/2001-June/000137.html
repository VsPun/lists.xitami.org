In message <3AA35E12.23619.E70D5C0@localhost>, "Tim Fiddimore" writes:
>What I want to do is pass the JSP file directly to the servlet, so I 
>can use a url of the form:-
>
>http://mydomain/examples/example.jsp
>
>I cant seem to find any way of associating JSP files with the servlet.

The main issue here is that the ServletRunner has no syntax for saying
to Xitami "give me all the files ending in this extension".  In theory
(I haven't tried this) if you can get the LRWP request to be for ".jsp"
then all ".jsp" files should be passed to that LRWP (and with the way
the ServletRunner works, to that servlet).

So if you are up to a bit of Java programming you might be able to hack
the ServletRunner to pass the ".jsp" string across for the servlet that
is handling the JSP files in its (or one of its) LRWP registrations.

Really what is needed is some sort of config file syntax for specifying
this, but that may have to wait until I can merge in the changes to the
ServletRunner that a user sent in.  (Ie, more spare time :-) )
