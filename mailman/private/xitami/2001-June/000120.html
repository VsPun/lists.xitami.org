In message <3A2A6CA6.13469.57D954@localhost>, "Dr Mike Waters" writes:
>I've just installed Xitami 2.5b4 service on a windows NT4 
>workstation. The http server seems to be operating perfectly well. 
>However, when I open a console windows and try to ftp to this 
>machine from another machine on the network or from the local 
>machine it puts up the initial xitami ftp server message and hangs 
>at this point (i.e. no carriage return, welcome message etc). 

If you were running the NT Service version of 2.5b4 you were possibly
being caught by the old-binary situation, causing it not to find the
script that output the rest of the FTP login sequence.  You might like
to try 2.5b5 and see if that has fixed the problem; it includes a
rebuilt binary that matches the included scripts.
