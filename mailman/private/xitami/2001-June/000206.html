In message <000201c0f463$5dfd60c0$0600a8c0@socal.rr.com>, "Kevin" writes:
>Using 2.4D9 Xitami server,activestate perl,windows ME,cgi enabled.
>I am able to execute an image-based counter cgi successfully if I link
>directly to the cgi.  Ie. http://host.com/cgi-bin/mycgi.pl
>But when using SSI call to the cgi inside a .html page the cgi is not
>exectuted and no cgierr is generated.

By default (and we recommend) .html files are not processed for ssi tags
(it is too much of an overhead).

>If I change the extension of the same page to .shtml then the page is
>not shown and I get a 500 internal server error.  Once again not cgierr
>is generated.

There is some problem in calling perl.  Make sure you have Perl
installed in a directory like c:\perl (ie, preferably with no spaces in
its path name), and your PATH includes c:\perl\bin in it.  If that
doesn't help, you'll need to tell us more about the "500 internal server
error" that you get -- the message includes details about what went
wrong which can be helpful in figuring out the problem.   Until perl can
be started, you probably won't see anything in cgierr.log.
