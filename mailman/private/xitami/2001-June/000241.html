BTW since TCP/IP stacks are supposed to check the hosts file before DNS 
(and in the MS 95 and 08 stacks do) then you can do some sneaky things 
by (for example) mapping ad.doubleclick.net to 127.0.0.1 and thus 
avoiding tiresome adverts

I have found however that occasionally this causes problems and run very 
very slowly. I am not sure that this is a Xitami problem but soemthing 
in the path of Netscape/Win95 IP stack/Xitami means that the time taken 
to resolve such addresses is very long.
