Hi.  I'd wondered who it was that was maintaining the FreeBSD port.

>I can state that for me, Xitami runs just fine, at least the HTTP and
>CGI part of it (I don't use the FTP part because FreeBSD has a FTP
>server of its own). I'd like the log to be written via the syslog daemon
>though, rather than directly into files.

It's good to hear it works well.  FWIW, most webservers write their
own logfiles (eg, Apache does as well, as well as almost all the "tiny"
http servesr I've seen), because they have special requirements for log
files which syslog doesn't handle quite so well.

We're in the process of trying to revamp the build system for all our
open source software tools to make it a little more flexible, and
hopefully a little more unix friendly.  (I develop mainly under Debian
Linux, but a number of the other iMatix developers are more Windows
focused, and this shows in a few places in Xitami.)

If there are particular things that cause a problem with the FreeBSD
port which we could look at please let me know, either directly or
ideally on the Xitami-Dev mailing list (you can subscribe at:

http://lists.xitami.org./mailman/listinfo/xitami-dev

or by emailing xitami-dev-request@lists.xitami.org)
