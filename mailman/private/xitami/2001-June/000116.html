>	I'm using Perl/CGI to redirect visitors to certain pages based on which 
>page they've come from.  I know there's a "last-page-visited" HTTP header 
>(HTTP_REFERER), and I know that Xitami logs it via NCSA, but for some 
>reason it's not passed to Perl.  

When I test this with I get a HTTP_REFERER header available in the
environment (eg, with /cgi-bin/testcgi) if the browser provides it. 
(Generally the browser will provide it if you've clicked on a link to
get somewhere, and it hasn't been configured not to, and there are no
proxies in between configured to strip it out.)

Do you not get that variable?  If so what HTML are you using to provide
the link to click on, and what version of Xitami?

As far as I recall, Xitami passes all HTTP headers through to the CGI
programs as HTTP_..., and in addition sets a few other variables (from
the CGI spec, and forms).
