In message <F133rIkyPzGNsokLQ8a0000e0d9@hotmail.com>, "hank doe" writes:
>http://www.dummy.com/users/<username>/.....
>
>i've got 2 problems now. the first one is about cgi's.
>1) how do i setup cgi-dirs for every of those users ?

Set the "cgi anywhere" flag (beware, this is a security risk if you let
people upload their own content: they can upload and run any program
they like on your system).  [CGI] wildcard=1 controls this.

>the 2nd) (i suppose this is the more elegant way)
>can i achieve the same with virtual hosts ? and how to setup them,
>probably there is an idiot proof instruction manual ? :-)

If you can live with urls like:

http://username.example.com/

then you can give each a virtual host.  You can set these up in the
virtual host portion of the web based admin -- and once you've got the
config files for one you'll probably find you can easily replicate them
by copying an existing one and changing the names.  With the virtual
host approach you can give each their own cgi-bin directory, and
webpages directory and so on.
