In message <3B1CFB2A.6DD6E408@evernet.net>, david writes:
>open source...does that mean anyone can start helping add
>new features to Xitami??? like ISAPI support??? :-) 

Xitami's always been open source, and we welcome contributions of new
features, code, etc.  Some parts of some versions of Xitami are "value
added" extras, and were closed source -- the Windows GUI/NT Service
portions used to be "value added" closed source extras.  But they're now
open source too.

The Xitami/Pro SSL support is still closed source, as the (small)
revenue from it helps fund Xitami development.  But we're trying to make
the source available for as many of the other bits as is practical.
