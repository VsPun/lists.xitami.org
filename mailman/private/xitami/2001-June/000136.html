In message <2D123C9B36C5D31182B90090279CF9A6295F04@CLIFFY>, Mike Shumko writes:
>I am using Xitami 2.4d7 on WinNT, the 'service' version.
>I do not want Xitami to produce a directory listing when there is no default
>web document present in a given directory. The dirlist switch does not seem
>to work for suppressing these listings. That is, regardless of whether I
>enable or disable it, listings are always produced. 

The dirlist option appears to operate correctly with 2.4d9 under Linux;
if set to "1" then directory listings are allowed, if set to "0" then a
forbidden message is generated.

Beware that there are two dirlist options; one for the CGI directory and
one for the overall server security.  The one you want to disable all
directory listings is:

[Security]
  dirlist=0
