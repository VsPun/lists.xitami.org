You can have Xitami behind the firewall. You have to configure the =
router to pass all traffic for port 80 to the PC running Xitami.

Here's how the request process should work:
Someone far away requests www.yourdomain.com. The appropriate DNS =
servers hand back www.yourdomain.com =3D 192.168.0.1 (These addresses =
are made up for this discussion). A request to port 80 of 192.168.0.1 =
for www.yourdomain.com is made, and Xitami returns the webpage.
If you are running more than one domain name on Xitami, Xitami Virtual =
hosts will recognise which domain is being asked for, and supplies it.

So now let's say you want to access the website from inside your =
network. You type is www.yourdomain.com. an appropriate outside DNS =
server hands back www.yourdomain.com =3D 192.168.0.1. Your PC says "Hey! =
That's only one PC away from me!" And connect directly to it.

As you can see, it's exactly the same mechanism for being inside the =
network or outside the network.

So what we are dealing with here is some other problem. If all the above =
works from the outside, but not the inside, it is almost certainly a =
firewall problem.
If you are having problems with both the outside and the inside, it =
could be either DNS not configured properly, firewall not configured =
properly, or Xitami not configured properly.

Hope this helps,
Tom
