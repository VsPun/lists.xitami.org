Very simple.

You add 101 files, not 2000.

Disable online administration and disallow the web admin interface for your
VHosts, as it would render Xitami wide open to security leaks having web
admin enabled.

As such, you point all AUT files for your VHosts to one single disabled
admin file.

Voila, 99 files out of 100 removed.

As for the other 100 files, I find it extremely useful to have them in
separate files, thereby simplifying the management of documents.

Besides, it is a very simple dead to write a CGI for your self, to run off
of the root on the server,
to effectively write your own admin panel, which only you have access to and
is hidden from 
the publicly known view, by the documentation files of xitami.

As such you can in fact turn off the web admin completely, disable the LRWP
from port 81, unless you use it for some ssl functions.
You should already have disabled the ftp, which is controlled from the
default.cfg anyway, in a INI file style.

All of which is really simple to manage.

That's the way I do it at least.
Pretty simple and 99% of my work is done by remote control anyway.
In fact I have not had a monitor on any one of my servers for the past three
years.
Except once, when installing a new machine 4 months ago.
