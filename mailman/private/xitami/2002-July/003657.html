>Tony, the only thing XiFusion does for PHP is correct redirection and a few
>minor issues dealing with the environment.  When you use the header()
>function in PHP under Apache, Apache will automatically modify the redirect
>if it is not an absolute HTTP-compliant URL and may perform additional
>processing locally (depends on the situation).

This is exactly my point. If I want to set a http-header (in any script 
langugage) I don't want the webserver or anything else to correct this for 
me. I want an error to be thrown if something is wrong. People working in 
this environment try to write portable code, but webservers unsolicited 
correcting things destroy portability and readability.

>PHP will work fine until you encounter a script that uses relative
>redirects.  At that point you either have to change the script, modify
>Xitami's source, or use a wrapper like XiFusion.

My opinion:
-) change the script: YES
-) modify xitamis source: NO WAY
-) use a wrapper: POOR SOLUTION, camouflages lost portability (and raises 
lots of questions in messageboards)
