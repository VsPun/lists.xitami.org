>Thanks - now all I need to do it get ftp working ;-(
>
>I am able to connect when I am within my home nework, but not from outside..
>
>I have set up my DSL router to forward port 21 to port 21 on the machine
>with the ftp server.
>
>I am using passive mode on my the far end.

Disable passive mode on your FTP client.  Unless it is "smart" enough to
understand protocol level commands, it won't allow anything through.  Your
end node has to accept connections through its firewall/routers or you
won't be able to do FTP data transfers.

>The far end seems to connect then tries to read the contents of the folder -
>it then times out and gives an error.

Disabling passive mode should fix this problem.  If it doesn't, then you
won't be able to FTP either way since there are two firewalls not willing
to talk to each other in the way.  You can try setting up a range of FTP
ports for incoming traffic to the server, but that opens a whole other
security issue that I don't have time to deal with.

Hope this helps!
