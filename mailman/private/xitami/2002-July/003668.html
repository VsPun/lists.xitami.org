> DHCP is designed to assign the same IP to the same adapter if the
> network permits it. each NIC has a specific identifier code, somehow  
>(and I have no clue how) each NIC sniffs out the DHCP server, and     
> talks to it.

For what it's worth and this isn't the networking 101 mailing list... 
but the DHCP server is listening on the broadcast address of whatever 
subnet everything is set to (usually in a 24bit subnet that would be 
x.x.x.255)... If there are routers between the subnet of where the 
client is and the (different) subnet of where the DHCP server actually 
lives on then the router(s) will have a helper address programmed in of 
where to forward on the DHCP request from the client... This is a 
really simple answer and I won't go into any more detail - if 
anyone wants to understand subnets, subnet masks and bootp broadcasts 
etc. in more detail then you will find plenty of information on the web.
