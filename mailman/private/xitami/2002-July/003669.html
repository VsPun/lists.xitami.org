>I have used the added perl code as a hitami *.html filter (for
>compression). Does anybody know why  my IE6 doesn't decodes the data? 

IE doesn't support that header as far as I know.  Actually, to be brutally
honest, both IE and Netscape rely on the HINTERNET set of functions
provided by Windows.  That set of functions wastes bandwidth by (usually)
sending out two requests for a webpage instead of one (for GET, two GET
requests - for POST, one POST and one GET).  I wouldn't be surprised if IE
depends on HINTERNET to do the entire decoding/translation and doesn't do
it.  Netscape might do post processing on the data if that header exists.
