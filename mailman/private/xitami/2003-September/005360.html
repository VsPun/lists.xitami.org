For those that don't know, reverse proxying is a way to connect
multiple servers on multiple ports on same machine to one port
or multiple servers on the local intranet to one port on one machine
connected on the internet.
So you need just one IP :)))

Here you have link for nice small and fast reverse proxy for HTTP protocol
http://www.at32.com/doc/rproxy.htm
Read the details on their page. You can also download demo version and test
it :)))
It's really neat stuff if you want to beat the number of connections really
cheap.

I took this desktop picture, if you like to see how it looks like having
10 xitamis, 10 cgis with LRWP and reverse proxy all tegether :)))
Click on this link to view the picture
http://www.normstudio.com/10servers.jpg

Now i have another problem - how to increase bandwith.
I'm looking for solution that has something with compressing html pages
before you send them so you could send 10 times less data... Or maybe just
some ballancing with server machines on different providers.
