"Josh A" <beta@coaxial.ca> wrote:

> I'm having a world of difficulties with the new packages as well,
> under winxp and win2003 machines, all have xitami running as a
> service, tried upgrading by overwriting files, adding new files, and
> reinstalling the service, but had alot of problems

What problems have you had, specifically?  Remember, you cannot mix 
files from 2.4 and 2.5, for such an upgrade you should install 2.5 in 
a separate directory and copy your webpages, scripts, and 
configuration files across.

To debug (config and other) problems with the service version, run it 
as a normal console process.  To do this, pause the service (if it's 
running), then open a console window, and run the service in debug 
mode in the Xitami directory:

  xisrv32 -d

> Is there any chance that an installer will be built that can handle
> both new installs and upgrades ? 

Whatever information you can provide about the problems you're having 
will help us make such a thing.
