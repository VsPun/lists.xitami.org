This is clearly not a Xitami error but a windows thing.
 
Defective RAM module? 
Replace the memory module with another one or run the server with less memory
 
Corrupt system files?
Typically caused by installing XP and SP2, and then copying the whole config from the old server to the new one
 
It seems windows is tring to run javascript/vbscript
Typically a virus or something like that trying to access internet explorer to do it's thing.
Check the registry (HKLM/.../run HKCU/.../run) and the task manager. Check for new files in the windows/system32 directory
