At 08:55 28/06/03, you wrote:

>Is there a reason you need to run two Xitami servers?
>All other things being equal (which they may not be), you can serve both
>domains from a single server.

There is a reason.
We have a "public" server (accessible on port 80) for our customers and a 
"private" server running on port 119. The router directs the requests to 
the correct server using NAT.
Both are on different machines for security reasons; with the private 
server it is possible to access our corporate data, mail, SQL. It is just 
another layer of security.
