If you know this, why are you using that port?  I made specific note of it
and suggested using a different port altogether...but that seems

>On any computer, if i type web.verfaillie.com:119 the connection can't be 
>made, since the browser expects a NEWS response of some sort.
>If i type http://web.verfaillie.com:119 the connection is made, because the 
>browser expects a HTTP response.
>The problem is the latest version of IE (the version you can download and 
>that's included with XP).
>It does tot allow http connections over a news port.

And I told you why.  HINTERNET is probably interfering and attempting a
NEWS connection.  HINTERNET is how IE communicates its desires for
information across the Internet.  Change the port number to some non-mapped
port that is not 21, 25, 80, 110, or 119 and your problems dealing with
HINTERNET will go away.

>It is not a problem with firewalls and ISP: when using a Netscape browser 
>(on the same computer), the page can be retrieved.

Your assumption that, at the firewall level, packets are not affected is an
incorrect assumption.  Firewalls can and _DO_ interfere depending on
whether they are passive or not.  They can do anything from just blocking
packets of certain types to modifying the packet itself before delivering
it.  IE 6.0 (via HINTERNET) _could_ be sending out packets that are getting
dropped by your firewall for being ill-formed or redirecting them to the
wrong machine.  I just wanted to make sure I covered all the bases and made
you aware of the fact that your firewall could be the problem.

>For now i suggest users to use Netscape. For now the site is only for a 
>limited user group.

Well, you will be continuing to limit your user group if you force them to
use older versions of IE or use Mozilla/Netscrape.

>I'm looking for some register fix to allow IE to connect.

to modify the Windows registry to visit your site.
