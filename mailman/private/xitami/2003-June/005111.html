> I have Drewhoo's home page at: sorcep.com/drewhoo/Drewhoo.html and
> you CAN'T point drewhoo.com to that address from Domaincow's DNS
> management!

Of course not. DNS is for machine-names, not sub-folders.

In DNS:
 sorcep.com -> sorcep.com
 drewhoo.com -> sorcep.com

DNS gets them to your machine. Then you tell XITAMI:

"If they ask for sorcep.com, give them (webpages)/"
"If they ask for drewhoo.com, give them (webpages)/drewhoo"
"If they don't say what they want, give them (webpages)/"

(That last case will happen with version 1 browsers, or if someone
types an IP number instead of a host-name. You can set IP numbered
V-hosts but it is rarely worth the trouble.)

Go into XITAMI's main Administration page, click "Virtual hosts".
Click "New".

Create virtual host profile:  [ drewhoo ]
Select IP address: [ (all) ]
Or, enter DNS host name:  [ drewhoo.com ]
Web page root directory:  drewhoo
Create directories if necessary?  [ ] (not checked)
Click "Create"

Since you already have the pages, I told you to NOT "Create
directories" so if you get the syntax wrong, Xitami will complain
and let you guess again.

The way you are set up now, (webpages)/drewhoo/ will pull up a
directory listing, unless you have an index.html in there. You
probably want to rename Drewhoo.html to index.html, so it will pop
up when drewhoo.com/ opens (webpages)/drewhoo/
