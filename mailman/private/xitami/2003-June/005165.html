>Folks, I recently installed Xitami on Windows 2000 as a service (v

>Properties" page and that did not seem to have any effect.  In fact once
>it starts hogging the CPU the only way to kill it is to reboot the
>machine (killing it from the task manager would not work).
>
>Has anyone else seen this CPU hog under Windows 2000 and know how to fix
>it ??
>
>Matt
>
>PS: On a possibly related issue, I was wondering why Xitami was using so
>much of my hard disk when I found a .lst file of >1Gb.  Due to size

>some sort of core file.

I've never seen that problem occur except during development sessions when
a CGI process ran rampant from an infinite while-loop or something similar.
 However, since I have dual-processors, I never really notice CPU usage as
Xitami only utilizes one processor...leaving me the other one to do other
things like compile applications, etc.

If Xitami is running as an NT service, you have to kill it from the service
manager or a command prompt using the 'net stop' command.  Task manager can
only kill processes owned by your user...you can't kill "SYSTEM"-owned
processes directly.

Hope this helps!
