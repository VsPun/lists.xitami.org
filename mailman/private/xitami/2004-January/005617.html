Hi Steve,

Maybe it would be best to really understand what your router does for you.
I'm not familiar with the Linksys, I personally have D-Link. So details may
vary, but the jist of the story should be fairly familiar.  I am, by no
means, an expert, but here's my take on it.

When someone sends a request to your domain name, say www.mydomain.com, it
really goes to an IP number, connected to the internet, say, 24.86.62.150
(don't try this address, I just made it up). Now, that would be, in our
example, the IP address of your router.

What your router does with that request, is that it looks up in its list of
information to see if the requests for the port specified should be
redirected somewhere. So let's assume this was a webpage request, on port
80. Your router needs to know to which computer to forward that request. So
say it goes to your computer at 192.168.0.121. The router will take that ip
packet, and forward it to that "behind the wall" address to be dealt with.

What you need to tell your router, is that whenever there is a packet coming
in for port 80 (the standard port for web servers), it needs to be forwarded
to 192.168.0.121 (or whichever LAN IP number your computer has.

I hope this is helping you.

Cheers,

Real Martel

      I am trying to run the Xitami web server on my WIndows XP machine that
is connected to the internet via a cable modem (RoadRunner/Motorola
Surboard).  The internet connection is split amongst 2 computers using a
Linksys router.  When I start the web server everything seems fine, but when
I try to access the web page I have loaded on the server by connecting to my
internet connection's (cable modem) IP address, I am directed to the
router's configuration page.  This also happens when the server is turned
off, so it's not a big suprise.  What is odd is that when I try to connect
to the router's config page by entering the router's default IP address into
IE, I am not taken to the config page as usual, but I'm instead directed to
the web page I have installed on the Xitami web server.  When the web server
is turned off I go straight to the config page as expected.  It seems like
somehow the router's IP is bouncing me back from the router to the computer
with the server installed, and the cable modem's (internet connection) IP is
forwarding straight to the router and never reaching the computer with the
server installed.  I assume this is a relatively simple fix for any of you
that are knowledgeable with networks, but I can't figure out what to do to
fix this & make it work with my set up.  I also cant run the server and a
file sharing program like KazaA or WinMX.  Is there a way to allow for these
programs and the server to all run together harmoniously?  Thanks in advance
for any help, and I apologize if this question has already been beaten to
death.

      Steve
