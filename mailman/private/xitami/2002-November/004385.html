Hello !

I think you will have to do it by configuring your router to allow your 
requests.
imho since port 800 is not a well know port, no good gateway will allow 
requests through this port if not configured adequately.

My mailserver (Merak) allows imap, web and wap.
To use wap, i have configured it to listen to port 119.
This is the port used for news, but i don't have a newsserver.

Portnumbers are actually just numbers, and you can use any number if you 
wish, but since this portnumber is a well known number, gateways will not 
block it.

You still will have to do a single redirect from a webpage (so you can use 
a easy to remember name like <wap.maddrive.com> instead of 
<http://mail.maddrive.com:119>)
Attention, a wap page is not like a normal html page. Even Internet 
Explorer won't display a wap page correctly, so debugging has to be done 
using some PDA.

Marc

PS: i'm quite interested in this matter since i'm also struggling to create 
wap pages.
But it's a lot of work and nobody actually uses it!
