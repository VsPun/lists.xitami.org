I always enjoy these forums....even after 6 years of use I constantly learn little tid bits that ultimately help me better manage Xitami.  A recent post about PHP and CGI processes made me wonder about a few things.  hopefully some folks here have some insights into these items.

I recently learned that PHP, in Xitami, is actually run as a CGI process....if thats true then I see no benefit in running PHP scripts - what advantage would PHP have over CGI.  Maybe smaller more adaptable HTML coding can be achieved with PHP, but as far as the server is concerned, a PHP script will still generate a CGI process.  Yes/No?

And nearly every perl script does at least 3 or 4 mysql queries to pull data needed for the html pages.  What if a 1,000 users hit my site at the same time, can Xitami handle it - can the server cpu/memory/xitami handle approx 1000 perl.exe running at the same time?  What about 2,000 or more?  Who has extensive experience with load issues on Xitami that could comment on this, how to deal with these issues and ultimately suggestions on how to optimize windows and xitami to have the best performance?
