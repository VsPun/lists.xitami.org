Hello All,

I am in the middle of porting Xitami (24d9) to the AS/400.  I have it all 
compiling and am working my way through the platform inconsistencies and 
assumptions (i.e., some things don't work :)

However, I have some questions regarding the capabilities of Xitami that 
are not apparent from my reading of the documentation.

Q/  Does Xitami provide full support for HTTP/1.1?  Specifically HTTP 
header directives relating to compression. This is part of the Content 
Encoding and supports must_server and must_client (something like that) tag 
directives.

I see the 24d9 documentation indicates that virtual hosting from HTTP/1.1 
is supported and the Xitami 2.5b5 (Beta) docs say the following:

"Further support for HTTP/1.1, including partial GETs. We now consider 
Xitami to be a HTTP/1.1 server, although a number of optional HTTP/1.1 
features are not yet implemented. The critical features implemented are: 
keep-alive handling, document publishing, resume (partial GET), and 
HTTP/1.1 virtual hosts."

which suggests to me that Xitami does not support these functions.  Is that 
true?  If so, is it likely that they will be supported?  If so, any idea on 
time frame?

Regards,
Simon Coulter.
