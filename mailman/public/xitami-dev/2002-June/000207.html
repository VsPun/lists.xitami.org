In message <F183xLJQHRWT5K5TEOY000140dd@hotmail.com>, "Fabien Royer" writes:
>I am just about to re-write a C++ version of the LRWP C library [....]
>Before re-inventing the wheel, I'd like to know if someone has already done 
>this? A quick Google search came out with nothing on the subject.

AFAIK no one has done so already.  (The implementations that I know of
are C, Perl (an example), Python and Java, plus some sort-of-released
ones for things like Visual Basic, etc.)

>IMHO, the fact that the current implementation of the LRWP C lib is bound to 
>SFL (which requires single threading) is a mistake.

The LRWP C library was the original proof-of-concept library, written by
Total Control Software, with minor updates as things went along.  By all
means rewrite it using something else.  The Java implementation (in the
form of the Sokrates Servlet Runner from Conacom, etc) is probably the
most complete multithreaded version.
