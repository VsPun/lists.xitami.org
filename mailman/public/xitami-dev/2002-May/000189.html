In message <10c.1275faf9.2a24ec41@aol.com>, TonicPeddler@aol.com writes:
>Is there a way to customize the way that xitami lists directory contents?  
>For instance, what information about the files it make available to the 
>viewer, like date of last modification, file size, things like that?  Is 
>there an easy way to point xitami to a cgi application to handle directory 
>listings?

If you are using the Xitami 2.5 beta, then you can customise the GSL
script that generates the directory listing, in the templates directory.

If you are using the release 2.4 version then unfortunately it's a fixed
format.  The cloest thing you could do would be to create index.html
files, or use something like PHP and set index.php as the default
document for the directory.
