Hello,

There are lots of different "kill" tools out there. Most of them can be set to different "brutality" levels. First they try to stop the running task by sending a kind of "Alt-F4" command. When the task is still running properly it will react by closing all open files and terminate the regular way. After this, when no reaction, the running .exe can be really "killed" and can be kicked out of memory.

I'm running a scheduled task that is testing the Xitami server every minute by a simple http-Get, and if there is no answer probably Xitami was shot by unknown reason (maybe my own PHP scripts do so by fault). This happens 2-3 times a day to me. After a defect is recognized, Xitami becomes killed and restarted in the way I mentioned. I never found problems when doing so. No problems with open files, even when PHP scrips are called.
