I've not done any multi-threaded development myself either, but AIUI, the
support for multi-threading in OS/2 is an order of magnitude more
efficient than that in ANY dialect of Unix.  If you cannot properly
"corral" multiple execution threads, perhaps Xitami needs to make a
conscious effort to disable threading.  (Given that OS/2's multi-
threading support runs to 64 processors; also given that eComStation is
going to make a cheap SMP O/S available to all those with dual Pentium
motherboards.)
