In message <3B150CD6.A420E427@trmail.triumf.ca>, "Rod B. Nussbaumer" writes:
>I've recently upgraded to 2.5b4 on Win95 (!), and now whenever 
>I try to download files from it using either the FTP server or 
>using an http: URL, the FTP client or browser sees the download 
>filename as one long name with the period(s) represented as %2E, 

This happens when downloading a file out of a Xitami-generated directory
listing?  In which case it sounds like Xitami is being a touch too
paranoid about escaping the filenames.

You might want to experiment with changing httpdir.gsl; there are two
versions of each filename passed in: one for display, and one that has
been "escaped" to solve any problem characters.  The latter is used in
the URL by default.  You could experiment with changing it to use the
unescaped name, and see if that avoids your issue.  (It may introduce
other issues though, especially if you have filenames with spaces in
them, etc).

I'll put this on the bug list for more investigation.
