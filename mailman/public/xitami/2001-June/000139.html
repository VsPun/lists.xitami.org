"The function you requested is not implemented" is usually generated
when you refer to a WSX which is not compiled into Xitami.  In this case
you are explicitly setting WSX references for xixssi, which shouldn't
normally be required (they are in xitami.cfg for the versions have
xixssi):

>[Wsx]
>     .shtml=xixssi
>     .ssi=xixssi

You don't say what version of Xitami you are using.  If you are using
Xitami 2.4 you need to use perlssi (as a filter, it should be
preconfigured in the default configuration).  If you are using Xitami
2.5, then xixssi should be installed and set up automatically, in which
case you don't need those [wsx] entries.
