Greetings;

Yes, there were errors in the log. Exactly as I would expect from
what I discovered yesterday after Matt started me thinking in a 
different direction.

Perl is passed the full path to the cgi script, and that had a space
in it. OK for windows, not OK for perl!

And thanks to Xitami's ability to be reconfigured almost as fast as
you can click the mouse, I found out a few other interesting things.

Unlike a well known server Xitami doesn't care what the script name is.
(The other server requires a cgi extension in some configurations.)
And, perl will accept any valid windows path, except for the above
problem ... No Spaces! Verylongdirectorynames and very.long.file.names
cause no problem. There may be a max length path restriction but I
did not encounter it.

Happy Xitami-ing!
Dennis
