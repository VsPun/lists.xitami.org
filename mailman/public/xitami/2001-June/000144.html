either MS Personal Web Server or IIS to exploit your server.
I'd guess IIS, since they tried to run a command using the
interpreter located in c:\winnt, meaning you were running
Windows 2000 or NT.

61.139.59.73 - - [08/Jun/2001:16:38:03 -0500] "GET http://www.yahoo.com/ 
HTTP/1.1" 404 0

I bet some script was just looking to see if you were running
a web server; if you have a telnet server installed on your
machine, try telnetting into port 80, and type:
GET http://www.yahoo.com
with two CRLFs after it.  This should produce the same error
as this one in your access.log.
