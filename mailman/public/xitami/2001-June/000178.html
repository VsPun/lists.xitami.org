In message <001101c0f4b9$69b80a90$123d60cb@cable.net.nz>, "garry" writes:
>Hi i have used xitami for many years and am happy with it
>my question is how do you run say apache beside xitami and have the right
>server pick up when the request is to a domain hosted on it ?

I'm assuming that you want to run both on the same server, and both
without any special port numbers, in which case your choices are:

(a) run them on different IP addresses, and have the DNS for those
domains point at the appropriate IP address.

(b) have one server forward requests for some domains to the other
server (Apache can do this with mod_proxy; Xitami needs some sort of
add-on, perhaps a LRWP program, to do this).

Of these (a) is somewhat easier to handle, but (b) will work just fine.
However if your reason for running Xitami is to handle "simple" requests
without bothering Apache, then (b) might not be such a good solution.

For just testing, simply running them on different ports is often the
easiest solution.  (Both Xitami and Apache can be told to use different
ports.)
