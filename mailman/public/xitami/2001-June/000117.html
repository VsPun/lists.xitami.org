In message <3A7A515A.9020500@pacbell.net>, tom mcpharlin writes:
>I'm running Xitami 2.4.d7 on a windows 2000 system. When I try to access 
>the admin page, I enter the name/password in defaults.aut and then get 
>the message:
>"The function you requested is not implemented"

As far as I can remember this message will appear only if Xitami detects
the path as a WSX (web server extension, like the admin pages), but 
then cannot find the extension that is supposed to be there (eg, because
the name is wrong).

The [Wsx] section of the config file specifies that "admin" should be
handled by "xiadmin"; if you've changed your config files (eg,
defaults.cfg, etc) for some reason so that "admin" points at another
name that might cause the problem.  So perhaps check carefully through
your defaults.cfg file.
