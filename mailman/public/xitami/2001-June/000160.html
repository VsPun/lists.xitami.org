>The 4th string appears to be what the ftp service logs in xitami when a user
>issues an "ls" command. 

Yes, that's correct.  As far as I can tell what gets logged is:

ftp://YOURIPADDRESS/TEMPDIRNAME/TEMPFILENAME

wher YOURIPADDRESS is the IP address they FTP'd to; TEMPDIRNAME is
whatever name you've told Xitami to use for temporary files (by default
it is "temp" which uses the value of the TEMP environment variable if
set), and TEMPFILENAME is the temporary file Xitami creates with the
directory listing in it prior to sending it.

>This seems like a confusing way to log this? ewen?

The difficulty is that Xitami tries to log these in the same log as the
HTTP requests, and hence needs to make them "distinct" in some way.  And
"ls" type requests don't transfer a real file.

Suggestions for alternatives welcomed.  Maybe it has to be a separate log?
