>ALIAS directory where DIRECTORY lists are enabled so that the downloadable
>files are listed.  The ALIAS lists and ALIAS directories portion works well,
>but whenever a file gets downloaded,  the extension is either duplicated
>(i.e. "song.mp3" becomes "song.mp3.mp3") or an additional character is added
>to the extension (i.e. "file.exe" becomes "file.exe(1)" ).

Does this ONLY occur in directory listings via aliases?  Or does it
occur with any directory listing?  My guess is that it would occur with
any directory listing generated by Xitami, and I suspect it is related
to the URL-encoding being done by Xitami.  You can perhaps work around
this issue in httpdir.gsl (in the templates directory), by using "name"
rather than "nameurl" in the links. (This will cause some other problems
though, especially with names with spaces in them.)

If it does occur in all listings it is already on our bug list.  If it
occurs ONLY in directory listings via aliases we'll need to narrow down
what is special about the directory listings via aliases so we can add
it as a new bug.
