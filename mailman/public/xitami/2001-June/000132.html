In message <E158S7R-0000Ee-00@Mail.MH-Hannover.DE>, "Dr. Martin R. Hadam" writes:
> I've been moving machines and today, Xitami provided some message, for
>the first time:
>2001/06/08 21:32:16: smthttp: HTTP service is now running
>Fatal error in application - aborted
>Abort at smtsock:Check-For-Expired-Requests: (Getting-Event, Empty-Event)
>Abnormal program termination

Ah, that sort of problem.  You might want to keep an eye on whether it's
always at the same point, or whether the problems shift around.  If they
shift around it's most likely that something is writing to memory it
shouldn't.  If it's always (or usually) one place, then possibly there's
something wrong with the way that some requests are handled by smtsock.

Unfortunately either way they're difficult to diagnose problems, since
they're not easily reproducable.
