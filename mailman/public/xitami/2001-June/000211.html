Folks,

There is another way, I think better. I have a site
http://www.altair7.com/index.html that uses a perl script for opening page,
see for your self.

I do not have the SSI_INSECURE set.  I do have MIME types defined as following
for the master config:

	 .pl = c:\perl\bin\perlis.dll
	.cgi = c:\perl\bin\perlis.dll

This solved all my problems.

Steps:

1. http://127.0.0.1/admin
2. configuration -> mime
3. enter the above values
4. save
5. click on restart from the main page

Done.

With that done, then this works:

	http://www.altair7.com/cgi-bin/od.pl

This works with xitami 2.4d1 through 7 and xitami pro versions.

Hope that helps, always make sure that you protect your perl scripts.

Cheers,
-Roberto

> 
> Click OK
> 
> Reboot?  (I don't know if this is required.)

> SET SSI_INSECURE=yes to the autoexec.bat.
> to run SSI.
