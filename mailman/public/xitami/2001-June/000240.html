 I think there is misunderstanding: I did not build Xitami myself, I
only used the patched version that was made by Bernd Hohmann. HOWEVER,
the same applies also EACH version after v2.4d6 that was available on
the Xitami website. So it's probably independent on the tools used to
build the binary.

 When I first noticed (without doing the execmode test), I though that
it might relate to my EMX version used, which is the newer than the one
recommended. My current version is:
EMX : revision = 61
EMXIO : revision = 60
EMXLIBC : revision = 63
EMXLIBCM : revision = 64
EMXLIBCS : revision = 64
EMXWRAP : revision = 60
 I should add, however, that this does seem unlikely in view of my
later observations using execmode. Also, prior revisions (still above
the recommended level) showed the same problem.
