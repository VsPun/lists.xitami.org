Xitami is even more generous than Apache: providing the #! line _ends_
with "perl", _and_ the perl binary is on the Path, Xitami will run it.
(It tries the exact path first, and if that is not found, it strips off
the path name, and tries that instead.)

So put perl in c:\perl, and add c:\perl\bin to your path, then start
Xitami.  And your #!/usr/bin/perl lines should work just fine.
