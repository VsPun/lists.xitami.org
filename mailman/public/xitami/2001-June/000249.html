I assume you are using the NT Service version.

The Service version is appropriate if you want Windows to auto-start the
service even before any user logs-on. My web server sits at the log-in
prompt for months on end, because there isn't any reason to log-on at
the terminal since Xitami starts before that as a Service.

If you or some process you wrote wants to be able to start and stop
Xitami at will, I'd think the Console version is more appropriate.

If Java can call an arbitrary DOS command, I think there is a
command-line option to kill the Service. But a stop-start mode is more
what the Console version is made for.
