Not that we are aware of.  I suspect that for some reason something is
overwriting a bit of the memory that Xitami is working with, causing it
to get confused and crash like this.  Unfortunately they're very
difficult bugs to track down.

appreciate any comments on whether it is better or worse.  (At least
with those versions I know we've got the source code it was compiled
from safely in our source archive which makes it somewhat easier to
investigate problems.)
