Congratulations, you have found a bug in Xitami :-)

What seems to be happening is that Xitami was lower casing the URL
somewhere along the way, but since this server is running on Linux, when
it does that with paths that are actually in upper case, it can no
longer access the directories: hence 404.

I've worked around this temporarily (symbolic links), which will last
until the end of the month (I hope), and obviously we'll put this bug
onto our bug list :-)

Thanks for chasing this up.
