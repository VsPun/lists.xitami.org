To go along with this line of thinking..

The non-paged memory pool on Win32 platforms is *not* something you want to
run short of. In an earlier message it was stated that the RAM disk was held
in this memory pool. Bad idea, on a high-load server, if this pool runs short
your machine will crash and die a horrible death.

Ewen McNeill wrote:
> 
> So basically a ram disk is betting that you know better than the OS what
> is being used a lot: it certainly used to be true with DOS, but it's
> far less true with a modern OS.  Usually the ram is better just left for
> the OS to use for caching; but experiment and see what you think.
> Remember that ram you take for the ram drive won't be available for
> running programs; which if you have limited ram can force the machine to
> swap, which is rather... bad.
