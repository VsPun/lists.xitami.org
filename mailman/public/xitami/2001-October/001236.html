The hint I immediately think of with Java is that the filenames are case
sensitive; whilst Windoze is quite happy with any spelling of source and
class names, java.exe and similar programs all look for the file with the
exact spelling.  It can be very confusing on a computer that, like
Windows (or, for that matter, OS/2 with HPFS) can permit filenames to be
written in upper, lower or mixed case, and yet the filesystem itself will
find the file with any form of the letters.

Could that be a factor here (I know nothing about supporting JSP through
ServletRunner)?

The other idea that springs to mind is do you have a CLASSPATH
environment variable?  If this does not include '.', the current
directory, classes will only be sought in the directories listed in the
path; this is a security feature.
