We received a CGI program from a third party that is suppose to redirect the
browser using the "Location:" header instead of outputting the usual
"Content-type: text/html" MIME type header.  At this point it is not working
and we aren't sure yet if it is because the program is not outputting the
Location header correctly.

What I want to verify is that Xitami does NOT output the "Content-type:
text/html" header by default when executing CGI programs from the cgi-bin
directory.  If it does, then this is obviously the cause of the problem.
And if so, is there anyway of preventing it from doing this?
