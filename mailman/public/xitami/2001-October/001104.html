In message <20010922114028.M77@eyup.org>, John Poltorak writes:
>#4  0x58753 in _closedir ()
>#5  0x1a99d in close_dir ()

Ah.  I think I know what's going on -- there was a bug in Xitami where
if it failed to open a directory it would still try to call close_dir(),
which would then get very upset because it didn't have a valid directory
handle.

I think this problem is fixed in the version in CVS, as I seem to recall
a bug like this was reported a while ago.

If you want to fix it yourself and rebuild the binary have a look in
smtftpl.c (for ftpc_make_listing()), and change the logic for
opening/closing the directory around so that it only tries to close it
if the open was successful.

Xitami for OS/2 was built using the EMX compiler suite, plus REXX.  Otto
will generate REXX build scripts (but pre-built versions of these should
already be in the source kits), and Libero is used for generating some
of the other code (again, pre-built versions are in the source kits).
