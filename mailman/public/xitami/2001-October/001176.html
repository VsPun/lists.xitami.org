Hello,

I'm trying to setup an NT4 System as an FTP Server for Xerox Docutech job
files. We currently have a SunOS server, but need massive amounts of cheap
disk space and have an existing NT system available.

Running Xitami 2.4d9, I can get the DT (Docutech) to connect and login, but
its LIST request is not handled by Xitami the same way as the Sun FTP
server. I put Xitami in Debug mode to examine the requests. Then I used a
command window to compare output from the offending command on NT and Sun.

The DT is sending a list request as follows:
ftp> ls \dtf-tests\*\*Job.type
200 Command okay
150 Preparing to transfer ASCII file
226 Closing data connection

As you can see above, this returns nothing on Xitami.

On the Sun I get the following output:
ftp> ls /dtf-tests/*/*Job.type
200 PORT command successful.
150 ASCII data connection for /bin/ls (192.168.0.3,2072) (0 bytes).
/dtf-tests/test1/DocuJob.type
/dtf-tests/test2/DocuJob.type
/dtf-tests/test3/DocuJob.type
/dtf-tests/test4/DocuJob.type
...
226 ASCII Transfer complete.

The above list response is what the DT needs in order to recognize jobs on
an FTP server.

I assume this problem has to do with the wildcards in the path, because I
can list the files on Xitami using a complete path. However, I have no
control over the formatting of the DT list command. Is there a way to get
Xitami to handle this kind of list request?


- Jeff Hamblin
