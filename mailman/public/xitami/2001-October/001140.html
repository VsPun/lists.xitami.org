Andrew Morris wrote:

> Has anybody found that Perl scripts run intolerably slow on Windows 98/NT
> with Xitami? I'm using ActivePerl 623 and the problem doesn't appear to be
> specific to the machine or the installation. Even a small test Perl script
> (e.g 'hello world') type stuff takes up to _30 seconds_ to appear in the
> browser. I'm only running this on a local-loop. Does the problem go away
> when you're calling the server from a remote computer?
> Any info or insights appreciated.

Using Win 95 and win 98 I have had no problems with perl - I am
currently using ActiveState v 629 and Xitami 2.5b5 but I have had the
same results with earlier builds of both.

What I have found is that sometimes when I really run out of memory (I
have a 64MB machine) then things are very slow. But this is not related
to perl or Xitami per se.

One other thing that may be is that your perl script is not completing
correctly and is hanging around. In that case Xitami eventually kills it
but it takes some time. Finally the 2.5b1 and 2.5b2 versions did not
pass input to STDIN correctly for CGI scripts. This was fixed in 2.5b4
and b5 ...
