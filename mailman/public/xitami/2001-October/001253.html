In message <3BD59C16.731E49D0@rci.rutgers.edu> Paul Reid writes:

> Tip: when setting up webservers and their frills, unless told otherwise,
> only use the forward-slash "/" to define paths, even on DOS\Windows. In
> almost all cases, this works (even at a Windows command line); C and
> Perl programmers use "\" for something else so if they didn't handle the
> DOS case perfectly the \-ed path gets mangled. 

Errm, paths with forward slashes are accepted just about anywhere in
Windoze *except* in a command window; there, the forward slash introduces
switches (cf. with use of '-' in Unix).  Certainly internally all the
file manipulation functions and APIs do indeed accept forward slashes.
But not in an "MS-DOS Prompt".  Once upon a time, one could run a little
utility program which changed the "switch character", which is held in
one byte within DOS's interrupt vector, and thus use '-' to introduce
switches on a command-line: one could then indeed use '/' as a path
separator.

Sadly, MICROS~1 deliberately broke this at DOS V5.0, so even if the
vector content is changed, command windows still only accept '\' as a
path separator.

However, I fully support your viewpoint that ONLY the forward slash
should be used within configuration files; only use a backslash (which
might require escaping as "\\") in situations where the forward slash is
unacceptable to some bit of Windoze code.
