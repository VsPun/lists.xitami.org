ProtoNova is a closed source web server designed to accomplish several goals:

1)  Demonstrate that the Unified RFC Protocol is a viable communication
protocol to the IETF (Internet Engineering Task Force) and get it accepted
for standards track development.  This indirectly dealt with the CGI DoS
issue found in Xitami since CGIs are a side-effect of ProtoNova rather than
a part of it.
2)  Provide an alternative to Xitami users (as you already pointed out).
3)  Solidify the LibLink code base (the core library operating behind the
scenes...all application code is 100% portable once LibLink is ported to
another platform - LibLink is currently weighing in around 31,000+ lines of
very stable, modular C code).
4)  Simplify the application code base.  ProtoNova is only about 5000 lines
of C code in one easy to maintain file (each remote module is about 1000
lines of code).
5)  Create several new innovations such as database logs (instead of log
files).
6)  Simplify server configuration (uses a tree-style INI format).
7)  Introduce HTTP/1.1 compliance that supports chunked transfer-encoding.
8)  Provide basic clustering, load-balancing, and server farm support while
maintaining simplicity.
9)  Add SSL/TLS support.
10)  Create symbolic links for a server running in the Windows environment.
And many more...

When comparing ProtoNova to Xitami, it should be compared with Xitami/Pro.
Both have SSL/TLS support, a pricetag, and are closed source.  Xitami is
different in that it has none of those three major things in common with
ProtoNova or Xitami/Pro.

Basically, I performed an analysis of Xitami and determined that it would
be better to start from scratch than to modify the source due to the fact I
had so many changes I wanted to make.  A _VERY_ detailed Microsoft Excel
comparison chart of several different servers can be found in the ProtoNova
web server 'httproot\docs' directory (I can make this available if enough
people want to see it).

$35 for Standard Edition.  $75 for Enterprise Edition.  $300 for Custom
Edition.  All three editions are described in detail on the Shining Light
Productions website.

You have probably heard that "a fast nickel is better than a slow
dime"...I'm hoping to make an ultra-fast penny.

As to the issue of money, I really don't like to charge money for my
products.  However, when a product takes a long time to develop, I need to
get something back for it...*especially* if there is to be PFR support
(Priority Feature Requests - 24-72 hour turnaround times/product
releases/product updates - only Custom Edition users can issue PFRs) and FR
support (Feature Requests - usually 3-4 weeks, but can vary - owners of any
Edition can issue FRs).  Given that information, the product is practically
free.

Basically, as you can probably tell, the cost of the product is to just
cover support for it.  The fact it is so cheap is because of the
distribution system used (without the distribution system, it would be too
costly for end users for Shining Light Productions to even consider
developing it).  I'm not sure how iMatix distributes Xitami/Pro since I
haven't purchased it, but I would be interested in knowing how your system
works and I might be able to provide a few pointers to allow you to reduce
the price (or at least make it more cost-effective).
