I am running a student webserver under Xitami v2.5b4 on an old Win95 box.
Works fine.

Recently, I tried setting up virtual hosts (using the wizard, mostly) for
different teams, and trying to give each team their own logs, so they can
see what their errors look like.  Xitami created the logs for me
(teamXX.log, teamXXerr.log), and in the correct directory.  However, none of
the logs has any entries -- all the information is going into the default
logs under C:\Xitami.  This seems to be because Xitami is ignoring the VH
entries and using the defaults.  Here are the contents of defaults.cfg, a
team config file, and an extract from debug.

In defaults.cfg I have:
[Virtual-hosts]
    Eaglebay.ewu.edu/teamXX = "teamXX.cfg"
----------------------------------------------
In teamXX.cfg I have:
#*END

[Accesslog]
    Filename = "C:/Xitami/webpages/teamXX/logs/teamXX.log"
    enabled=1

[Errorlog]
    Filename = "C:/Xitami/webpages/teamXX/logs/teamXXErr.log"
    enabled=1

[Ftp]
    Root = "C:/Xitami/webpages/teamXX"

[Server]
    Defaults="defaults.cfg"
    Cgi-bin = "C:/Xitami/webpages/teamXX/cgi-bin"
    Webpages = "C:/Xitami/webpages/teamXX"
    Hostname="eaglebay.ewu.edu"
----------------------------------------------
In debug I see:
2002/04/23 13:12:06: Looking for VH using Host value: eaglebay.ewu.edu
2002/04/23 13:12:06:  - looking for VH using IP address: 146.187.25.27
2002/04/23 13:12:06: - using default virtual host values

So, any obvious errors? Any recommendations?
