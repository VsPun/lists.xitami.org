Do you have a setting for [server] webpages in your defaults.cfg file?
In theory the bug that caused that to override every virtual host
setting has been fixed, but I'm not 100% certain the fix made it into
2.5b5.

Debug mode will probably reveal if it recognises the virtual host name
or not, and if so which directory it is trying to use.  If it recognises
the virtual host name, but goes and uses the default directory, chances
are that there's still an issue with setting [server] webpages.
