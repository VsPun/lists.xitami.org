In message <3C971647.12443.10300F9@localhost>, "Becky R. Izen - MSSP" writes:
>Whenever they make an entry, I use the REMOTE_USER variable to capture which 
>employee is doing the work.  Up until March 5, this was working fine.  
>However, on March 5th I created an SSL certificate (private) and issued it 
>to everyone so everyone would be logging in via HTTPS.  

The REMOTE_USER setting comes from the HTTP authentication (ie, password
protected directories); it sounds to me like you've changed from HTTP 
authentication to some other authentication (eg, the HTTPS
certificate), so the REMOTE_USER value isn't being set.  You'll probably
need to change your script so that it extracts the user from the other
authentication method.
