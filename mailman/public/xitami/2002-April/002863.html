In message <007501c1d910$05c5fc90$dd31200a@JAZZ>, "Norman Makoto Su" writes:
>> One request per connection, at a time.  If you need to handle multiple
>> requests at once just open multiple connections.  If there is more than
>> one LRWP of the same "name" connected, then they'll be given the
>> requests in either round-robin or first-free fashion (depending on the
>> configuration option).
>
>So, in anycase, there is no way for one LRWP to know about the other LRWP
>connections (w/o some kind of explicit messaging via files/sockets/etc between
>the LRWP programs)?

Yup.  But if it's all in one process (eg, using threads, or SMT) then
the LRWP connections will "know about each other" anyway.  The Java
Servlet Runner uses this sort of approach to handle state between the
servlets.
