CGI
>envirionment variable QUERY_STRING?

No.

However it should be relatively trivial to parse either <STDIN> or
$ENV{'QUERY_STRING'} to create a suitable array. The problem is that the
output of a program that uses a CLI fo its arguments is unlikely to
suitable for display on a webserver as it won't generate HTML. What you
probably need to do is create a wrapper progrm (using for example the
CGI.pm library) that you can then wrap around all these scripts.
