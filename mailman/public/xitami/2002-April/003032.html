In message <000f01c1ea1e$1a3e6620$23a24118@kevin>, Kevin Brogan writes:
>for example, right now, i need to define two virtual hosts. one for http://mydo
>main.com and another for http://www.mydomain.com
>It it possible to instead, define a virtual host as http://*.mydomain.com?

No, there's no wildcard mechanism (other than making it he default
host).  You can, however, point multiple domain names at the same
virtual host config file, eg.

[Virtual-Hosts]
  mydomain.com=mydomain.cfg
  www.mydomain.com=mydomain.cfg

which can save some time in setting things up.
