hard.C.1@bwc.state.oh.us writes:
>When I run any test program, all is well. When I run a cgi program (that
>runs fine on my ISP with apache/unix), the browser "loading" bar at the
>bottom of the screen moves slowly then I get a 502 message saying that "The
>server is currently overloaded - please try again later". 

What this means is "it took too long".  Typically when you're testing 
that's not because the server is overloaded but because something your
CGI is trying to do is taking far too long.  Common things are looking
up names in the DNS, accessing a database, etc.  Basically in these
situations you have to try to figure out which bit it is that is taking
so long.
