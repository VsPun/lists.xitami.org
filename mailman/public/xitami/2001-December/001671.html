In message <001d01c17f66$92873210$6401a8c0@vastdeath>, "Vastdeath" writes:
>hey, I setup an alias to another directory of my hd

Add:

[alias]
   aliasname=directorypath

to the approriate virtual host configuration file (or defaults.cfg if
you are not using virtual hosts), and restart Xitami.  (You can also do
this through the web based admin.)

>and I wanted to know if I can deny get access from it, but allow 
>list access? this is with the http server

No.

You can do the reverse (allow GETs but refuse to provide a listing), but
not allow a listing and refuse to allow GETs.

If you just want people to be able to see what is there and not download
it I suggest you:
(a) don't use an alias
(b) create a HTML page (or a CGI to output one) that contains (outputs) 
    a list of files that are in that directory with whatever details
    you want.

The point of providing access to a directory is to allow people to
retrieve (GET) things from it; so it sounds to me like you don't want to
allow people access to the directory after all.
