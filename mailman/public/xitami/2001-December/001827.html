Peter (and Bruce)
     The thing that brought me to Xitami first was the easy way it handled
multiple domains.  I sell DSL service and had to offer web hosting to be
competitive.  Xitami was a perfect fit, especially for the price!  The
easiest way to create a virtual host is by using the VHost Wizard from the
main menu of the Web Based Administor (http://localhost/admin).  Looking at
the VHost configuration page:

1. Virtual host profile: (The configuration file created for all custom
settings for this domain).  I use the top level domain name less the
extension. (ie. for imatix.com I would use imatix).  The extension .cfg is
automatically used.

2.  Over-write existing file:  There usually isn't one, but if there is,
check this to replace the old configuration file.

3.  Select IP Address:  If you have multiple NICs and you only want one for
Web hosting, select it here, otherwise (default) let Xitami use "All" IP
addresses.

4.  Or DNS Host Name.  Enter the Web address here, ie: www.yourdomain.com

5.  Web Page root directory:  The location on your drive where the web site
pages are stored.  This is the directory where your customer has FTP access
to update page in.

6.  cgi-bin directory:  I usually cut-n-paste the directory above and add
\cgi-bin to it.  You can point this to any directory where you want where
cgi scripts for this domain are allowed to run.

7.  FTP root directory:  I don't use Xitami's FTP server, but I assume this
is the "home" directory for the admin of this domain.  (same as #5 I would
guess)

The rest of the items are pretty self explanatory.  I usually let Xitami
create the above directories for me.  Once you save it, you need to restart
the server and your all set!  The DNS entries for the domain only needs to
have a CNAME or A record for www.yurdoamin.com (which is normally a default)
that points to your public IP address.  My default, xitami will try to use a
page named 'default.htm' when no specific page name is specified on the
address line.  If that fails, it will look for index.htm.  You can change
and add to these in the VHost configuration screen.  If you look in the
xitami directory, you will find configuration files (and .aut security
files) for each host (domain).  These can be manually edited just like the
default file, or you can do all your maintenance through the web based
administration app.

Its all very easy, even though it looks complicated at first site.  Once you
get two domains running, you'll be off setting up web sites for all your
friends!  (just make sure you charge them something....xitami may be free,
but bandwidth isn't!!)
