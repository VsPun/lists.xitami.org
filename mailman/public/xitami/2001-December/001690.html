In message <20011122043023.40086.qmail@web12508.mail.yahoo.com>, Jesse Foster writes:
>I found this line in the crash log.
>21/Nov/2001:18:11:38 -0500 Xitami v2.5b4
>Abort at smtsock:Send-Timeout-Event-To-Self: Getting-Event, No-Activity-Event)
>
>Anyone know what this means in English terms?

This is Xitami's (somewhat cryptic) debugging information.  It means
that while running smtsock (the module in Xitami that sends and receives
things over the network), in the "state" Send-Timeout-Event-To-Self, it
ran into problems with one of the events it mentions.  

Basically this shouldn't happen :-)

Unfortunately this sort of thing seems to happen occassionally, "at
random" which suggests that Xitami is occassionally being confused by
something, and crashing.

If we can track down any pattern of thing that reliably happens before a
problem like this, so that we can reproduce the problem, then it'll be
easier to fix.  Until then the best we can do is try to fix likely
causes of the problem.
