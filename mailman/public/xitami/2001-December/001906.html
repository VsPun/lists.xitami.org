1.  #!  is the first line in your perl script.
2.  C:\perl\bin needs to be in your PATH
3.  all perl scripts need to go in your /xitami/cgi-bin directory (or what
ever directory you have designated to hold your CGI files
4.  The URL alias reference would be: <a
href=/cgi-bin/guestbook.pl>Guestbook</a> OR if you wanted the full path it
would be: <a href=http://208.33.152.220/cgi-bin/guestbook.pl>Guestbook</a>
5.  Your reference below is calling an .html file (guestbook.html), not the
perl .pl file

When calling a CGI file you need to call the script itself, in this case the
perl "guestbook.pl" file.  Xitami allows you to define where you are going
to keep your CGI scripts at.  You only get one root location.  By default it
provides you the /cgi-bin directory but you could modify that to
/my_cgi_files or /perl_scripts or what ever you want, assuming that the
directory EXISTS in c:\xitami\.  So when you are calling CGI files through a
href tag the CGI directory needs to be the first directory in the reference
path: <a href=http://x.x.x.x/cgi-bin/....>click here</a>.  Xitami see's the
/cgi-bin in the URL call and then goes to the /cgi-bin directory to complete
the rest of the URL reference.

valid calls assuming Xitami defaults (/cgi-bin must be at the beginning of
the path):
<a href=/cgi-bin/file.pl>
<a href=http://x.x.x.x/cgi-bin/file.pl>
<a href=/cgi-bin/perlscripts/file.pl> ## /perlscripts is a subdirectory and
it must exist for the URL call to work
<a href=/cgi-bin/blah/blah1/blah2/file.pl>

If Xitami is configured to accept /cgi-bin anywhere in the path:
<a href=/personal/cgi-bin/file.pl>
<a href=http://x.x.x.x/files/yourfiles/cgi-bin/myfiles/perlscripts/file.pl>

hope this helps.
