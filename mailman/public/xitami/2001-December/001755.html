In message <3C1BA07E.E901ADE3@rci.rutgers.edu>, Paul Reid writes:
>> If it's a "cgi" then it should be "/cgi-bin/CGINAME"; ie, it should be a 
>> relative URL not a full URL. This is particularly necessary with Xitami 2.4 
>> and the PerlSSI, which takes a fairly naive approach to resolving the 
>> path (stick the CGI directory name in front of CGINAME and hope for the 
>> best).
>
>Then if PerlSSI "blindly" sticks the CGI directory in front of
>the CGINAME, shouldn't it be "CGINAME" rather than "/cgi-bin/CGINAME"? 

No.  The "/cgi-bin" is stripped off, and then the CGI-BIN directory is
placed in front of what remains.  Looking at the way it is stripped off,
it _might_ work with a full URL, coincidentallly, if the URL names the
same site, but I'm not sure that's particularly portable.
