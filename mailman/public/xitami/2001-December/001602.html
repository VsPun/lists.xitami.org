In message <3C0CF1F9.4040101@freesbee.fr>, Francis Turner writes:
>Does gcc -DDEBUG make a performance difference to Xitami?

-DDEBUG enables the assertions in the code which check that things are
"as expected" when it is running.  (The assertion should never failure,
they're doublechecks that the program logic is correct.)

There will be a small perfomance overhead, but I wouldn't expect it to
make much of a difference except perhaps if your server is using every
last bit of CPU time, and begging for more.

Yes, -DDEBUG is enabled by default.  You can disable it you setting the
production build model, with: CCPRODLEVEL=production.  (That will also
strip the debugging symbols off the binary.)
