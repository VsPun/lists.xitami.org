>since Roadrunner blocks incoming port 80, I had to adust the portbase to
>55000 to use port 55080 with allows HTTP to work.  The problem is that I
>cannot get FTP to work at port 55021.
>So, I tried adjusting the FTP port varible to -54979 to get FTP port 21
>again.  But this does allow FTP to work either.  I also tried adjusting the
>data-port variable to -54800 to get the data port back down to 200.  

BTW, this data-port should probably be adjusted to be "20", ie, -54980.
(There was a typo in some versions of the configuration file that
wrongly listed the data port as 200.)

You should watch the Xitami start up messages and make sure that it is
starting things on the ports that you expect them to be started on.

Beyond that, FTP with the server behind a firewall (eg, your roadrunner
config) can be pretty tricky to get going, especially passive FTP when
the server is behind a firewall.  (Active FTP tends to work better when
the server is behind a firewall, and passive FTP tends to work better
when the client is behind a firewall -- if _both_ are behind firewalls,
you can end up with no FTP working at all, even if both firewalls are
trying to let FTP through :-( )
