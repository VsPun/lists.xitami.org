Interesting... Mine reads like this:

      Aliases Errors SSI MIMEs WSX Filters DDNS Advanced  Server Vhosts CGI
Security Logging FTP



The same screen says I have Xitami v2.5b1 even though I downloaded the b4.

You shouild tweak c:\xitami\defaults.cfg, based on c:\xitami\xitami.cfg.
Don't change c:\xitami\xitami.cfg

This is what I have in mine, for example:


[Cgi]
    Dirlist = "1"
    Form-query = "1"

[Filter]
    .php = "c:\\php\\php.exe"
    .php3 = "c:\\php\\php.exe"
    .pl = "c:\\perl\\bin\\perl.exe"

[Mime]
.php=application/x-httpd-php
.php3=application/x-httpd-php3
.pl=application/x-httpd-perl


[Ftp]
    Enabled = "0"

[Security]
    Filename = "defaults.aut"

[Server]
    Autostart = "1"
    Priority = "2"
    Recover = "1"

[Win32]
    16bit-cgi = "1"


    And this is what you can override in it about SSI: (note the exec=0 by
default)


#   ------------------------------------------------------------------------
-
#   SSI configuration section
#
#   This section controls Xitami's built-in server-side includes (SSI)
#   handling.

[Ssi]
#   Lets you enable/disable SSI handling.

    enabled=1

#   Specifies the format for times shown by SSI pages.

    timefmt=%A, %d-%b-%y %H:%M:%S %Z

#   Specifies whether size should be shown as bytes or as
#   abbreviations (Kb, Mg). Anything other than the string "bytes" is
#   taken to mean that sizes are shown as abbreviations.

    sizefmt=bytes

#   The timeout for CGI programs called from the SSI agent.

    timeout=30

#   Error message shown when an SSI command fails. The %s symbol is
#   replaced by the cause of the problem.

    errmsg=<p>[xitami SSI: %s]

#   If 1, allows CGIs to be executed using the #exec command. Since
#   this is a security weakness, this option is switched-off by
#   default.

    exec=0
