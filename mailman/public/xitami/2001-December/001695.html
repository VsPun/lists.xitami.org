In message <20011112060400.63871.qmail@web12502.mail.yahoo.com>, Jesse Foster writes:
>Does anyone know if it is possible to input a dynamic
>ip like <anything>.dyndns.org for an IP webmask which
>would allow access to the server?  

It's not.  It's a good idea, but it's not implemented that way at
present.  The way the webmasks work at present is that it converts the
IP address being considered into a dotted-quad (aaa.bbb.ccc.ddd) and
then compares it section by section.  This is to handle the wildcards.
