The following applies only if you want the /cgi-bin/ to appear in the
url of the path to your cgi-program: 

You have to name the cgi script exactly as you have set up the name of
your default page.

I.e. if default page is set as index.htm, then the script has to be
named index.htm.  

Xitami/Perl will not mind the file extension as long as 
1) you have the shebang line at beginning of script and 
2) the cgi-bin file-mixing is turned off. I think it's this one: 

[Cgi]
    Mixed-url=0

On the other hand, if your intention is to leave out the /cgi-bin/ from
the script's request path completely, then

1) you either set your root directory as your cgi-bin
2) or you make another alias to the full path of the cgi-program (named
again index.htm if you want it to look like the default page of a
directory).
