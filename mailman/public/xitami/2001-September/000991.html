>  when I test the VHost by typing in "www.mayanarch.com" I get the following result= C:/Xitami/wwww.mayanarch.com

With four "w"s? Anyway, that URI box isn't for testing vhosts. 

> Document root is set up as C:\Xitami\webpages\MayanArch\index.htm

That should probably point to a Folder, not a File. Remember, the
vHost will have many files. Point the vHost to a folder. There is
a different place to tell Xitami what file to look for if no
specific file is requested. 

> Can someone test this from outside??

http://www.mayanarch.com/ = Xitami default

http://www.mayanarch.com/mayanarch/ = "MayanArch WELCOME"

The files are there. But your vhost is not pointing to the right
directory. "./mayanarch" is where it should point to.
