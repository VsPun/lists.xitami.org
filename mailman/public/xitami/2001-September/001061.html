> 8 characters long and I could CD all day long

If that is how you want to spend all day . . . <G>

But seriously. It IS possible to navigate the NT command line with
long-names. 

Names with spaces need quotes, of course. 

And NT offers at least two command interpreters. "COMMAND" tries to be
similar to DOS; "CMD" has some new tricks. 

Either one should handle long-names.... but I know if I shell out from a
16-bit program I get a short-name 8.3 view of the filesystem. Invoking
Start, Run, "COMMAND", OK gives a longname world. Certain types of
shortcuts seem to force a SFN 8.3 world. 

That's why I was asking what editor you used and how you were getting to
a command line. Think about how you get there. Run "DIR" a lot because
it will show if you've dropped into 8.3-land.
