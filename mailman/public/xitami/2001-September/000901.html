In message <20010906175830.52975.qmail@web14910.mail.yahoo.com>, Shane Smith writes:
>aliases to my drives to get directory listings of
>them.  They all seem to work fine (floppy drive,
>cd-rom drives, and hard drives) except for the drive
>letter that Xitami is installed on!  

Are you sure that it's Xitami's protections which are preventing this,
not NT's protections?  Depending on what user you're running Xitami as,
it may not have permissions to access some of the root directory.

If you're certain that it's got OS-level permission to access those
directories, try turning on Xitami's server debugging ([server] debug=1)
and look to see what path it is trying to access when you request access
to that alias.  It may be that it's picking up the wrong directory path
for some reason.  (Check debug.log, after trying to access it with the
server debugging turned on.)
