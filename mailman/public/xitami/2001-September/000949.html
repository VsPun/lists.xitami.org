> Earlier this month, I sent a request for help installing PHP with

> 
> The server is running Win2000 and the latest retail release of 
> Xitami.  Is it a permission problem on Win2000 where I haven't set the
> directory/file permissions correctly?  Is it a problem w/ Xitami? 
> Since I purchased the retail version, is there a IMatix customer
> service line I call or a customer service email address I should

If you're running Xitami as a service, make sure it runs under an 
account that has access to write into the Xitami temporary 
directories.  To check whether it's a file permissions issue, try 
running the Xitami executable from a DOS window ('xiprosrv -d').  You 
can also try running the service under the Administrator user id 
(which should have access to all file systems).
