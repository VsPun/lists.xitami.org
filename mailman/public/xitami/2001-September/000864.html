In message <3B95209B.3553099B@relativedata.com>, David Patte writes:
>I am using PHP on one of my servers. I am noticing problems with a
>script that contains relative redirection using the php instruction:
>
>Header("Location: somepage.php");
>
>When I execute this command from my php script, i get a 404 error (page 
>not found)

Xitami doesn't understand as wide a range of server-side redirects as
some other webbrowser seem to understand; it understands only a full
absolute path (as a server side redirect), and a full URL (as a client
side redirect).

You can try:

Header("Location: /somepage.php")

if the page is in the root of that virtual host.  Otherwise XiFusion
provides a bridge between what many PHP scripts output and what Xitami
expects.
