Yes, thanks it was you Owen (new I'd remember if I saw the name). My further
investigations indicate that the true nature of the problem
is that php.exe get's loaded but can't load it's runtime dll (php4ts.dll).
Placing bith php.exe and php4ts.dll in Xiami's working directory allows the
dll to be found.

A more elegant solutions is to simply move php4ts.dll to windows system
directory (winnt\system32 in the case of NT), then it gets located & all
works well. It's a shame php doesn't report this in it's error log, it would
have made it a lot simpler to locate.

In any case, thanks once again for your help Owen, it saved me a lot of hair
tearing at the time.

Cheers, Paul
