BTW since we are on this portion of the code there is a fairly ugly
"lowercasing" bug in the ALIAS area which means that (IIRC) aliases on
case sensitive OSes (i.e. linux) have a problem with paths where the
path after the alias contains an upercase letter

E.g. if you have /alias as an alias of /usr/local/Alias is OK. But a
subdirectory (/alias/SubDirectory/ returns a 404 even if
/usr/local/Alias/SubDirectory exists

I fixed it once in one of the 2.5b releases but I stopped using Xitami
on Linux shortly thereafter and mislaid the code otherwise I'd provide
the patch. I seem to recall it required using a copy that is
lowercased for comparison and then inserting the original in the
actual path
