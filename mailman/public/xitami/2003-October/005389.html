Hello !

In windows, a service runs normally as "LOCAL SERVICE".
This default user has access to all local resources, like printers and
drives.
However, it has no access to network resources.

On your w2k server, locate the Xitami service
(start/programs/administrative tools/services), it's at the end.
DoubleClick on it, select the second tab ("log on"). There you have
probably "local system account" checked.
Click "this account" and enter the parameters for a valid user (with
password).
Restart the service, and it will work.
