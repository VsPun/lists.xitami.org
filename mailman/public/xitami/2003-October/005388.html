>Hello !
>
>I have a strange problem with NSLOOKUP on a win2k server.
>
>When i call nslookup from the command prompt, it works fine.
>It works also fine when inside a batch script (writing it's output to a 
>file or to the console screen).
>
>When i call nslookup from a program that is called by Xitami (cgi-bin 
>application), i always get the error "Default Server: UnKnown".
>Same sort of error with another utility that also performs lookups ("no 
>data"), so i think the problem is Xitami-related. (Xitami has the correct 
>NT rights)
>
>Is this a known error or is there a workaround?
>I want to do a MX lookup on a domain to determine if a e-mail address is
valid
>(are they other alternatives than nslookup for performing a MX lookup? 
>-could be a API call to some dll)

The environment variable "SystemRoot" has to exist (with valid information)
during program startup in order for socket calls to work under NT/2000/XP.
I discovered this oddity when developing the CGI Remote Module for the
ProtoNova Web Server.

This happens when programs are started from an NT service.  So, if you run
Xitami from the console or use the GUI version, your socket-based programs
should work just fine.

Hope this helps!
