I have two instances of Xitami running, one for production and the other for testing.  Both servers are running on the same system but on different IP addresses (on the same NIC).   However, the test server refuses to start on port 80...any other port it works OR if I comment out the "Portbase=0" the service will start up without an error but I am not able to get to the base page.

Each server is assigned to its own IP address, but both IP addresses reside on the same NIC of the server.  I don't recall having this problem in the past as I have done production and test versions of Xitami at the same time, each using different IP addresses that were on the same NIC.

In this case though, I actually need to turn the test server into a second production server and need it to run on port 80.

Heres the base config of each

/xitamiproduction/xiwinnt.exe
     defaults.cfg
          ip address: 192.168.1.10
          portbase=0
          service-name=Xitami Prodcution

/xitamitest/xiwinnt.exe
     defaults.cfg
          ip address: 192.168.1.11
          portbase=0
          service-name=Xitami Test

If I comment out portbase=0 on the test server the service will start up but nothing will respond to port 80 at http://192.168.1.11.  However, if I don't comment out portbase=0 the service won't even start up.  If I change the portbase to "portbase=2" then the service will start up and the pages are accessible on http://192.168.1.11:82 .  But I need this test server to be up and running on port 80.

Anyone have any ideas as to what is going on?
