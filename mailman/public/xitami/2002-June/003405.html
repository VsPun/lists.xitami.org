Just don't share PHPNuke with the world at this point.  Xitami can only
handle about 60 simultaneous users with XiFusion v2.5 (Windows version)
before strange things start happening (e.g. 0-1 bytes returned contained in
the Xitami logs).  XiFusion v2.6 doubles that number (handles about 120
simultaneous users before Xitami panics).  This problem was discovered
during a distributed attack test on one of my machines (I was attempting to
test to see how robust the various scripting languages are under rigorous
conditions).  This is a known DoS/DDoS issue with Xitami.

If you don't expect that type of consistent load, then there is nothing to
worry about unless someone has the intent of bringing the server down.  At
that point, you would have to start blocking IP addresses/blocks.

Other than that little note, I am glad that you got PHPNuke to work using
XiFusion v2.5.
