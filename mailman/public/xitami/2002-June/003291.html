In message <APEPIPMGPBGPNHMJEMBEIECNCGAA.tom@mcpharlin.com>, Tom McPharlin writes:
>Just upgraded to 2.5 beta and found two problems:
>1. Virtual hosts no longer work.

Don't set the webpages directory in default.cfg.  There's a bug in doing
so which causes that setting to override _all_ directory settings.  

>2. Alias to path with blanks no longer works (when included in quotes).

I'm not sure what might be causing this, as I'm not sure what changed in
the handling of aliases between the two versions.  Generally it's best
not to include spaces in your paths/directory names anyway as they cause
all sorts of weird problems.
