>Ok  We can load our windows based Xitami with what?
>
>1) Any free e-mail servers? Any with web based admin?

EzMTS.  <http://www.ezmts.org/>

>2) How well does the cold fusion express work from shining light prod?

ColdFusion Express is a product of (former) Allaire Corp.  Allaire sold out
to Macromedia Corp. (the people who produce Flash) a little while back.
Macromedia promptly raised the price-tag of ColdFusion by $1,000+ with very
few changes to the product.

>I am on a cold fusion enterprise server now.  What should I expect to
>lose if I transfer?

Since ColdFusion doesn't normally work with Xitami, I wrote a wrapper
program that runs as a filter CGI (XiFusion) to the CFML executable.  This
will cause a performance decrease in initialization time of the scripting
engine.  Other than that, you shouldn't lose a thing.

>3) I imagine that if I just download MySql and PHP that I can install it
>right and expect it to run on Xitami?

You have to set up Xitami, but that is a one-line configuration issue.

>4) What is Python anyway?  They also make a Free Windows port
>http://starship.python.net/crew/mhammond/
>5) How does Sun Chili!Soft ASP compare against INstantASP Xitami?  MSRP
>is $495 http://www.chiliasp.com/chiliasp/default.asp?target=windows

You seem to be stuck on scripting engines and not necessarily understanding
the separation of the web server from CGIs.  Xitami runs all scripting
engines as CGI applications.  As with any web server, you will get
different mileage with different scripting engines.
