Greetings,

All those errors were really bugging me and since the server is only a P166,
I wanted them gone.  Solved it by changing the portbase variable in the
config file.  This moved my HTTP and FTP ports from non standards (80 and
21) to my own choosing (+portbase).  I haven't been scanned since.  The
firewall is getting hit a lot though...

My domain registrar (Stargate) supports URL forwarding, along with port
number.  So I forward to http://xx.my.ip.no:newport/valid_dir/index.htm and
it works.  Well, at least one domain does, anyway.  I've set up 5 more that
have yet to come online (this is day 3).

The next step is to register one domain with a DDNS service (probably
DynDNS.ORG) and see if I can forward to a port through that domain.

The only problem I have is getting the resident DOS FTP utility to access
the new FTP port.  IE gets in and I imagine any decent GUI does too.  Just
haven't spent the time researching it.

Much luck,
Charles

PS What is sad is that the IP's that scan me reverse to clients on ATTBI,
which means these are mostly corrupted personal machines.  Or have the IPs
been spoofed?
