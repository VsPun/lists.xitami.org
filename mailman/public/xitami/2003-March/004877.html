I am running the windows version of Xitami 2.4d9 under Win95 for student 
projects.  I have set up a number of virtual hosts, and gotten them working 
OK.   I have defined separate .cfg files for each,  and inside each .cfg I 
have defined access and error logs:

[Accesslog]
     Filename = "C:/Xitami/webpages/team01/logs/team01.log"
     enabled=1

[Errorlog]
     Filename = "C:/Xitami/webpages/team01/logs/team01Err.log"
     enabled=1

I have not defined either of those in defaults.cfg.

On startup, Xitami creates the specified .log files, in each team's 
directory, but then never seems to load anything into them -- all access 
and error information goes into the root logs.  I have RTFM'd the online 
documentation, visited a couple of websites, and trawled the 15MB xitami 
listserv file in my text file reader.  No enlightenment.

Any suggestions?
