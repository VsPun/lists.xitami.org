> >Is there a way to view Xitami access logs in real-time? i.e. to see the
> >entries to the log files as they are written?
> 
> It seems there are lots of people wanting to see the logs real time.
> If pushed enough, i can write you a small windows console application (exe) 
> that shows the last entries in the log file.

Would it not suffice to use one of the Windows' ports of the Unix tail
utility?  I regularly use an OS/2 port of tail, with the -f switch, to
view similar logs that all other programs refuse to open (because they're
supposedly locked).  I've not tried it with running Xitami logs, but see
no reason why it should not be equally efficaceous.  (For those
unfamiliar with the concept, "tail -f <filename>" will display the last
ten lines of a file, and then sit waiting for the file to be extended by
whatever is writing it, and display what gets added "in real-time", until
Ctrl-C is pressed.)
