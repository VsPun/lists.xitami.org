Steve,

Don't forget the default.ida file request that has that gibberishly long
request.  I believe that is an attempt to stuff and overflow some buffer
(that appears fixed in latest Xitami).  Yeah!

Virtual Hosting maintains separate log files for each domain.  (Those
numerical archive things are bugging me though.).  "Access.log" now logs
only the base host.  Since the scanners only go at your WAN IP, it logs into
access.log and the world is a neater, cleaner place.

To be rid of these scans, I first tried to port forward port 80 to another
port on my router but what I wanted to filter just followed the forwarding
(duh!).

So I changed my base HTTP port to 1975.  This eliminates ALL scanners that
come in under my IP (search bots come in by domain).  At my DNS service
(ZoneEdit) I set up a ww2.thisdomain.com(or any extension) for each domain I
have and point it to my server.  In Web Forwarding, each main domain
(www.eachdomain.com and eachdomain.com) to http://ww2eachdomain.com:1975/.
Cloaking is not necessary since you don't really need to hide anything.

This really cleaned up everything.  If you VH, another entry for each ww2
domain must made.

What I want to know is how do we tell these people that their computer is
compromised.  Or retaliate.  The IPs I log appear to be simple users, some
even in my cluster.  Since most Broadband providers frown on serving, it's
not like I can report these incidents to my provider!

Still searching...

Charles
