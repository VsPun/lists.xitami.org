I'm using ssi with Xitami 2.5b5 on windows 2000, and it works fine here, 
without even installing the perl-stuff.

Some ideas:

Does the page testssi.ssi work on your machine? 
(http://localhost/testssi.ssi), if so, ssi is set up correct and working 
(ssi should be working by default).
If not, check whether it's enabled in your defaults.cfg, the correct 
settings are: (from the xitami.cfg file)

[Ssi]
     enabled=1
[Wsx]
     .shtm=xixssi
     .shtml=xixssi
     .ssi=xixssi

Is the path of the included file correct? Put the include in the same 
directory as blah.shtm, and use <!--#include
 > virtual="blah.htm"-->
