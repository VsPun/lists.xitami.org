Scott,

The 'php returns a blank page' comment is a dead give away - this is usually
because php.exe loads but is unable to load php4ts.dll. The most common solution
is to put php.exe and php4ts.dll into the xitami directory. This appears however
to be problematic with aliases. The solution for my problem was to place an
entry in the  system path that points to the php directory instead of moving the
php exe and dll. I also experimented with placing the dll in the winnt\system32
directory.

I think this will solve your questions 1 & 2.

If you wish to run Xitami with a particular account. Install Xitami as a
service. Then open the services control applet and open the service editor and
set the account name and password to run the service - this is standard windows
NT/2000 stuff.

Best of luck

Paul
