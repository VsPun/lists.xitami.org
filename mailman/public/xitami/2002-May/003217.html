Typically you can think of the Runtime Environment (JRE) as the context in
which the Virtual Machine is placed. 

Eg., Am I running in a browser or directly on top of an OS?  In other words
the JVM is a specific component of the JRE, it is also the most important
component, the part that executes the Java Bytecode.

For example the "JRE" for applets is typically the browser, while any os
level application is run in a JRE for that platform (Windows, Linux,
Solaris).
