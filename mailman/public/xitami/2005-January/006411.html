Just a quick question:  You did restart the server after those changes?

Dynamically shutting down a socket server in the middle of running the
application hosting a socket server is not an easy task in C/C++.
Restarting may be the only way Xitami removes such things.  Deciding to not
start a core piece of functionality and then dynamically starting it later
is also hard (but not AS hard as shutting it down).
