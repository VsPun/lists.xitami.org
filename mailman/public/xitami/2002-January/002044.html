>Bingo!

Not 'Bingo'...doing that can cause *serious* problems and possibly even
destroy a working OS (even Linux can't stand up to changes like that if not
done properly and will crash if such changes are made foolishly).

The correct way to get the odbc32.dll (and other related drivers) onto your
system is to go to <http://www.microsoft.com/odbc/> and download the latest
ODBC drivers from there.  Be sure to update your Jet drivers (which you
probably don't have installed since ODBC isn't installed) as well in case
you ever do anything with databases.

Only in rare (and I mean *REALLY* rare) circumstances is copying DLLs a
good idea.  Most DLLs and applications need to be registered with the OS
via the registry.  Copying files over doesn't add the needed entries to the
registry and if you attempt to use any of the ODBC functions in PHP, PHP
will probably crash or will return garbage (and if it crashes, the system
will probably become unstable).

I highly recommend installing ODBC correctly to help keep your system stable.

Hope this helps!
