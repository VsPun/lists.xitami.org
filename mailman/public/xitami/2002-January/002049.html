It appears that Xitami might not have permissions to access the file or
some sort of really bizarre firewall is preventing the data from going
through.  Now I'm not sure how Xitami does this, but it is possible to
attain file information without opening the file (verifies that it exists).
 So, the permissions might be set up such that Xitami can get the file
information (size, etc.) and return it but not be able to open and read the
file.

One other, however highly unlikely, possibility is a corrupt Xitami
installation (TCP/IP stack or OS installation?)
