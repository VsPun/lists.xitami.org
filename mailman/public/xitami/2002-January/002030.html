"Robert Bowen" <robert.bowen@mubimedia.com> wrote:

> I am trying to transfer an entire web site to cdrom. Within the site
> there are a few searches which basically open and parse data files,
> and then dynamically generate html pages, using perl...

It's impossible to run a 'live' site with CGIs and Perl without 
TCP/IP.  However, you can probably make a CD-ROM that everyone can 
use if you bend the rules a little.

First, create a start page that is entirely static.  Anyone with a 
browser, with or without TCP/IP, will be able to access this.  If you 
have dynamic output on the first page, cheat by pregenerating 
something that will show without HTTP access.  Then, make as much of 
the site work in static HTML and make it clear that the rest of the 
site is only available to people who have TCP/IP installed (which is 
frankly trivial on all Windows platforms from 95 upwards, even if it 
simply means installing dial-up networking).

You will also need to start Xitami in the background.  You can 
probably do this using the autorun.inf file.
