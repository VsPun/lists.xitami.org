Robert Bowen wrote:

>Hello all. Hoping someone out there can give me a hand.
>
>I am trying to transfer an entire web site to cdrom. Within the site there
>are a few searches which basically open and parse data files, and then
>dynamically generate html pages, using perl.
>
>Can I do this with Xitami? I know that most any web server these days is
>capable of cgi, including Xitami. But can it do it, having the web site,
>xitami server and perl, all living on a cd, without tcpip?
>
As others have said this isn't really a web server problem its more a 
browser problem. If you were willing to investigate the possibility of 
distributing the mozilla browser on your CD then you could look at 
including protozilla as a way to run a kind of "client side" CGI. 
However I suspect its easier to assume a TCPIP stack.

If interested take a look at http://protozilla.mozdev.org/ and an 
article at http://linuxjournal.com/article.php?sid=4724&mode=thread&order=0

Also bear in mind this is definitely beta code at present
