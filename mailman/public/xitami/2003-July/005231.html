Re: Xitami and ASP / Coldfusion compatibility

I don't know much about the ASP and Coldfusion needs.  The PHP suggestion
sounds good to me. Another alternative that I have been using is the
combination Xitami and REBOL via CGI. I use Dreamweaver but now only rely on
it for layout.There are some good examples on the REBOL site about how to
run REBOL as CGI handler.

http://www.rebol.com/docs/cgi1.html

http://www.rebol.com/docs/cgi2.html

The file of drugs that you look at might be a database but if there are just
a few thousand of them, you could hold them as a text file and let the CGI
script load it.As an example, I can read 100,000 records of 121 characters
each in 0.219 seconds.  For me, that is not much overhead unless your site
is getting dozens of hits per second.

If you are getting a lot of traffic there are some Xitami approaches to hold
the file in memory (using Xitami's Long Running Web Process LRWP) that work
with REBOL. REBOL gives you some flexibility in how you can manage your data
on a file if you chose to do that. There is also the ability for REBOL to
access mySQL databases if you want to do that.  REBOL sells commercial
versions that access other DBs. But you might want to start with the mySQL
solution ($0).
