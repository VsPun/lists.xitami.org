>>XP machine?  If
>>xitami crashes when Im away for several days I'd like to
>>be able to
>>either remotely start it or have something that catches
>>the crash and
>>automatically restarts it.  I have the setting selected

>>recover from crashes, but that seems to do nothing for

>
>Presuming you run Xitami as a service. Look in the Services
>MMC snap-in. For each service there is a Recovery tab,
>where you can tell the OS what to do when the service fails
>one way or another.
>In addition, there is the sc command-line tool that comes
>with XP and was part of the resource kits of the earlier
>NT-based Windows versions. The name comes from "service
>control", I believe. The tool lets you manage services
>remotely. However, since it works via Windows networking
>(as a general term) and probably RDP as well, it is not
>well suited for running it outside a firewall.
>And there are multiple remote desktop utilities, starting
>with Windows XP built-in remote desktop. There is a free
>remote-desktop client for earlier versions of Windows, like
>W2K - you'll just have to dig it out of Microsoft's site.
>A final not-so-elegant approach. Use pslist from the free
>pstools package at www.sysinternals.com. Schedule a job
>that runs every now and then and executes "pslist xitami".
>If the process is running, this will show information on
>it. If it is not running, pslist will say "process xitami
>was not found on <computer name>". Just parse the output
>and restart Xitami, if needs be.
