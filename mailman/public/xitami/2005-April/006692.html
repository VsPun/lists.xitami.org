On 27/04/2005 11:50 am, Maurilio Longo wrote:

> I've been able to build available sources (2.5c2) on OS/2 ...
> It works ok but cannot send pages bigger than 32KB, smaller pages get sent
> ok and even CGIs are called ok but if a cgi creates an output bigger than
> 32kb only the first 32kb are sent back to the client.

There is a 32k buffer defined in the code which does the file transfers 
(smttran), so that's probably involved.  It should transfer the file in 
chunks until it's finished.  Only transferring the first file... I assume 
you're not getting any errors in the console.log?  If you are up to changing 
the code, you can add traces (printf) to smttran.c at various points around 
lines 1102-1122 to see what's going on.

You might also want to try the 2.4 package - it's unlikely to behave 
differently but if it does, we may get a clue as to what's happening.

We don't have any OS/2 systems here (our last working OS/2 box died some years 
ago), so it's going to be rather hard to dig into this.
