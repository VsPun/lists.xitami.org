>So what's the advantage to Squirrel Mail?

The main advantage is that I can mail the original authors of Squirrel Mail
in the first place and get a same-day response from them (Luke and Nathan
Ehresman).  Granted they aren't the owners of that popular project any
more, but I know them personally.

Squirrel Mail was made to allow PHP to talk to IMAP and POP3 servers
without the need for add-on modules by only using PHP's TCP/IP sockets
implementation.  You simply download it and it works.  At least that was
the idea.  It expanded from there to add multi-lingual support, custom
themes, etc.  You know...the way open source naturally bloats itself
needlessly :)

>From what I gathered, the Squirrl Mail code base used to have a lot of
unnecessary Apache/PHP-specific dependencies.  Maybe the current team has
fixed some of those more recently so IIS and other servers can use it
without nearly as many issues.
