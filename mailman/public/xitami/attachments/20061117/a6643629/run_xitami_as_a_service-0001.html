<P><H3>How to use SRVANY to run Xitami as a service</H3></P>
<P>Xitami runs quite nicely as a Windows NT service, and, following these instructions, you should be able to get Xitami to survive an unattended reboot in well under a half an hour.<BR><BR>In order to run a Windows NT program as a service, you'll need a set of programs from Microsoft called the "Windows NT Resource Kit". This is generally acquired from Microsoft through the Microsoft Developer's Network (MSDN), but if you don't subscribe to that, you can download it from the Microsoft FTP server. The URL is<BR>
<BR>ftp://ftp.microsoft.com/bussys/winnt/winnt-public/reskit/nt35/i386/<BR>
<BR>There you'll find the Windows NT 3.5 Resource Kit, which will work great on NT 4.0.<BR><BR>Once you install it, the Windows NT Resource Kit gives you 3 files that you'll need in order to run Xitami as a service. They are SRVANY.EXE, SRVANY.WRI, and INSTSRV.EXE.&nbsp;<BR><BR>Once you have these files, and you know where they are, you're ready to begin.<BR></P>
<OL>
  <LI>First shutdown Xitami by right clicking on the tool tray icon, and selecting "Terminate."&nbsp;<BR><BR>
  <LI>Copy the 3 Resource Kit files (SRVANY.EXE, SRVANY.WRI, and INSTSRV.EXE) to the same directory as Xitami. I have mine installed at d:\xitami, so I will be using this directory in the examples to come. Change the examples to match the location of your installation.<BR><BR>
  <LI>Run a Dos Command Prompt and type the following command, using your directories instead of mine.<BR>
<BR>d:\xitami\instsrv XITAMI d:\xitami\srvany.exe<BR><BR>This command will install srvany as a service named XITAMI. All we need to do now is to configure srvany.<BR><BR>
  <LI>Close the DOS prompt, and run regedit.&nbsp;<BR><BR>WARNING: Using the registry editor can be dangerous, if you don't follow the directions EXACTLY. Your machine may stop running completely! No matter what happens, you make these changes at your own risk, so be careful!<BR>
<BR>Using regedit, navigate through the tree to <NOBR>"HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\XITAMI".</NOBR> Be extra careful now, this is where you start making changes. They're no big deal, just be on your toes.<BR>
<BR>Right click the "XITAMI" folder in the left half of the regedit window. Click "new" and then click "key" in the submenu that pops up. A new folder will appear with the name "New Key #1". It should be highlighted, so you can change it. Change it to "Parameters", and press enter.<BR><BR>
  <LI>The "Parameters" folder should have an "open" folder symbol next to it, making that the current folder. If not, click on the folder to make it current.<BR>
<BR>Right click in the blank area in the right side window, and then click "New" and then "String Value". You should see "New Value #1" appear. Change the name of it to "Application" (without the quotes). If you double click on the "Application" value you just created, an input box should appear. In the box, type<BR>
<BR>d:\xitami\xigui32.exe<BR><BR>and press enter. Make sure you used your directory structure and not mine.<BR><BR>
  <LI>Right click in the right side window again, click "New", "String Value", and rename it to "AppDirectory". Double click it, and type in the box&nbsp;<BR>
<BR>d:\xitami<BR><BR>and press enter. Make sure you used your directory structure and not mine.<BR><BR>Close the registry editor.<BR><BR>
  <LI>Open up the control panel (Start--&gt;Settings--&gt;Control Panel) and double click services.<BR>
<BR>In the window that pops up, scroll down until you see "XITAMI". Click it to highlight it, and then click the "Startup..." button.<BR>
<BR>In the window that comes up, make sure that the "Allow Service to Interact with Desktop" checkbox is checked. Press OK.&nbsp;<BR>
<BR>Press the "Start" button next to the list of services (not the one on the toolbar that brings up the "Start" menu!), and you should see the word "Started" between "XITAMI" and "Automatic". You should also see the tooltray icon appear.<BR><BR>If you do, you're set! The Xitami server is installed as a service and will start up automatically at boot time. You can verify this any time you want by bringing up the task manager, or looking at Xitami's log. 
  </LI></OL>
<BR>
<P>One caveat: If you don't log in really fast after a reboot, or if you log out and log in again, the tooltray icon may disappear, and you won't be able to bring up the Xitami control panel. Not to worry. Just open up the services control panel, highlight "XITAMI", click "Stop", and then "Start". The icon appears, and you're all set!<BR>
<BR>Lastly, if you would like to remove the service, use this command from a DOS prompt:<BR>
<BR>d:\xitami\instsrv.exe XITAMI remove<BR>
<BR>And that's it!<BR></P>
