<div>i will mention this again, only because i suspect that i may not have been clear&nbsp;in&nbsp;my explanation&nbsp;the first time around.</div>
<div>&nbsp;</div>
<div>i&#39;m assuming that that you guys have&nbsp;a number of&nbsp;virtual hosts, and would like each of them to create their own set of logs (<em>without additional suffixed numbers being applied to the file-names</em>).&nbsp; if so, you would probably want the config file of <strong>each</strong> virtual host to include these lines:</div>

<div>&nbsp;</div>
<div>
<p>[Accesslog]<br>&nbsp;&nbsp;&nbsp; Cycle-arg = &quot;acc-<strong>insert&nbsp;pertaining v-host name here</strong>-%yy%mm%dd.log&quot;<br>&nbsp;&nbsp;&nbsp; Filename = &quot;<strong><strong>insert&nbsp;pertaining v-host name here</strong>-</strong>access.log&quot;</p>

<p>[Errorlog]<br>&nbsp;&nbsp;&nbsp; Cycle-arg = &quot;err-<strong><strong>insert&nbsp;pertaining v-host name here</strong></strong>-%yy%mm%dd.log&quot;<br>&nbsp;&nbsp;&nbsp; Filename = &quot;<strong><strong>insert&nbsp;pertaining v-host name here</strong>-</strong>errors.log&quot;</p>

<p>&nbsp;</p>
<p>i strongly believe that doing so will indeed solve your problem, as long as you&#39;ve&nbsp;supplied a <strong>unique</strong> log-file naming-scheme for <strong>each</strong> of your virtual hosts.</p>
<p>this will prevent xitami from attempting to create multiple logs with the exact same name (<em>a set for each virtual host</em>)... which is precisely where those extra unwanted numbers are originating.</p>
<p>hope this helps&nbsp; :)</p>
