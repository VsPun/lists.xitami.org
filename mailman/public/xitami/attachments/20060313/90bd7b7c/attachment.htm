Very simple.<br><br>
You add 101 files, not 2000.<br><br>
Disable online administration and disallow the web admin interface for
your VHosts, as it would render Xitami wide open to security leaks having
web admin enabled.<br><br>
As such, you point all AUT files for your VHosts to one single disabled
admin file.<br><br>
Voila, 99 files out of 100 removed.<br><br>
As for the other 100 files, I find it extremely useful to have them in
separate files, thereby simplifying the management of documents.<br><br>
Besides, it is a very simple dead to write a CGI for your self, to run
off of the root on the server,<br>
to effectively write your own admin panel, which only you have access to
and is hidden from <br>
the publicly known view, by the documentation files of xitami.<br><br>
As such you can in fact turn off the web admin completely, disable the
LRWP from port 81, unless you use it for some ssl functions.<br>
You should already have disabled the ftp, which is controlled from the
default.cfg anyway, in a INI file style.<br><br>
All of which is really simple to manage.<br><br>
That's the way I do it at least.<br>
Pretty simple and 99% of my work is done by remote control anyway.<br>
In fact I have not had a monitor on any one of my servers for the past
three years.<br>
Except once, when installing a new machine 4 months ago.<br><br>
