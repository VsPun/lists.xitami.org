However doing a 404 redirect may prove to be no good in terms of SEO because
some search engines penalize for 404 redirects, especially Google. I found
doing a simple PHP redirect to work out better for me. I use it on all my
PHP CMS sites, use it as a sitemap and have a shorturl redirect site like
TinyURL. 

I call the script shorturl and it can be generated very easy with a PHP
script. It can take something like 

http://www.causewaylighting.com/index.php?topic=bulbs_&menu=bulbs_

and rewrite it to this

http://www.causewaylighting.com/surl/bulbs.php

Here is the PHP script that I use

<?php
$shorturl =
'http://www.causewaylighting.com/index.php?topic=bulbs_&menu=bulbs_';
header("Location: $shorturl");
?>

I also have a modified version that instead of creating a specific page name
it creates a directory with one file index.php so then the rewrite would be.

http://www.causewaylighting.com/bulbs
or 
http://www.causewaylighting.com/bulbs/index.php

Since most browsers if not all browsers assume index.php you can usually
leave it off.

Also some CMS's actually provide their own version of rewrite. I'm pretty
sure Geeklog does and maybe NukePHP.

Actually assuming the use of Xitami 2.5 with GSL working then the 404
rewrite is extremely possible because - assuming there is as
reasonable rule - then conversion of the old URL to the new one within
the error page is quite possible This allows you to say "ooops moved
here" and auto move it via a meta http redirect

GSL is a somewhat limited scripting langauge but entirely adequate for
this sort of job assuming the conversion rule is well known and easy
to use. You may even be able to figure out how t make Xitami generate
moved rather than 404 messages although I never figured that out.
