alright..
i will mention this again, only because i suspect that i may not have been
clear in my explanation the first time around.

i'm assuming that that you guys have a number of virtual hosts, and would
like each of them to create their own set of logs (*without additional
suffixed numbers being applied to the file-names*).  if so, you would
probably want the config file of *each* virtual host to include these lines:


[Accesslog]
    Cycle-arg = "acc-*insert pertaining v-host name here*-%yy%mm%dd.log"
    Filename = "*insert pertaining v-host name here-*access.log"

[Errorlog]
    Cycle-arg = "err-*insert pertaining v-host name here*-%yy%mm%dd.log"
    Filename = "*insert pertaining v-host name here-*errors.log"



i strongly believe that doing so will indeed solve your problem, as long as
you've supplied a *unique* log-file naming-scheme for *each* of your virtual
hosts.

this will prevent xitami from attempting to create multiple logs with the
exact same name (*a set for each virtual host*)... which is precisely where
those extra unwanted numbers are originating.

hope this helps  :)

rick
