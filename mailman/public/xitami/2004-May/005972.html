Have you tried chm files? (compiled html files).
This is a thing only Microsoft uses, for their online manuals. But since 
nobody is interested in the inner working of a Microsoft manual, there are 
no hackers to convert chm back to html (google for htm2chm, and then chm2htm)

Basically, all the files comprised in a site are compiled into one single 
chm file, but once loaded into internet explorer, the chm file works just 
like plain old htm files.
And what's still better: these files are also compressed! (compression 
ratio equivalent to gzip).
You have a faster response (all the links are in one single file), but if 
you want to change one single html file, you must recompile the whole thing.
Try http://web.verfaillie.com/cgi/e.htm, where you can find both versions 
of the same html file.

I think this is the solution to your problem!
