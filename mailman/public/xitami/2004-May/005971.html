Hi Lee,

Two other troubleshooting things you might try:

1) Create a php file with an intentional error in it.  Do you get an error 
message on the resulting web page?  That might help determine if Xitami is 
even firing up the php compiler.  To create an even worse error, rename 
php.exe to something else, and see what error you get.  If you don't get 
an error THERE, then you know the server is not being properly instructed 
on how to treat php files.

2) Create a php file (either pure php or php/html mix), put it in the same 
directory as your php.exe, and in a DOS prompt, type php.exe <filename>.  
This will run the php compiler outside the web server environment.  The 
php.exe should spit out the html code for your page, right there in the 
DOS prompt.  That will tell you if php is working properly, regardless of 
what server you use.

You also might want to run Patrick's three line php file at the DOS prompt 
too.  Send the output to an html file, and view that output on the 
server.  For example, save the following three lines as phpinfo.php, and 
put it in your PHP folder:

<?php
phpinfo();
?>

Then at the DOS prompt, run php.exe phpinfo.php > phpinfo.html.  And then 
view phpinfo.html with your web browser.  Compare that page with what you 
get when you look at phpinfo.php from the server.

Oh, and make sure you aren't using a .css file that turns your text to 
white! (just kidding)
