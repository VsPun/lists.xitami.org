> >         1. Is this the meaning for port 20 and 21? 
> 
> Yes: port 21 is used for control connections (of course, the originating
> FTP client can use anything it likes).  Port 20 is used, by an FTP
> server, for uploaded [i.e. PUT] data; in the full protocol, downloaded
> [i.e. GET] data is sent TO whichever port the client has negotiated with
> a PORT command, from port 20 on the server.
> 
> >         2. If so and port 20 is used by Xitami should it 
> >            be discussed in the documentation?
> 
> Why should it?  Port 20/21 is part and parcel of FTP, which was
> standardized in RFC-765 way back in June of 1980.
> 
> It's only things like firewalls and NATted routers which cause difficulty
> with this standard, and it is in that area that further documentation
> ought to be expected.
> 
> RFC-765 includes a rather useful bit of "ASCII Art":
> 
>                                             -------------
>                                             |/---------\|
>                                             ||   User  ||    --------
>                                             ||Interface|<--->| User |
>                                             |\----:----/|    --------
>                   ----------                |     V     |
>                   |/------\|  FTP Commands  |/---------\|
>                   ||Server|<---------------->|   User  ||
>                   ||  PI  ||   FTP Replies  ||    PI   ||
>                   |\--:---/|                |\----:----/|
>                   |   V    |                |     V     |
>       --------    |/------\|      Data      |/---------\|    --------
>       | File |<--->|Server|<---------------->|  User   |<--->| File |
>       |System|    || DTP  ||   Connection   ||   DTP   ||    |System|
>       --------    |\------/|                |\---------/|    --------
>                   ----------                -------------
> 
>                   Server-FTP                   User-FTP
> 
>       NOTES: 1. The data connection may be used in either direction.
>              2. The data connection need not exist all of the time.
> 
>                       Figure 1  Model for FTP Use
> 
> PI = Protocol Interpreter; DTP = Data Transfer Port.
