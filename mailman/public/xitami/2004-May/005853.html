:(  We don't have OS/2 to test on.  Further, the Boom scripts assume a certain 
syntax for the c shell which the old c.cmd does not handle correctly.

You should be able to build from the source file simply by doing this (and I'm 
testing it as I write):

 - unpack the source archive and cd into xiopen
 - copy the contents of sfl, gsl, smt, xiadmin, and studio into xiopen
 - remove lrstudio.c and smtschm.c, which are unfortunately-named Libero 
schemas
 - compile all the C programs: c *.c
 - place all objects into a library: c -r liball *.o
 - link Xitami: c -l xitami

This will work with the OS/2 c.cmd file and should also work from the Win32 
command line.
