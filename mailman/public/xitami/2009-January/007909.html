The console-version-only gets my vote too. One thing I would add if anything
would be a debug switch (maybe "xidos32 -debug" for instance?) that
summarises in real time what the hell it's doing! (obviously not meant for
production environments - but great for fault finding on a test bed).
"Opening up port 80 for this, 81 for this, couldn't open 81 - already in
use, 81.81.81.81 requested / - return code 200, 81.81.81.82 requested
/something.htm - return code 404, etc"

The last time I ever used the GUI was under Windows 3.1! Never really looked
at it since. Come to think of it, it was around that time that I last used
the web-based interface too! (I just don't like admin panels diddling around
with carefully constructed config files).

The only other thing I would change would be making it easier (or better
documented?) to kill the /admin page (I ended up just #'ing out the xiadmin
WSX agent line - better ways of doing this?). But I really am just
nit-picking now..
