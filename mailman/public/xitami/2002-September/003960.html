>Another question, in the above access.log, why is there a 304 for every HTTP request?
>Is there a configuration parameter that causes this behavior?

304 means "not modified". if your browser has the most recent copy of
the graphics and webpages in the cache, xitami will return an 304
instead of the requested data.

you cannot switch off xitamis caching behaviour (only for the files
configured with "default1=..." to disable caching of your default files
like /index.html"

to disable caching try smthttpl.c line 2139 :

if (p_http-> default_page
  && *HCONFIG ("server:cache-defaults") == '0')
   return;
