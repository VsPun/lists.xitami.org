> ******************* Lesson Learned! ********************

NEVER, NEVER TEST Perl CGI SCRIPTS WITHOUT:
 1) use CGI::Carp qw(fatalsToBrowser);
 2) running with the " -w" option
 3) use Strict;

I am not sure if " -w" (warn) can be forced in the shebang line. It
may need to be set globally in Xitami's Filters setting. Even there,
it may not propagate in all versions of O/S, Perl, and Xitami. So
run the script with -w from a command-line before you finish
testing. While there may be extra errors because it isn't called
with all CGI parameters on hand, you should be able to account for
them. Turning off -w "usually" means there is something wrong with
the script, and any apparent success is dumb luck.
