If you plan to put 4 billion files on one drive, you should first check 
it's capacity ;)

Besides, if you want to provide services to such a huge number of 
customers, you should use other systems.

Some questions that needs adressing:
How will the user registration be done?
Directing one user to it's directory?
Checking it's used disk space?
You need a very good scripting language, at least ISAPI or fastCGI

What is the purpose of the whole thing?
Some sort of file repository (unfrequent access) of picture sharing (bursty 
activity after return from vacation) ???

Suppose you have 10.000 users, with maximum 100 users online at any moment 
(the maximal load to ensure fast response).
Each user has 10 directories containing 100 files, total 1000 files/user
Each file is 250.000 bytes in size (my camera creates 2MB images)
Each user will need 250.000.000 bytes, or 250MB.
Your 10.000 users will need 2.500GB of storage.

You are in need of a very professional package with clustered servers, NAS, 
redundancy, load balancing, ...
Xitami (and to a lesser extend Windows) is not able to provide this.
