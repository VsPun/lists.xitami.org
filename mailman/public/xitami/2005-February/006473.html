Hello Alex,

you need to create 2 files. You can do so  through the GUI or just edit the file directly. The 
files are ftdirs.aut  and ftpusers.aut
In the file ftpdirs.aut you name the directories you want to be accessed by ftp or browser from the 
users you define . 
Here an example entry
    [../webpages/dirname]
        username = "*"

    [dirname]
        username = "P"
         anonymus = "G"
Between the " " you have to define the access right. Make sure you have a "space" between = and "
For anyone to download you can use anonymus = "G" as username and user right for this particular 
directory.

For the second file ftpusers.aut  it would look as follows. In this file you define the users right 
to connect, the root (dir), password...
Here the sample
    [username]
        Access = "P"
        Aliases = "0"
        Hard-quota = ""
        Password = "whatever-you-want"
        Pipe = ""
        Root = "dirname"
        Soft-quota = ""
        Use-quotas = "0"

For your example you would have to create 2 entries for the same root; one entry for the user
( username) and one entry for the visitors (anonymus) who should be able to ONLY download, as I 
understood correct.
For the user  Access = "P"
For the visitor Access = "G"
Both entries would have the same root = "dirname"

The directories which you want to create don't have to be in  \ftproot . I have, as example, set 
\webpages with its directories for the virtual domains to access by ftp only for myself. 
If I'm correct, xitami finds the way to the correct directory mainly through the username / 
directory entries in ftpdirs.aut

I hope this description is clear enough.If not, let me know and I will try to explain it better.
