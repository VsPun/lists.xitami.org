Hi:

As some of you may have noticed, I placed an "request for help" in 
this forum for help with SSI.  I did obtain some help from one person 
but it did not resolve my problem.

I have reviewed several SSI tutorials from the net but although the 
set command is not described very often, from the few that I found 
that described its use, it would appear that I am using it correctly.  I 
am wondering if Xitami's implementation of SSI is either deficient or 
has a bug in it.

If I setup a link as follows:

template.shtml?bodytext1.txt

I can display the text after the question mark with the following:

<!--#echo var="QUERY_STRING" -->

However, I want the QUERY_STRING text, in this case a filename, to 
be the virtual filename as follows:

<!--#include virtual="QUERY_STRING" -->

but I get nothing.  I have changed virtual to file but to no avail.  I 
have used both absolute and relative paths and again, no success.

When my instructor for my webmaster course showed us this trick, 
he used a slightly different procedure whereby he "set" the contents 
of a variable to be the QUERY_STRING as follows:

<!--#set var="bodycontentfile" value="$QUERY_STRING" -->

but when I try this using Xitami, I get an error that states:

[xitami SSI: Can't open ssi file 
O:/XITAMI/webpages/cuac/public_html/$QUERY_STRING]

Any suggestions?
