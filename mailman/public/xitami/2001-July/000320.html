In message <3B42CFD9.11867.2173B68@localhost>, russ@iron.dynip.com writes:
>How does  one block a person from gettin on the web server?  is 
>there a way to  deny by ip in the defaults.aut?

You can set up a webmask list which denies that one address and allows
everything else _and_ allows access without a password.  But it may
block more than one person (eg, if they're using their ISPs proxy
cache), and may not block them completely (eg, if they have more than
one account).

If you need to explicitly block someone from a site that often means you
have to require everyone to log in, and then ban the person you don't
want.
