In message <3B3D082E.30131.7D1FBF6@localhost>, "Dennis G. Wicks" writes:
>Running V2.5b5 on NT 4.0 my browsers don't reload the page when 
>it has changed. I use IE 5.5 and Netscape 4.7. I have to force the 
>refresh or reload before I can see the changed page. (This is ctl-
>refresh in IE or shift-reload in Netscape.)

This is a browser feature: when browsers have something cached, they
send get-if-modified requests to see if they need a new copy of it,
which Xitami answers correctly.  However by default most browsers are
configured to check "once per session" -- which saves time and network
traffic, but can mean that you don't see the new page until you force a
reload if it changes during your session.

You can either change your browser setting, or force a reload -- or
force a particular page to not be cached (as others have described).

The only Xitami-related bit is that Xitami can be told to cache
index.html type defaults (to save opening/closing them all the time);
[server] cache-defaults=[0/1] controls this.
