>From the symptoms that you describe, it does sound like that is the

For example the images located in ./webpages/images do not load
>if I type http://localhost/images/image_1.gif, but they do if I type
>http://localhost//images/image_1.gif. On the other hand
>./webpages/xitami/images works fine and
>http://localhost/xitami/images/image_1.gif displays as desired. 

This is rather puzzling.  It's almost as if the path definitions are
wrong.  You might want to turn on server debugging mode ([Server] debug=1)
and see what path locations it is trying with each request.  And you
might also want to compare the configuration file in the Windows version
(or source package) with the ones in the BeOS one, just in case they are
wrong for some reason.

>Another serious issue for me is that when Xitami is started, cpu load for my
>amd duron 750 goes up to 100% and stays there until I stop the server. I
>have no idea what's going wrong. 

It sounds like it's getting stuck in a loop for some reason.  Normally
Xitami should "go to sleep" waiting for the OS to tell it that someone
has sent it a request to deal with.  For some reason this doesn't seem
to be happening for you.

Has anyone else found this with the BeOS version?  (I don't have access
to a BeOS version to test.)

If there's anyone who's built Xitami from source under BeOS, they could
perhaps turn on the tracing functions and try to figure out where it is
looping.
