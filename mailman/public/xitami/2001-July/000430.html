Don't forget Xitami has an inbuilt feature to enable simple testing out for error messages - copied from the documentation it is:-

To test a specific error, you can use the special URL 'error?xxx'. For instance, to simulate a 402 error (Payment Required), use a URL like this: 

'http://127.0.0.1/error?402'. 

You can simulate a 302 error (temporary redirection) using a URL like this: 'http://127.0.0.1/error?302:/somedir/somepage.htm'.

Works for me on Win98se and with IE4 and Netscape and I get my custom error messages with header and footer complete with graphics.
