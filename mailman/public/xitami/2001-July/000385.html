>By default xitami is running on Http -80 and ftp -21.
>Now if i change Http to any other port like 1080 and
>ftp will become-1021.Now when i say ftp from dos
>prompt.Then it say connection refused.

Currently the only way to change the HTTP port is to change the port
base, which shifts _everything_ up by that base amount, including FTP.

Some FTP clients let you specify which port number to connect to, in
this case 1021, eg:

ftp HOSTNAME PORT

which may let you work around that.

Alternatively as a work around you can set the FTP port in the
configuration file to a sufficient negative number to offset the amount
it is shifted, and get you back to the standard port 21, eg, (21-1000) =
-979.

This is on our bug list, and eventually there'll be a way to change just
the HTTP port number.
