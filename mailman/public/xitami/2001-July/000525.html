At 10:12 AM 7/26/01 -0600, you wrote:
>something I've been wondering for a while.... Sometimes when I request a
>page that is on my //localhost, the browser takes several (2-10) seconds
>to get a reply, but other times it is really speedy.
>
>Is there something in my configuration of xitami or networking control
>panel that I can check to help my machine find localhost quickly all the
>time? My internet connection is thru @home which has no gateway defined,
>no DNS (they use DCHP enabled), dynamic IP addresses and the like.
>additionally, I just added a dlink 713p wireless router/firewall so I
>can share my connection between two home computers and have home file
>sharing.

Are you typing '//localhost' into the browser?  If so, then you should note
that it might be looking for a machine called 'localhost' using the NetBIOS
protocol (hence the really long delays).  Try the fully qualified
'http://localhost/' URL instead.  Also try 'http://127.0.0.1/' as well.

Also, I don't see how you get out onto other servers if you don't have a
gateway defined.  I'll bet a gateway shows up if you run 'winipcfg' and
select your network card.  I would even bet you have DNS as well.  DHCP is
just a protocol to set up gateway, IP address, DNS server list, WINS, etc.
information.  DHCP just makes all of that transparent to you, but it can
come in quite handy at times to know that information.
