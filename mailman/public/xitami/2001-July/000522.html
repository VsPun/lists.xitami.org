In message <200107271425590.SM00258@default>, "Julian Rickards" writes:
>Although no one has yet answered my question, "Is Xitami's SSI 
>implimentation buggy or incomplete?", I have determined that, for 
>whatever reason, Xitami won't or can't impliment SSI's #set command 
>but the same code works under Apache 1.3.20.

AFAIK the #set command is an Apache extension; if you require that
Apache extension then I guess you'll have to use Apache :-)

There has been some discussion about extending the Xitami SSI handler
(which is based on the NCSA include facility; Apache was originally
based on the NCSA code too), in particular extending it to support some
of the common Apache and other extensions, but this has not happened.

As always we welcome someone taking the time to write the code to handle
the extensions.
