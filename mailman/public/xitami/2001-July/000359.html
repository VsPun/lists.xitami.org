In message <000f01c0fb14$b2488a30$1600a8c0@docs3a>, "KD4SFD" writes:
>I have to frequently change from a max of 3 users to 20 users (under W98SE
>xitami ver 2.5b.)  I limit to 3 when I am home as to not slow up my
>connection, then the maximum users when I'm not around.  I find that even
>after save, halt, and then start (or restart) of the server, the previous
>max connections, either 3 or blank, is still there.  If I reboot, then the
>new settings are used.  

It sounds like Xitami isn't picking up the new settings for some reason;
some of the settings are only checked fairly early in the startup phase,
and it's possible that is one of them.

Have you tried exiting from Xitami (ie, not just stopping it in the
control panel, but actually quitting the application) and restarting it?
Did that make a difference?  (Other than forcing Xitami to start from
scratch I can't see what a reboot would do differently.)

If we can narrow down what is causing this problem we can put it onto
our bug list.
