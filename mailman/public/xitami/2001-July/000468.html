>   a program I can run on my win2k machine without being connected to the internet and for testing of certain e-mail functions in my websites which I am testing ...  

OH!

You don't need email? You need something to catch what a script sends to
sendmail? 

Create a plain text file, named SENDMAIL.BAT , and put this line in it:

 more >> C:\sentmail.txt

Save SENDMAIL.BAT "someplace". Where? Since most perl scripts using
sendmail assume it is some place like /usr/bin , you could try making a
folder of that name on the same drive as Xitami, and put it there. OR,
put it some place you like and edit your perl scripts to point to it.
Whichever is easier. 

For initial testing, find or make a small text file and copy it to the
same folder as SENDMAIL.BAT. Let's say the text file is called TEST.TXT. 
Open a DOS window and CD to that folder. The command 

  SENDMAIL < TEST.TXT

should run without complaint. Now look for the file C:\sentmail.txt and
open it. Should contain the words in TEST.TXT. Each time you run
SENDMAIL, the text sent through it will be appended to the end of
C:\SENTMAIL.TXT

Then try your scripts. 

I *think* that will emulate enough of a real sendmail to do what you
want.
