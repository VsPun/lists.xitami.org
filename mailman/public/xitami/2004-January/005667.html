  Chestnut,

  Entering your external IP WILL access your web site, even when typed =
at the server (just tested it).  Your access.log file will show your =
gateway address (in Steve's case 192.168.1.1) instead of the usual =
external access IP.  Sometimes, in my instance anyway, you have to enter =
the "http://" along with the IP.  But ... if virtual hosting is active =
and there's nothing at base host, then you get nothing.

  I don't understand why it doesn't work the same as my operation.  For =
the sake of comparison:

  Router is a Linksys BEFW11S4 wireless 4-port (first version, nearly 2 =
years old)
  Router's address is changed to 10.168.1.1 and is also the gateway
  Subnet mask is 255.255.248.0 (I don't know why)
  DHCP services are on with addresses starting at 10.168.1.100
  DMZ Host is set to 10.168.1.50 (notice outside DHCP territory but =
shouldn't matter)
      (*I have tried Port Forward and that works slick too!)

  Computer is a hand-me-down P166 that I just upped the mem to 256MB; =
3.25GB HD.
  Running a Xitami 2.5c1 console (xidos.exe) in a DOS windows on Windows =
98SE (does service work?)
  HTTP port is changed to 1975 and FTP is disabled
  Virtual Host is active (and actually working) but base host is blank.
  Server is manually configure to 10.168.1.50
  Gateway is entered as 10.168.1.1 along with subnet mask 255.255.255.0
  DNS IPs are also entered manually and can be found on the router's =
Status page
  Other servers installed: Fictional Daemon (FTP and Telnet) and Mercury =
Mail
  Also on board: MySQL and PHP

  I think that's all.  Hope this is useful.
