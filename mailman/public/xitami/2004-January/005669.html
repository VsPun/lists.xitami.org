Didn't know it scanned.  When I ran the FTP service, it was shifted to port
1916.  Maybe directly setting the port might help.  I had download issues
with the FTP service (premature completeness) so switched to another.

[cut another faux pas]

I did find this in Xitami.cfg (for use in defaults.cfg):

#   Specifies the port at which data connections will be made. Xitami
#   scans for free ports, so this is simply the start of a range of
#   ports. The port is shifted by whatever value was used for the
#   server portbase.

[ftp]
    data-port=200

Whatever that means.  It may either allow you to shift the starting scan
point or set the starting scan point (say at 20) and it might stop there.

Don't know.  Worth a shot.
