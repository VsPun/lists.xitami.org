	This question comes up again and again, when in fact the PHP "InstallShield"
will automatically configure Xitami for PHP.

	So, I am not even sure why people try to manually configure it. Maybe because
they try to get the PHP zip instead of the "InstallShield" ?

	Frankly, all you need to do is keep clicking the Dialog for the PHP install.
The only gotcha now with the latest PHP is that you must edit php.ini and
re-enable PHP for standard CGI, they disabled that in the pre-configured files
for security reasons.
