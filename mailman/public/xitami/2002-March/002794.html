That's the case, webmail runs on diferent port.
To simplify things for the user i just added a link on main site, like this
they don't have to type ":port".

I know this is not xitami related, but with webmail you can also run WAP,
for mail, and even build a site on it. Just add /wap/ after the port number.
Ex.: Web site -> www.mysite.com
       Webmail -> www.mysite.com:4096
       WAP      -> www.mysite.com:4096/wap/
I would like to have Wap with xitami. Like this i couls leave my mail server
just for mail.
