> I'm just curious to see what type of machine everyone is running
> xitami on.  I want to get an old system just to host a website, but
> want to be sure it will handle it.  I'm thinking about a pentium 233
> with 128 MB RAM.

You're not telling us what type of load you expect, if you want to 
run PHP as a CGI module, etc. This is very important information.

>  Let me know what processor, RAM, hard drive, ...etc you have. 

200MHz MMX, 96 MB RAM, cheap-o Fujitsu IDE drive, Tyan Titan Turbo 
motherboard, running NT 4 Workstation with Xitami webserver and 
Mercury/32 mailserver. Xitami and Mercury/32 takes up VERY little 
RAM. Right now, with an uptime of ten days, the total system uses 38 
MB of physical RAM.
PHP (CGI mode) is kind of slow. Compare my personal site
http://mw.9000.org/
(which uses PHP) with a HTML-only site
http://www.brandkonsulten.se/
Those two are hosted at the same machine. Initial lag for PHP parsing 
is quite noticeable.

> Also, how
> many hits per day does it handle without slowing down, crashing, etc.

This I haven't tested. It never crashes. I have had months/years of 
uptime depending on the mail server and me installing stuff. Then, 
the websites don't see very much use, only the mail server. But I 
don't think Xitami has ever crashed -- if you avoid the beta version, 
that is...
