On 4 Mar 2002, at 6:57, Stan Schretter wrote:

> Is there a way to shutdown Xitami from within a Visual Basic Program

The answer is yes and yes.  But only one is relevent to this list.

Advanced (and off-topic for this list).  Use C++ or Delphi.  If you 
insist on VB, you can use API calls to identify and kill any running 
shell32 process.  If you are not familiar with api, this will ruin any 
social life you had hopes for and may require 2 gallons of Mountain 
Dew.

Much more simple (and slightly more relevent to this list) just 
minimize Xitami without sending it to the systray.  It's easy when
using Xitami to overlook this ability.  Now you can recognize and 
kill Xitami just like any other app on the taskbar using Sendkeys 
(or a few other standardized methods) and using only a couple 
lines of code.
