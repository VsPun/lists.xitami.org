RO> I got this to work but there are a few catches:
RO> 1.  cgi.force_redirected = 0   ;;; make certain the ";" is removed from the
RO> front of the line..
RO> 2.  All php files must go into the *./webpages directories for each of your
RO> domains
RO> 3.  Pros/Cons of using the "doc_root" variable:
RO>     CONS:
RO>         1. #2 won't apply if you define it - ALL php files for all domains
RO> must go into that directory, thus your files start to get jumbled together
RO> and files for virtual domain "blah.com" will sit in the same directory for
RO> another virtual domain "blah1.com".  There could possibly be a way around
RO> this if you did something along the following (I haven't tested it though):
RO>             PHP.INI: doc_root = /xitami/php
RO>             Xitami blah1.cfg: [Alias Section] *.php = /xitami/php/blah1/
RO>             Xitami blah2.cfg: [Alias Section] *.php = /xitami/php/blah2/

RO>             You would have to manually created each virtual domain directory
RO> reference in the /xitami/php directory - you would then also have to force
RO> FTP paths so those domain admins could upload php files for their domain,
RO> ONLY to their directory.  I think this might work.  You definently have to
RO> jump through hoops though.
