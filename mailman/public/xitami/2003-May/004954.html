> I've been away from Xitami for awhile in a production environment, but

> one machine.  We've now converted all our servers from Linux to OS X

If it will help, I sent the output to a text file which is pasted 
below.  I'm experimenting with the build on a Titanium Powerbook 
running OS X 10.2.5 Jaguar, compiling from a tcsh shell.  I'm running 
xibuild under my user login, but su root makes no difference - I get 
the same output.  As the output notes;

"If the ANSI C compile command is wrong, you will have to modify the
'c' scripts in these directories:
     /Users/chris/desktop/xitami-24d9/src/sfl, 
/Users/chris/desktop/xitami-24d9/src/smt"

however, the gcc C compiler in Darwin/OS X is the same one used in 
Linux or BSD, so I would assume nothing has to be changed in that 
respect.
