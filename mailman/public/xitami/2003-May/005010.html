> 
> I know that OS/2 can do it, but only on the newer 32 bit TCP/IP 
stacks. 
> It is undocumented, but you can use aliases to configure it. With:
> 
> ifconfig lan0 192.168.0.100 netmask 255.255.255.0 metric 1 mtu 
1500
> REM *** assign IP address to same NIC
> ifconfig lan0 alias 192.168.0.200
> 
> It works. You can tell Xitami to use virtual hosts based on IP 
address, 
> so you can have multiple domains running with it (that's how I've 
been 
> using it) - all on OS/2 though.
