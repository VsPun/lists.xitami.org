	The first thing I would do is to sniff the network packets to see who is not
telling the truth - Xitami or the web browser. One sniffer that I use is
http://www.analogx.com/contents/download/network/pmon.htm , but unfortunately it
does not seem to be able to sniff packets on the loopback interface (127.0.0.1),
only on a real network card. So, you may have to test with Xitami on a separate
machine. Or easier - simply telnet to port 80, and issue a GET command and see
what Xitami answers.

	You can also install PWS (Personal Web Server) or another free web server and
test the same page, just to compare.

	Also, make sure you are not running a pop-up killer. These programs sniff the
HTTP replies and change the HTML on-the-fly to kill pop-ups.
