>One suggestion was to add "127.0.0.1 www.thunting.com" to my hosts file.
>This does make the off-line debugging work, but it also forces the browser
>to use my local scripts even when on-line, which is what I expected. In
>other words, it hard-wires www.thunting.com to 127.0.0.1. I do not want
>this, and did not have it on the old system.

Hello !

I don't think you could make Windows that intelligent.
When resolving a name, windows first looks in the hosts file for hard coded 
addresses, then try a dns and/or wins query (depending on how your system 
is configured). I don't think you could reverse the search order, that is 
look in the host file when there is no network present. (even if windows is 
able to "sense" the presence of a network connection).
