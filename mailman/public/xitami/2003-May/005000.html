Hi - I don't know if this is strictly a Xitami question or not, but here it
is

I installed Xitami at a location for use as a CGI server.
I have a different directory security password for my App directory and a
different password challenge for the cgi-bin dir, so my .AUT file looks
something like this:

[App]
   Reguser=password1

[cgi-bin]
   Advuser=CGIpassword


Everything works as expected for users who access the web server from OTHER
machines.  But the main user (who sits at the machine where Xitami is
installed) gets javascript and XSLT errors despite successfully negotiating
the password challenge dialog boxes for both the main App default web page
and some interactive CGI operations.

So it appears that on the computer where Xitami is running the biggest web
application user can use the default web page without problems but pages
that reference JavaScript files can't seem to load the JavaScript;  likewise
the CGI operations that return and embedded HREF to an XSL stylesheet cannot
load the stylesheet:  the message is  Access Denied

Default web page, Javascript files and XSLT files are all in
%XITAMI%\webpages\App directory

On the other hand, all other users who use my CGI web app seem to experience
no such problems at all?

Does this ring a bell for anyone?

OS is Win2000;  the browser is IE6   Xitami ver 2.4d9

Thanks

- steve
