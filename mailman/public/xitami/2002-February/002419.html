I read the article Andreaz and as a systems programmer for many years I have
written many task schedulers.

It is a common misnomer that bad processes use more CPU time and that they can
damage a system. Let me set this straight...
this is total rubbish ! Any process that performs a lot of work is capable of
using 100% cpu time (even a simple calculation loop). The
real measure of an operating is how fairly is shares time amongst two CPU
intensive tasks (windows is not very good (read terrible)
in this regard.

In any case the only time the CPU time used by Xiami in turbo mode would be a
problem would be if you were using the same PC to
perform other tasks other than serving web pages. If it is a dedicated web
server run it in turbo mode and get the best performance
BECAUSE it will use as much available CPU time as it can.

Regards, Paul
