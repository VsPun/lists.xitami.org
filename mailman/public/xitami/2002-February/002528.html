Hi Russel,

I seem to recall having a similar problem some time ago when I was evaluating
Apache and Xitami on a server that also had IIS installed. You must ensure that
both IIS and the IIS Admin service are stopped.

My problem turned out to be that another process was already using one of the
ports required by my Xitami configuration. Try using "netstat -a -n" and
'netstat -a' to see if something else is already using port 80 etc.

Paul

  Check the xitami logs and see if there is an error in there.

on a machine that runs for long periods, that changes made last week don't occur
till this week when you reboot.

  The vitamin log should tell you if there is an xitami start problem and what
it is.
