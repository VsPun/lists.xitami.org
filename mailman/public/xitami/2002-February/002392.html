Hi there:

I am hoping someone on the list can shed some light on a problem I'm having
accessing my server from the Internet.

I have a network running at home (Win98SE, ICS, and Xitami), and am able to
get an HTTP response from Xitami on Port 9080 from within the network but
not from outside , i.e. the Internet. Here's the setup:

- Cable Internet connection. The cable provider has confirmed that it's
blocking Port 80.  Hence my server is running on Port 9080.

- The cable modem is connected to a Compaq 200 (w/164 MB RAM) running Win
98(SE), Microsoft ICS, and Xitami Ver. 2.4d9.

- This Compaq is connected to an 8-port hub which in turn provides Internet
access via ICS to 5 other machines on the home network.  This is also the
machine on which Xitami (Ver. 2.4d9) is installed (http on Port 9080).

All machines including the server can access the Internet and each other.
All machines including the server can successfully access Port 9080 from
within the network using all methods (Cable provided IP address,
192.168.0.1, 127.0.0.1, and computer name).  But I can't get Xitami to
respond from outside the network.  I assumed ZoneAlarm was causing this.  I
uninstalled it, but it didn't make a differnce.

Xitami is installed with the defaults, except that http is on port 9080, and
FTP on port 9121.  No other changes.

Can anyone offer any suggestions, please?

Thanks for taking the time to read this.

Regards,

Viraf
