>I just contacted RoadRunner and they said, no they don't block port 80, =
but residential accounts don't permit web servers. >So, don't know if =
I'm getting the straight answer or not. Can't tell if I should beleive =
them or not.



I am also on Road Runner (Central Florida).  They told me they block =
port 80 nationwide.  I was never able to get port 80 to work.  =
Fortunately the tech I spoke with also runs a server from his home, and =
told me it could be done which made me persistant enough to find =
available ports.

I use dns2go ( www.dns2go.com - its free) handle my dynamic ip adress.  =
In the dns2go client, under the Service Settings tab, I checked =
"Redirect web requests (with a www prefix)" and picked "To my current IP =
address ont port:" and set it to 6080.

Then I edited my defaults.cfg file in my xitami directory to say:

#  Created at installation time
#
!  Loading defaults.cfg file...
[Server]
priority=3D1
portbase=3D6000
background=3D1
[FTP]
port=3D-5979
dataport=3D5800
user-file=3Dftpusers.aut
directory-file=3Dftpdirs.aut
[Security]
filename=3Ddefaults.aut

[Filter]
-php=3Dc:\PHP\php.exe

The stuff that matters is:

[Server]
portbase=3D6000  This sets the web server port to 6080 which is where =
dns2go directs it.

[FTP]
port=3D-5979    This brings the FTP server port back down to 21 (make =
sure to use -5979) as portbase is an offset (+80) for all ports
data-port=3D5800  this is actually a mistake, but it works for now.  I =
shoulda used -5800 cause my intent was to bring the FTP data port down =
to 200.  I have issues where some people have trouble connecting with =
FTP client software, but everyone can connect and log-in with IE, so I =
leave it alone out of laziness.  I found out later that xitami being set =
to data-port 200 was actually a type-o in the xitami.cfg file in my =
particular release.  It should start at 20.  So this should actually be =
set to -5980, then it should work with all FTP clients.

What this does:
dns2go forwards all www prefixed web requests to port 6080 so my url is =
still www.mydomain.com but, since www.prefixes are forwarded to that =
port my FTP address is just ftp://mydomin.com=20

I really don't think you are ever gonna get port 80 to work on Road =
Runner. =20

I also set up www.mydomain.dns2go.com (which is free too) to get =
everything working until all the dns servers pointed to dns2go so that =
www.mydomain.com would work.

Good Luck,

Jay Aldrich
