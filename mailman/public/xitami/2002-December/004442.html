No, it is not. I've deployed Bugzilla 2.14 successfully on a Windows 
NT machine. The main problem is that you have to hack your way through  
the mail sending routines - they are written with sendmail in mind. 
Because I didn't want to mess with Sendmail for Windows, I replaced 
the calls with the respective functions from the Perl SMTP package. 
And there are several other places of Unix-specific code, which can be 
either bypassed at all or easily modified. 
The other catch is that Bugzilla 2.16 is not available for Windows 
yet. But it seems that 2.14 is capable enough - I believe the main 
problems were some security issues. Of course, the Bugzilla site lists 
more on the differences between versions.
