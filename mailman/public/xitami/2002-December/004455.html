At 17:54 11/12/02, you wrote:

>Hi,
>
>When I run Xitami and its cgi program on a laptop computer, winlogon.exe
>starts taking %90 or more
>of the CPU time (for several seconds) as soon as my cgi program exchanges
>data with Xitami.
>
>But I don't have this problem when they run on a desktop!

Maybe the network configuration is not right. Maybe it's waiting for a WINS 
server?
Even with a faulty network configuration, the computer will boot up. Some 
services like file sharing
can be unavailable, other that don't rely on windows (like Xitami) will be 
operative.
When the cgi needs to access a resource, windows rely on WINS and/or DNS to 
find it. If the server is not found, windows will still find it by 
broadcast, but this can take some time.

Try to duplicate the network configuration from your desktop.
