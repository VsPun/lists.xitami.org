At 06:56 PM 12/15/2002 -0600, John Scrivner writeth:
>I am setting up a voting script that requires specific settings for
>CHMOD of 755 on a file called cspoller.cgi. Attempts at setting the
>CHMOD using FTP clients has proven unsuccessful. Does anyone know if
>CHMOD can be set on files on a Windows 98 computer? If yes can you
>please explain the method? I have tried different FTP clients and
>servers with no success. I am running the most current version of
>Xitami.

Windows 95/98/Me doesn't have the security settings that Linux has.
Windows NT/2000/XP doesn't either, but the security is a whole lot better
than 95/98/Me...

In other words, your script running on Win98 does not need to be chmod'ed
since there is no such thing as permissions in the FAT32 file system.
Almost every file on a Windows system is 775'd (777 in some cases where
someone has decided to share their C drive with Full Control to everyone
via NetBIOS on a LAN...).

Hope this helps!
