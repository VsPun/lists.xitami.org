Sigh!  You seem to be in some difficulty in understanding Internet
terminology.

SOCKS != sockets.

The AnalogX product, inter alia, provides an open SOCKS proxy.  Anyone
connecting to this on the appropriate port can use the proxied machine to
post spam.  The spam is then relayed via the owner's usual ISP's servers
as if originating from him/her.

AnalogX is *dangerous* in the hands of the uninitiated.

Certainly it CAN be configired so as not to have the open SOCKS proxy;
however, the great majority of Windows users would probably have no idea
whether they wanted SOCKS or not, and quite likely leave the system with
the default configuration.  Since the product as originally distributed
left this dangerous option switched on by default, I felt it necessary to
remind potential users not to accpe the default without further thought.

> Then you use it behind the firewall.

What?  Something like ZoneAlarm (which is also dangerous in the hands of
most of its users, although usually only to the extent of worrying them
through reporting false positives)?

99% of "home Windows systems" that are equipped with a "firewall" are
really using a chocolate fireguard, if they hope for the firewall to run
on the same machine that is supposedly being "protected".
