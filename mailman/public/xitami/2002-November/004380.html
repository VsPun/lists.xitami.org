I have Xitami running on port 80. I also have a web-mail server on the
same computer running on port 800. I'm trying to set things up in such a

- If the "Host: " field is mail.mydomain.com, the packets should be
routed to the web-mail server on port 800.
- All other "Host: " values should be handled by Xitami.

As a diagram:


Incoming data-->Port 80|-->Port 800 (webmail; Host: mail.mydomain.com)
                                          |
                                          |-->Port 801(xitami; Host:
<anything else>)

virtual server for mail.mydomain.com, and the index page of this virtual
server redirects the browser to http://mydomain.com:800. But you see,
that's not what I want. Does anybody have any suggestions, either for an
alternative way or for a program that will do such routing?
