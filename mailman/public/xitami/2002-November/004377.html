Hello!

It's not a problem with the webserver.
When asked for something that is not a file but is a valid and allowed 
directory, the server will serve the page that matches the preferences set 
in the configuration (default server properties). Or if there is no match, 
will send the directory list (if allowed).

A proxi server does some more job than simply redirect the queries. It has 
also to track down who ask some page. If the response page is not what was 
expected, the proxi server can have trouble locating the user who requested 
it. Some proxies are better than others (i'm using AnalogX and ProxyPlus)
