Hi,

I am running xitami on Solaris under a constant load (about 10 connections
from IE browser refreshing the same page every 60 seconds) and I am noticing
that, over time, the web server consumes more and more file descriptors.
Eventually, the web server reaches the system limit (1024 open file
descriptors) and exits.

Running utility "pfiles" shows the state of each file descriptor.

Connected sockets look like
 7: S_IFSOCK mode:0666 dev:230,0 ino:28736 uid:0 gid:0 size:0
    O_RDWR|O_NONBLOCK
    sockname: AF_INET 172.16.4.14  port: 80
    peername: AF_INET 192.168.1.198  port: 31602

But there is also an ever increasing number of sockets like -
 9: S_IFSOCK mode:0666 dev:230,0 ino:29651 uid:0 gid:0 size:0
    O_RDWR|O_NONBLOCK
    sockname: AF_INET 0.0.0.0  port: 0

Can anybody suggest what the problem might be?
