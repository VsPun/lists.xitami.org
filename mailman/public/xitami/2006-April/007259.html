There isn't anything special needed. Just install perl and make sure
that you add per to the path - generally speaking taking the default
location(s) works better than customizing but I can't see it failing
even then. I assume you are using activestate perl. If so there really
are no other majic things to do. It would be good to install the HTML
manuals but it sin't compulsory. If you do this then adding an alias
of
perl="c:\perl\html"
to your Xitami settings is noce because it allows you to read the perl
manual from http://localhost/perl/ but it isn't a must do merely a
nice addition

Then use the sample CGI I sent earlier today to test and see if Xitami
fins perl. Ignore the fact that perl is not (on windows) at
/usr/bin/perl.

There really isn't anyhting else to it.
