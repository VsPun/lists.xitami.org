OK you have a problem with perl being called by Xitami. Perl should
not be opening a comand prompt window. I've never seen this behaviour
so I'm not completely clear why it is doing so but I repeat my
suggestion to try and run some perl simpler CGI scripts and see if
they do the same thing.

E.g. create a file called example.cgi in your cgi/bin directory with
the following contents and see if http://localhost/cgi-bin/example.cgi
produces displayable HTML or a blank screen
------- start of example.cgi ---
#!/usr/local/bin/perl

use CGI qw(:standard);

print header;
print start_html('A Simple Example'),
    h1('A Simple Example'),end_html;
------- end of example.cgi ---

(note this file works on my Xitami/Windows/Perl setup just fine).

If it does the same thing (window/blank html) then we probably need a
dump of your environment to figure out what is going wrong ...
