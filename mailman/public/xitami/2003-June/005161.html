Ahh...that helps answer a few things.  There are two solutions to this
problem that make sense and one that might work:

1)  Get a different ISP (probably the better route).
2)  Get another static IP Address with a different domain from your ISP.
3)  Ask your ISP to open a specific port for you (not likely to happen, but
worth asking about).

>I'm not a firewall specialist, but how could a request from Netscape be let 
>through, and a request by IE blocked? (Using the same computer)
>After upgrading from IE 4.0 to IE 6.0 the request was also blocked.
>So i assume it's a Internet explorer thing.

Upgrading IE from 4.0 to 6.0 also applies kernel-level and driver patches
to the Windows OS itself...I've fixed several machines with this approach
that had viruses on them (I removed the viruses first, of course, but this
fixed the collateral damage).  So, when I say it has to do with HINTERNET,
that is the communication mechanism IE uses.  Mozilla/Netscape uses normal
socket communications...hence, it works and IE doesn't.  The HINTERNET
library, during the upgrade process to IE 6.0 probably got replaced with
something "better" that breaks the communication with your server by
sending a NEWS request instead of an HTTP request.  I originally suggested
dropping the firewall temporarily so that you could see the exact packets
coming through in case some packets were being malformed by HINTERNET (most
decent firewalls usually drop malformed packets).
