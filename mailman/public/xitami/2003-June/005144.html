> Bill,
>
> The best (fastest) way to test out your websites, cgi, etc. is locally
> (127.0.0.1).  However, your virtual host must be setup for that address
> (127.0.0.1).  If you have no virtual host pointing explicitly to
127.0.0.1,
> Xitami will default to either the default or baseline (if there is one)
> virtual hosts when you try to browse 127.0.0.1 (http://localhost resolves
to
> 127.0.0.1 by the way, they are interchangeable).
>
> Now suppose you get that working but you want to test it from the
"outside".
> You would need to setup another virtual host that corresponds with your
> cable IP address (or a domain name pointing to that address, if you have
> one).  Or again, Xitami will use it's default/baseline settings because it
> won't find a virtual host corresponding to the IP you are trying to
browse.
>
> I hope this makes some sense!
> -Ross
