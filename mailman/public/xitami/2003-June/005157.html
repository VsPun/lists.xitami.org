Hello !

I know port 119 is used for news.

On any computer, if i type web.verfaillie.com:119 the connection can't be 
made, since the browser expects a NEWS response of some sort.
If i type http://web.verfaillie.com:119 the connection is made, because the 
browser expects a HTTP response.
The problem is the latest version of IE (the version you can download and 
that's included with XP).
It does tot allow http connections over a news port.

It is not a problem with firewalls and ISP: when using a Netscape browser 
(on the same computer), the page can be retrieved.

For now i suggest users to use Netscape. For now the site is only for a 
limited user group.

I'm looking for some register fix to allow IE to connect.

Marc

>Port 119 is a port for news servers.  My guess is that HINTERNET (the
>specific API IE uses for retrieving data from the web) sees port 119 and
>attempts to connect as a news browser instead of HTTP (or refuses the
>connection altogether since it usually doesn't understand the news protocol
>- there are also mapping files on the hard drive that may be affecting
>this).  My suggestions to you are:
>
>1)  Check your firewall settings and maybe temporarily drop the firewall
>altogether to see if it is a firewall issue and not a browser issue.
>2)  Change to another port and have your ISP forward to that port instead
>of 119.
>3)  Verify that your ISP is not blocking port 119 from specific packet types.
>4)  Get a packet sniffer and watch what happens to requests for that
>particular site if you can't change the port number.
>
>Hope this helps!
