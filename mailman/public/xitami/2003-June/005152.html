>Hello !
>
>Does anybody know how to open non-standard ports for web (client side)?
>Webmail is already on port 80.
>
>The internet explorer that ships with w98 and w2k allows the use of 
>non-standard ports.
>IE 6.0 does not allow the use of this port, and i can't use any port above 
>1024 because my isp only allows standard ports.
>
>Is there some registry-trick to force the latest IE version to use 
>non-standard ports?

http://www.yourwebserver.com:85/

I'm not sure what your problem is exactly or what you are trying to
accomplish, but I have IE 6.0 with all security patches applied under
Windows XP Professional and have no problem connecting to any port (< 1024
or otherwise).  This really doesn't seem like a request that should be here
unless you are trying to put Xitami on another port besides 80...in which
case I would suggest making sure Xitami has started properly and isn't
complaining in the log files about not being able to start on the port
number you specify...make sure you counter whatever you add to the HTTP
port with an equivalent decrement to the FTP port or you won't be able to
FTP on port 21.

Hope this helps!
