  I am using Xitami 2.4d10 under Windows 2000 Pro and had been fighting for
months with how Chrome (only Chrome) completely ignored external css files.

  But all was fine after I told Xitami yesterday that .css files were of
type text/css in defaults.cfg:
  [Mime]
       css=text/css

  Now it does not matter what <!DOCTYPE> I use:
  - HTML 4.01 strict
  - HTML 4.01 transitional
  - XHTML 1.0 Transitional
  - even with a missing <!DOCTYPE> line the external CSS works !


  My current working test site setup:

  HTML 4.01 strict in pages:
  <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN"
"http://www.w3.org/TR/html4/strict.dtd">

  In the <head> I specified charset:
  <meta http-equiv="Content-Type" content="text/html;CHARSET=iso-8859-1">

  In the external CSS the matching charset specification was the first line:
  @charset "iso-8859-1";

  External stylesheet reference was:
  <link rel="stylesheet" type="text/css" href="../css/style.css">

  How is your page wacky?  Can you confirm that 2.4d6 understands the new
lines in defaults.cfg ?

  I will send you the link to the working page.
