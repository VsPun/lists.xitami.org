  Hey all,

  The "errors" section of the Xitami admin has predefined values for 400, 401, 404 errors etc, but nothing for 301 redirects.  I am in the process of renaming my static links from ID based links such as /postShow/3/26/0/383472 to name based links like /postShow/chat-groups/local-politics/page-0/id-383472 or something to that affect.

  However, I don't want search engines such as google or yahoo that have already crawled and indexed all the ID based links to error out, or drop the indexed link, after I make the change - I have over 100,000 indexed pages.  This is more important for the search engines than it is for the users. If the search engine sees that the ID based links are gone without info one where they have gone to, they will drop all the indexed links and it could take weeks if not months for them to recrawl the new links.

  Other webservers such as apache can use 301 redirects to basically tell the SE, the user or the browser, that although the old links have changed the content still exists - its just simply at a new location.  The SEs will simply change the index link versus dropping it completely.  Doing a simple "redirect=0" in the HEAD or using JS "location.href=..." isn't good enough because to a search engine the original page still exists but now its content is simply gone or empty.

  Is it possible to define errors or anyone know of a way to code in 301 redirects at the server level?

  Thanks.
