Hi Peter,

There's no need at all to alter the extension settings in the php.ini file
unless you atually move the directory the extenstions are placed in. php parses
the php.ini file at startup and looks there expicilty to load extensions.
There's also no need to place the extension directory in the path (although it
shouldn't do any harm).

With php.exe in the Xitami directory (eg if Xitami is installed in C:\Xitami
this is where php.exe should be placed - NOT in any subdirectory such as
C:\Xitami\cgi-bin etc) the filter should simply be set to

.php php.exe

without any paths.

I was assuming from your original post that you weren't getting error
messages... just blank pages, is this correct ???
