> Does someone have the AXS Site Tracking System (http://www.xav.com/) up and running. ...whenever i try ax.pl (the script that actually counts the visits) I'm always getting...  ax.pl - working okay - no logging command received - use ?debugme query string for more info. 

You don't normally call ax.pl directly. If you do, you get that message. 

You embed ax.pl in an IMG (or SSI) tag on the page(s) you want counted.
There is a link to instructions in localhost/cgi-bin/axs/ax.pl?debugme

I could not get it to work at first. There seem to be typos in the
documentation. I use Xitami but this isn't an Xitami issue. 

"if you are not using SSI, place this code on each of your HTML pages:

 <SCRIPT LANGUAGE="JavaScript">
 <!--
   document.write("<IMG SRC=\"\cgi-bin\axs\ax.pl?trans.gif&ref=");
   document.write(document.referrer);
   document.write("\" HEIGHT=1 WIDTH=1>");
 // -->
 </SCRIPT><NOSCRIPT>
 <IMG SRC="\cgi-bin\axs\ax.pl?trans.gif" HEIGHT="1" WIDTH="1">
 </NOSCRIPT>  

I'm pretty sure the DOS-type path slashes "\" MUST be changed to
unix/perl/javascript type slashes "/". (Even on Windows.) However, the
escape slashes in front of quotes must be back\slashes. 

This is my interpretation. I assume (for testing) that all clients have
javascript so I only have to mess with one part. (I also highlighted the
"invisible" image so I could verify it was being shown.)

 <SCRIPT LANGUAGE="JavaScript">
 <!--
   document.write("<IMG SRC=\"/cgi-bin/axs/ax.pl?trans.gif&ref=");
   document.write(document.referrer);
   document.write("\" HEIGHT=10 WIDTH=10 border=2>");
 // -->
 </SCRIPT>

Since this seems to be a problem with AXS's skimpy instructions, not any
interaction with Xitami, you should probably seek further help on their
support forum http://www.xav.com/scripts/forum.pl
