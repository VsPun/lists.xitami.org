This would possibly suggest something is not quite right with stdin (where
POST variables are placed) under W2K (Ewen...any known issues here?).  Have
you tried phpinfo() to see if PHP is receiving anything at all from the
Xitami webserver?

Also, are you running PHP3 or PHP4?  There are significant differences
between the two that might be causing the problem under Xitami.  Try the
latest 4.0.6 release build if you are running any version other than that.

As long as you don't run mod-ssl, you will be fine (maybe one or two Apache
child process crashes a day).  If you run mod-ssl, watch out, mod-ssl,
mod-php (or even PHP in CGI mode), and Apache don't mix well (they crash
about once every 10 page requests).

The headers of every e-mail to the Xitami mailing list contain:

List-Help: <mailto:xitami-request@lists.xitami.org?subject=help>
List-Post: <mailto:xitami@lists.xitami.org>
List-Subscribe: <http://lists.xitami.org/mailman/listinfo/xitami>,
	<mailto:xitami-request@lists.xitami.org?subject=subscribe>
List-Id: Xitami Users General Discussion <xitami.lists.xitami.org>
List-Unsubscribe: <http://lists.xitami.org/mailman/listinfo/xitami>,
	<mailto:xitami-request@lists.xitami.org?subject=unsubscribe>
List-Archive: <http://lists.xitami.org/pipermail/xitami/>

You want the "List-Archive" link.  If you can't read the headers in "The
Bat!" (Your mail client), then it is probably time to get something else.
I highly recommend Eudora Light v3.0.6 (I can't stand the interface on
anything newer from Qualcomm and all the other freeware clients can't do
everything Eudora can with the ease of Eudora - and 3.0.6 is about the only
non-HTML mail client out there anymore).
