In message <003701c121d5$d90d6950$9600000a@MINE1>, "Toernooi Assistent" writes:
>after upgrading Xitami from 4d7 to 5b5, loading of an host xml file by an
>html page with jscript (xmldom) stopped working. 

I take it you mean 25b5.  In 25b5 configuration, XML files are marked
for handling by the xixxml extension (which runs GSL over them).  If
you just want to serve out XML files you need to defeat this, which
currently can only be done by commenting out the [wsx] .xml=xixxml
line in _xitami.cfg_.  (You can't override it in the other config files;
there's a bug against this.)

24d9 shouldn't have this problem, as it doesn't try to handle XML files
in any particularly special manner, it just sends them out.

BTW, the debug log can be very useful for tracking this sort of thing
down. (Providing you set [server] debug=1 and try it again.)
