In message <4137535C.60908@coaxial.ca> Josh A writes:
> Jill Douglas wrote:
> 
> >Someone recently asked me how it is that a site that I host for them, is
> >unable to be obtained if they leae off the www. component of the URl when
> >they visit.  Is this somethign that is correctable?
> >
> I have run into this problem before when I don't make an extra entry for 
> virtual hosts. Basically when I add a new virtual host 
> www.somedomain.com  =  somedomain.cfg I then go into the default virtual 
> hosts table and add "somedomain.com = somedomain.cfg"
> 
> that way you end up with :
> 
> "somedomain.com = somedomain.cfg" and
> "www.somedomain.com = somedomain.cfg"

That is a necessary step, of course.  However, what this customer of
Jill's also needs to do is ensure that the DNS entries for the two
domains both point to the single machine.  Remember, there are people out
there too stupid to understand that www.somewhere.com is NOT the same
thing as somewhere.com --- unless the DNS makes it so.
