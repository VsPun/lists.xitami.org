Just a quick thought:

    My regular Xitami server (p166 orphan) runs flawlessly for months at a time, although I'm not pushing the traffic you guys are.  However, in order to add some disk space without having everything go down for a while, I moved all server stuff to an older computer (docked Toshiba 205 Laptop) with Win98SE and test ran it.  CRASHed overnight!  My working laptop is also a 205 and since I upped it to Win98SE, I crash regularly, usually with the network pc card installed but sometimes not.

  The best I can figure it is that somewhere a wakeup command doesn't process right and it locks up.  Usually the computer has been inactive for a long while (hour +) and has screen and disks timed off.  Sometimes it wakes up, sometime it doesn't.  Sometimes it went to sleep with IE on screen, sometimes not.  None of this seems Xitami related although an active TCP/IP connection is suspicious.  Running test right now without any software loaded.

I thought display drivers were involved but not.  Thought maybe a TCP/IP hangup (memory leak?).  Just not everything wakes up after a nap. ACPI maybe?  Power Management?
