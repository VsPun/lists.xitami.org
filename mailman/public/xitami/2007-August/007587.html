I think your server fails because of the database access. My server can
handle a peak of 25 requests (not connections) per seconds, and it's only a
pentium III 600MHz. 1/4 of the requests are cgi-bin requests.
The requests are local (my router would crash well before attaining that
number).

Since the footprint of Xitami is so small, i suggest you put Xitami on the
MySql server itself and route the requests to the server instead of using
two servers. I don't know how much data is exchanged between sql database
and server, but i don't think it's a good combination.
