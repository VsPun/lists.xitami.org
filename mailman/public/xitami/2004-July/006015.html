I'm trying to test a Perl CGI program (form processor, POST) on localhost
(WinXP).  I'm experiencing a series of problems:

1)  When I invoke the pgm.pl from a form with
action="http://localhost/cgi/alwmsg.pl" it fails with a "Page not found"
error; however, if I type the same URL into a browser the program runs, but
(see problem 2)
2)  When the program executes, it seems to be simply executing under
Windows. Running the manual debugger, I see that none of the HTTP variables
are in ENV, and none of the form fields are passed in.

I've set up a CGI Alias called "localalw" and associated the exact directory
where the Perl script is.
 
I've also set two environmental variables to try to invoke the ActiveState
Komodo debugger instead of the default one:
 
PERL5LIB
c:\Program Files\ActiveState Komodo 2.5
PERLDB_OPTS
RemotePort=localhost:9011 PrintRet=0 
 
But this does not work; default (command line) debugger still gets invoked.

My Goal: When I invoke the Perl program from a HTML page form, I would like
the debugger (Komodo) to be invoked for the program, normal HTTP/CGI
environment variables to be set up for the program, and the output from the
program (an HTML page) returned in the browser.

Any idea why HTTP isn't handling my Perl script?
 
Thanks for any help.
