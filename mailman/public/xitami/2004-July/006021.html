>
>actually, it is the Xitami.cfg that is supposed to be hardwired and 
>your not supposed to edit that.  In Xitami.cfg is a line pointing to 
>your true config
>- default.cfg (or whatever name you want to change it to.).  The 
>default.cfg should have all your baseline configs which should override 
>the xitami.cfg configs.  If you edit the xitami.cfg and mess things up 
>then you could potentially have to reinstall everything.  I have 
>actually corrupted mine before b/c I kept toying with it - took me a 
>week to figure out it was corrupted (it was the last thing I thought of 
>why things weren't working).
>
>What version of xitami are you using - I recall hearing some early XP 
>versions of xitami that had CGI (and/or PHP) problems.  Maybe someone 
>else can comment on this.
>
>The cg-bin path offers two options - 1) /cgi-bin is the root path (or a 
>variation of it), as in http://blah.com/cgi-bin/test.pl and 2) /cgi-bin 
>can be anywhere in the URL as in, 
>http://blah.com/site/cgi-bin/myscripts/test.pl
>- but the path, obviously, must exist.  Remember that webservers need 
>to know where the CGI scripts reside - unlike PHP where they can be 
>executed from anywhere.
>
>The two MAIN cgi configure lines are:
>Main CGI directory:  	cgi-bin
>      * (or where ever you store them)
>      * all your scripts must be in that directory or a sub-dir of that 
>main directory
>      * it is also wise to have that directory sit at the same level as 
>your "webpages" root dir CGI URLs start with:  /cgi-bin
>      * As I do, this makes the full URL path to the cgi directory 
>start with /cgi-bin
>            * correct: http://blah.com/cgi-bin/test.pl
>             * incorrect: http://blah.com/scripts/cgi-bin/test.pl
>      * you can anchor it another spot, but why?  :-)
>
>There is a sub-CGI configuration too, but don't confuse that with the 
>root configuration
>Working directory:  	'-'
>      * '-' means use program directory, refer to main config Match 
>/cgi-bin anywhere in URL?
>       * if selected this allows you to put cgi-bin anywhere in the URL 
>path, but can be confusing at first
>        * if not selected (recommended) this anchors all URLs to 
>http://blah.com/cgi-bin/test.pl
>
>g-luck.
