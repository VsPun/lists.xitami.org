SSL is extremely CPU intensive as calculating/generating keys and
behind-the-scenes session management is a very complex process.  You will
want to have enough processing power behind the server to handle all
SSL/TLS and normal HTTP requests.  Too many SSL/TLS requests can quickly
drag a server to its knees and HTTP will be less responsive as a result.

You mentioned Windows Me.  I tried at one time to assign multiple IPs to
one network card by manually editing the registry and it worked somewhat.
I don't know how much mileage you will get from it.  A quick search on
Google found this link:

<http://www.wown.com/j_helmig/multiipa.htm>

Hope this helps!
