> At 10:27 AM 4/1/2003 -0500, Vince Napoli writeth:
> >When running Xitami Internet Explorer is not automatically refreshing
> >the
> default.html 
> >even though we have IE set to automatically refresh the page.
> >
> >Is there something in the web server that dictates this action?
> 
> Check your server logs for HTTP '304' responses when you request the
> 'default.html' page.  That response indicates to the browser that the
> page in its cache is current and to use the cached copy instead of
> downloading the page from the server.  This cuts down on bandwidth
> usage and increases the overall speed of retrieving the webpage, but
> does introduce fun problems for developers to deal with.
> 
> Hope this helps!
