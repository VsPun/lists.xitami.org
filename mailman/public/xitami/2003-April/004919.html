At 12:06 PM 4/8/2003 -0500, Tom Baker writeth:
>Hey folks!
>Recently I changed to the service version of Xitami 2.4d9. I had been using
>the app version, and was having stability problems. So far (at least), the
>service version seems stable. But here's a strange catch:
>Upon system startup, the Xitami service loads, but does not accept incoming
>connection attempts. If I restart the service, everything is AOK.
>
> System specs:
>P2-350
>Win2k with all latest patches
>Static IP business DSL
>DNS server
>Mercury/32 mail server
>ZoneAlarm Pro
>
>So here's my guess: Xitami service is loading before ZoneAlarm. When
>ZoneAlarm starts, it disrupts the port monitoring by Xitami.
>
>So... If this is true, does anybody have an idea on how to fix it?

Yup.  That happens.  There's a way to fix it, but it requires writing code
(as far as I know).

Delay start Xitami by making it a dependency of ZoneAlarm in the Services
Control Panel.  To do this, you need to write program that calls
OpenServiceManager(), OpenService(), ChangeServiceConfig(), and
CloseServiceHandle().  I don't have time at the moment to do this, but
maybe someone else here will write a quick program that does that for you.
If you want to wait a week and e-mail me to remind me about this (if no
one's done this yet), I'll write it.

Hope this helps!
