I have several virtual domains set up and on each one of them I have allowed 
web-based administration for each domain's admin.  However, I am having some 
serious problems accessing the admin sections of the domains.

Problem 1:
I login as Superuser to the default domain and verify virtual domain 
"blah.com" has "Web Based Admin" checked in its Security section.  I confirm 
the blah.aut file has the /admin=HELLO password set.  The alias is pointing 
to: /admin = /xitami/blah/htdocs/admin.  The Superuser field for domain 
blah.com is empty.

* I then open a new browser and connect to: www.blah.com/admin
* It asks for user name and password, I enter: admin , HELLO.
* It then gives me the FULL Superuser access to every virtual domain on the 
server.  From the blah.com admin login I can control and change and edit 
info for every other domain.

Is this a BUG?  Or am I doing something wrong?

Problem 2:
I believe this is a problem relating to the first one above.  If I define a 
FTP user for the default domain that user shows up in ALL the password FTP 
password sections for all the other Virtual domains.

Default domain FTP login: User: jdoe  Pass: hello  Path: /

Then under virtual domain blah.com I go to add FTP users and the account I 
just defined for "jdoe" above is now present in the Blah.com FTP password 
file.  Whats up with that?

Please help, this has been driving me nuts for along time.
